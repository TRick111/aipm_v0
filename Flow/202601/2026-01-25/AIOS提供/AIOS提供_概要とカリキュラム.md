## 1. AIOSを使う背景

### 現状の課題（ChatGPT中心の運用）
- ChatGPTで多くの作業を行っているが、**成果物がChatGPT内に残る**ため、作ったものを貯めて活用することが難しい
- Google Drive連携等は可能だが、ファイル管理・整理が煩雑になりがち
- 会話が途切れると**コンテキスト（前提情報）が引き継がれない**

### AIの進化に伴う新しい課題
- 現在のAIは**コンテキストウィンドウが非常に大きい**（数十万トークン）
- 精度よくタスクを実行するには、**いかにコンテキストに適切な情報を詰め込めるか**が重要
- ChatGPTの会話形式では、必要な情報を毎回手動で渡す必要がある

### AIOSで解決すること
- 成果物をファイルとして**蓄積・整理・再利用**できる
- AIが自動で**過去の成果物・背景情報を参照**して作業できる
- **長期コンテキスト保持**と**作業の連続性**を実現

## 2. AIOSの概要

### 基本コンセプト
- **作業はすべてCursor上で行う**
- AIとの対話を通じて成果物を作成し、ファイルとして保存
- AIが「どのプロジェクトか」「どのファイルを参照すべきか」を自動判断

### 3つの領域（フォルダ構造）

| 領域 | 役割 | 特徴 |
|------|------|------|
| **Flow** | 日々の作業場所（WIP） | 月/日付で整理。作業中のドラフト・メモを置く |
| **Stock** | 確定版の保管場所 | AIが参照する"正"。Program/Project単位で整理 |
| **Meetings** | 会議記録の集約 | 横串で保管。プロジェクトからはリンク参照 |

### 運用の基本原則
- **ユーザーは最小入力**（箇条書き・URL・メモでOK）
- **AIが作成/更新を実施**し、必要ならユーザーが微修正
- **Flowで作業 → 確定したらStockへ保存**

## 3. 機能一覧

### 3.1 コア機能

| 機能 | できること | 具体例 |
|------|------------|--------|
| **AIとの対話** | 自然言語でAIに指示し、成果物を作成できる | 「提案書を作って」「投稿案を3つ考えて」 |
| **プログラム単位での情報管理** | 事業/テーマごとに関係者・背景・取引先情報を保管し、AIが自動参照できる | 「バンコクPonさん案件」の背景・関係者を把握した上で作業 |
| **プロジェクト単位での情報管理** | プロジェクトごとに背景・目的・進行状況を保管し、作業時に自動参照できる | 「AIOS提供」の目的・現状を踏まえて成果物作成 |
| **会議記録の一元管理** | 会議トランスクリプト/議事録を集約し、関連プロジェクトに反映できる | 会議の決定事項・宿題をプロジェクトに自動反映 |
| **日次タスク管理** | 今日やることを整理し、プロジェクトと紐付けて管理できる | 「今日やること」を伝えるだけで日次タスクファイル生成 |
| **成果物の蓄積と再利用** | 作成した成果物をファイルとして保存し、後から参照・再利用できる | 過去の投稿案・提案書を参照して新しいものを作成 |
| **Gitによるバージョン管理** | すべての変更履歴を保持し、任意の時点に戻れる | 誤って削除した内容の復元、変更履歴の確認 |

### 3.2 AI自動処理

| 機能 | できること |
|------|------------|
| **プロジェクト自動特定** | 「〇〇の提案書作って」と言うだけで、対象プロジェクトを自動判断 |
| **関連情報の自動参照** | プロジェクトの背景・過去成果物を自動で読み込み、作業に反映 |
| **成果物の自動保存** | 作成した成果物を適切なフォルダに自動保存、Indexも自動更新 |
| **進捗の自動記録** | 作業内容・変更履歴をlog.mdに自動記録 |

### 3.3 マルチデバイス対応

| デバイス | 役割 | 用途 |
|----------|------|------|
| **PC** | メイン作業環境 | Cursorで作業・AIへの指示 |
| **モバイル** | 外出先での作業 | Cursorモバイル版で指示・確認（※検証中） |
| **タブレット** | 閲覧・軽作業 | Obsidian/Cursorで閲覧・簡易編集 |

### 3.4 Obsidian連携

AIOSのファイル閲覧・軽微な修正は**Obsidian**で行う。

**運用フロー:**
```
AIに指示（Cursor） → Obsidianで確認 → 修正（ObsidianまたはAIに指示）
```

| 機能 | 説明 |
|------|------|
| **ファイル閲覧** | Obsidianのリンク機能・グラフビューで関連ファイルを可視化 |
| **軽微な修正** | Obsidian上で直接編集可能 |
| **自動Commit/Push** | Obsidian Gitプラグインで定期的にバックアップ |
| **モバイル閲覧** | Obsidianモバイルアプリで外出先から確認 |

### 3.5 定型作業

定期的に実行するタスクは、AIOSの枠組み内で**定型化**する。

**定型作業の構造:**
```
Stock/
  定型作業/              ← プログラム
    <定型作業名>/        ← プロジェクト
      README.md          ← 作業の目的・頻度・手順概要
      ProjectIndex.yaml
      log.md
      指示書.md          ← AIへの具体的な指示内容
```

| 機能 | 説明 |
|------|------|
| **指示の再利用** | 毎回同じ指示を書く必要がない |
| **ワンコマンド実行** | 「〇〇の定型作業を実行して」で完了 |
| **改善履歴の蓄積** | log.mdに実行履歴・改善点が残る |

**定型化の例:**
- 週次レポート作成
- Instagram投稿案作成（週3回）
- 会議アジェンダ作成

### 3.6 MCP活用

**MCP（Model Context Protocol）** を活用して、外部サービスとAIを連携させる。

| 機能 | できること | 具体例 |
|------|------------|--------|
| **外部サービス連携** | AIから直接外部サービスを操作できる | Notion、Google Calendar、Slackなど |
| **業務特化のMCP** | PONさんの業務に合わせた有効なMCPを導入 | ヒアリングを通じて選定・設定 |
| **ワークフロー自動化** | 複数サービスをまたいだ作業を自動化 | 「予定を確認して会議アジェンダを作成」 |

### 3.7 モバイル活用

PC版とは別に、**モバイルデバイスからの運用**を定義する。

| 機能 | できること | 備考 |
|------|------------|------|
| **Cursorモバイル版での指示** | 外出先からAIに指示を出せる | ※検証中 |
| **Obsidianモバイルでの閲覧** | 外出先から成果物を確認できる | Git連携で最新状態を同期 |
| **軽微な修正・追記** | 移動中にメモや修正ができる | Obsidianで直接編集 |

**想定シーン:**
- 移動中にアイデアをメモ → 後でAIに展開指示
- 外出先で成果物を確認 → 修正指示をメモ
- 会議中にリアルタイムでメモ → 後でAIに整理指示

### 3.8 他サービスとの使い分け

AIOSでできないこと・他サービスが得意なことを明確化し、**使い分けを定義**する。

| 用途 | 推奨サービス | AIOSの役割 |
|------|--------------|------------|
| **スライド作成** | Manus | AIOSで構成案・原稿作成 → Manusで仕上げ |
| **深い調査・リサーチ** | ChatGPT（DeepResearch） | AIOSで調査依頼を整理 → ChatGPTで実行 → 結果をAIOSに保存 |
| **画像生成** | Midjourney、nanobanana | AIOSでプロンプト作成・管理 |
| **動画編集** | 専用ツール | AIOSで企画・構成案作成 |

**使い分けの原則:**
- **AIOSは「司令塔」**: 作業の指示・成果物の保管・情報の整理
- **他サービスは「専門家」**: 特定タスクの実行
- **成果物はAIOSに集約**: どこで作っても最終成果物はStockに保存

## 4. システム構成

### 4.1 必須ファイル群

| ファイル | 役割 | 誰のため |
|----------|------|----------|
| `MasterIndex.yaml` | 全プロジェクトの地図（どのプロジェクトを見るか判断） | AI向け |
| `ProjectIndex.yaml` | プロジェクト内ファイルの地図（どのファイルを見るか判断） | AI向け |
| `README.md` | プロジェクトの背景・目的・現状・次アクション | 人間+AI向け |
| `log.md` | 更新履歴（いつ・何を・なぜ） | AI向け |
| `mmdd_daily_tasks.md` | 日次タスク管理 | 人間+AI向け |

### 4.2 フォルダ構造

```
<AIOS Root>/
├── Flow/                          ← 日次作業（WIP）
│   └── YYYYMM/
│       └── YYYY-MM-DD/
│           └── mmdd_daily_tasks.md
├── Stock/                         ← 確定版
│   ├── MasterIndex.yaml
│   └── <Program>/
│       └── <Project>/
│           ├── README.md
│           ├── ProjectIndex.yaml
│           └── log.md
├── Meetings/                      ← 会議記録（横串）
│   └── YYYY-MM-DD_<title>.md
└── .cursor/rules/                 ← AIルール
    └── aios/
```

## 5. 環境・設定

### 5.1 必要な環境

| 項目 | 内容 |
|------|------|
| エディタ | **Cursor**（AI機能統合のVSCode派生） |
| 閲覧ツール | **Obsidian**（ファイル閲覧・自動Git連携） |
| バージョン管理 | **Git**（Obsidian Gitプラグインで自動Commit/Push） |

### 5.2 初期セットアップで作成するもの

1. **フォルダ構造**
   - `Flow/`、`Stock/`、`Meetings/` の3フォルダ
2. **MasterIndex.yaml**（Stock直下）
   - 全プログラム/プロジェクトの概要・キーワード
3. **Cursorルール**（`.cursor/rules/`）
   - AIOSの運用ルール（AIが常時参照）
4. **Obsidian設定**
   - Obsidian Gitプラグインの設定（自動Commit/Push）

### 5.3 運用開始後に自動生成されるもの

- プロジェクト配下の `README.md` / `ProjectIndex.yaml` / `log.md`
- 日次フォルダ `Flow/YYYYMM/YYYY-MM-DD/`
- 日次タスク `mmdd_daily_tasks.md`

## 6. ユースケース

### 6.1 日常業務での活用例

| ユースケース | AIへの入力例 | 期待される動作 |
|--------------|--------------|----------------|
| 今日のタスク整理 | 「今日やること：〇〇、△△」 | 日次タスクファイル生成、対象プロジェクト紐付け |
| Instagram投稿案作成 | 「採用向け投稿案を3つ作って」 | 過去の投稿方針・トーンを参照して案を生成 |
| 会議準備 | 「〇〇MTGのアジェンダ作って」 | 過去会議・宿題を参照してアジェンダ生成 |
| 議事録反映 | 「この会議メモを反映して」 | Meetings保存 + 関連プロジェクトREADME更新 |
| 資料作成 | 「〇〇プロジェクトの提案書作って」 | 過去成果物・背景を踏まえてドラフト作成 |
| 定型作業実行 | 「週次レポートの定型作業を実行」 | 指示書を読み込み、自動で作業実行 |

### 6.2 ChatGPT運用との違い

| 観点 | ChatGPT | AIOS（Cursor） |
|------|---------|----------------|
| 成果物の保存 | 会話内に残る | ファイルとして蓄積 |
| コンテキスト維持 | 会話ごとにリセット | 自動で過去情報を参照 |
| 作業の連続性 | 毎回説明が必要 | 前回の続きから再開可能 |
| 整理・検索 | 困難 | フォルダ・Index・キーワードで整理 |
| 定型作業 | 毎回同じ指示が必要 | 指示書を参照して自動実行 |
