# AIOS提供 躓きポイント一覧

作成日: 2026-01-25  
対象: 近藤Ponさん（Rio's Innovation）

---

## 概要

AIOSを習得・運用する際に想定される「障害・躓きポイント」を一覧化。
各ポイントに対する**予防策**と**対処法**を整理する。

---

## 1. 環境構築・セットアップ関連

### 1-1. Cursorのインストールが難しい

| 項目 | 内容 |
|------|------|
| 発生タイミング | 第1回講義 |
| 症状 | ダウンロード方法が分からない、インストールが止まる |
| 原因 | PC操作に不慣れ、権限の問題 |
| 予防策 | 事前にインストール手順書を共有、画面共有でサポート |
| 対処法 | 講義時間内にリモートサポートで解決 |

### 1-2. Obsidian Gitの設定が複雑

| 項目 | 内容 |
|------|------|
| 発生タイミング | 第1回講義 |
| 症状 | Git連携がうまくいかない、自動同期が動かない |
| 原因 | Gitの概念が分からない、認証設定ミス |
| 予防策 | 設定済みの環境を事前構築、手順書を細かく作成 |
| 対処法 | 最初は手動同期で運用、後から自動化 |

### 1-3. フォルダ構造が理解できない

| 項目 | 内容 |
|------|------|
| 発生タイミング | 第1回講義 |
| 症状 | どこに何を置けばいいか分からない |
| 原因 | ファイル/フォルダの概念に不慣れ |
| 予防策 | 視覚的な図解を用意、実際のフォルダを見せながら説明 |
| 対処法 | 最初はAIに「〇〇を保存して」と任せる |

---

## 2. AIとの対話関連

### 2-1. 何を指示すればいいか分からない

| 項目 | 内容 |
|------|------|
| 発生タイミング | 第1-2回講義 |
| 症状 | AIチャットを開いたが、何を打てばいいか分からない |
| 原因 | ChatGPTでの経験はあるが、Cursorでの使い方が分からない |
| 予防策 | 具体的な指示例を多数用意、「こう言えばこうなる」を見せる |
| 対処法 | 最初は一緒に指示を考える、成功体験を積む |

### 2-2. AIの出力が期待と違う

| 項目 | 内容 |
|------|------|
| 発生タイミング | 継続的 |
| 症状 | 思っていた内容と違うものが出てくる |
| 原因 | 指示が曖昧、コンテキスト不足 |
| 予防策 | 「具体的に指示する」「修正指示を出す」を繰り返し伝える |
| 対処法 | 「〇〇に変えて」「△△を追加して」で修正 |

### 2-3. 英語で返答される

| 項目 | 内容 |
|------|------|
| 発生タイミング | 初期 |
| 症状 | AIが英語で回答してくる |
| 原因 | ルール設定ミス、言語指定なし |
| 予防策 | ルールファイルで「日本語で回答」を設定 |
| 対処法 | 「日本語で答えて」と追加指示 |

---

## 3. 運用・習慣化関連

### 3-1. Cursorを開く習慣がつかない

| 項目 | 内容 |
|------|------|
| 発生タイミング | 講義後 |
| 症状 | つい従来のChatGPTを使ってしまう |
| 原因 | 新しいツールへの移行抵抗、習慣の問題 |
| 予防策 | 毎日使う必然性を作る（日次タスク管理等） |
| 対処法 | ChatGPTを開いたらCursorを思い出すルール、定期MTGで確認 |

### 3-2. Flow→Stock確定反映を忘れる

| 項目 | 内容 |
|------|------|
| 発生タイミング | 運用開始後 |
| 症状 | Flowに作業結果が溜まり、Stockが更新されない |
| 原因 | 確定反映の習慣がない、タイミングが分からない |
| 予防策 | 日次タスク完了時に確定反映をルーティン化 |
| 対処法 | 定期MTGで確認、まとめて確定反映 |

### 3-3. プロジェクトを指定せずに作業してしまう

| 項目 | 内容 |
|------|------|
| 発生タイミング | 運用開始後 |
| 症状 | AIが過去情報を参照せず、毎回ゼロから作成 |
| 原因 | プロジェクト指定の習慣がない |
| 予防策 | 「〇〇プロジェクトの」を付けるよう繰り返し伝える |
| 対処法 | AIに「どのプロジェクトですか？」と聞かせるルール追加 |

---

## 4. 同期・バージョン管理関連

### 4-1. PC↔モバイル間で同期されない

| 項目 | 内容 |
|------|------|
| 発生タイミング | モバイル活用開始後 |
| 症状 | PCで編集した内容がモバイルに反映されない |
| 原因 | Gitの自動同期が止まっている、コンフリクト |
| 予防策 | 同期状態の確認方法を教える、定期的にチェック |
| 対処法 | 手動でPull/Push、コンフリクト解消（サポート） |

### 4-2. ファイルが消えた（ように見える）

| 項目 | 内容 |
|------|------|
| 発生タイミング | 継続的 |
| 症状 | 作業したファイルが見つからない |
| 原因 | 別のフォルダに保存された、同期されていない |
| 予防策 | ファイル保存時に場所を確認する習慣 |
| 対処法 | 検索機能で探す、Git履歴から復元 |

### 4-3. 間違えて上書きしてしまった

| 項目 | 内容 |
|------|------|
| 発生タイミング | 継続的 |
| 症状 | 大事な内容を消してしまった |
| 原因 | 誤操作、AIへの指示ミス |
| 予防策 | バージョン管理の安心感を伝える |
| 対処法 | Git履歴から復元（サポート） |

## 5. MCP・拡張機能関連

### 5-1. MCPが動かない

| 項目 | 内容 |
|------|------|
| 発生タイミング | MCP活用開始時 |
| 症状 | 「メールを送って」と言ってもエラーになる |
| 原因 | MCP設定ミス、認証切れ |
| 予防策 | 講義時に動作確認、設定手順を明確化 |
| 対処法 | 設定を再確認、再認証 |

### 5-2. MCPの概念が理解できない

| 項目 | 内容 |
|------|------|
| 発生タイミング | MCP活用開始時 |
| 症状 | なぜ外部サービスと連携できるのか分からない |
| 原因 | 技術的な仕組みへの理解不足 |
| 予防策 | 「仕組みは分からなくてOK、使い方だけ覚えて」と伝える |
| 対処法 | 実際に動かして成功体験を積む |

## 6. 心理的・モチベーション関連

### 6-1. 覚えることが多くて挫折

| 項目 | 内容 |
|------|------|
| 発生タイミング | 初期〜中期 |
| 症状 | 「難しい」「面倒」と感じて使わなくなる |
| 原因 | 一度に多くを教えすぎ、成功体験不足 |
| 予防策 | 最初は最小限の機能だけ、段階的に追加 |
| 対処法 | 成功体験を積み重ねる、効果を実感させる |

### 6-2. 従来のやり方に戻ってしまう

| 項目 | 内容 |
|------|------|
| 発生タイミング | 講義後 |
| 症状 | 結局ChatGPTやメモ帳に戻る |
| 原因 | 新しい方法のメリットを実感できていない |
| 予防策 | 「これはAIOSでしかできない」価値を見せる |
| 対処法 | 定期MTGで使用状況確認、成功事例を共有 |

### 6-3. 忙しくて学習時間が取れない

| 項目 | 内容 |
|------|------|
| 発生タイミング | 継続的 |
| 症状 | 講義以外の時間で触れない |
| 原因 | 経営者として多忙、優先度が下がる |
| 予防策 | 「日常業務の中で使う」設計、別途時間を取らない |
| 対処法 | 実際の業務タスクをAIOSで実行する形式 |

## 7. 経営者特有の躓き

### 7-1. 細かい操作をする時間がない

| 項目 | 内容 |
|------|------|
| 発生タイミング | 継続的 |
| 症状 | PCに向かう時間自体が限られる |
| 原因 | 経営業務が優先、移動が多い |
| 予防策 | モバイル活用を早期に導入、最小操作で成果が出る設計 |
| 対処法 | 定型作業化で操作を最小化 |

### 7-2. 秘書・スタッフに任せたい

| 項目 | 内容 |
|------|------|
| 発生タイミング | 中期以降 |
| 症状 | 「自分でやるより誰かに任せたい」 |
| 原因 | 経営者として委譲思考が強い |
| 予防策 | AIOSが秘書的役割を担うことを説明 |
| 対処法 | 将来的にスタッフへの展開も視野に |

### 7-3. 効果が見えにくい

| 項目 | 内容 |
|------|------|
| 発生タイミング | 中期 |
| 症状 | 「結局何が良くなったの？」と感じる |
| 原因 | 定量的な効果が見えにくい |
| 予防策 | ビフォー/アフターを記録、具体的な成果物を見せる |
| 対処法 | 「これがAIOSなしでは作れなかった」を振り返る |

## 躓きポイントの重要度分類

| 重要度 | 躓きポイント | 対応優先度 |
|:------:|--------------|:----------:|
| **致命的** | 環境構築失敗、同期不能、習慣化失敗 | 最優先で予防・対処 |
| **高** | AIとの対話困難、確定反映忘れ、モチベーション低下 | 早期に対処 |
| **中** | MCP動作不良、細かい操作の時間不足 | 段階的に対処 |
| **低** | 効果実感の遅れ、委譲願望 | 中長期で対処 |

## 講義での予防策

| 講義回 | 重点的に予防する躓きポイント |
|:------:|------------------------------|
| 第1回 | 環境構築失敗、フォルダ構造理解不足、最初の指示が出せない |
| 第2回 | Cursorを開く習慣、プロジェクト指定忘れ |
| 第3回 | 同期トラブル（モバイル）|
| 第4回 | MCP動作不良 |
| 第5回 | 他サービスとの使い分け混乱 |
| 第6回 | 自走運用への不安、効果実感不足 |
