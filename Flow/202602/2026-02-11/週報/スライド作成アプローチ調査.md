# 週報/月報スライド作成アプローチ一覧

**作成日**: 2026-02-11  
**目的**: Manusによる画像生成スライド作成（高品質だが高コスト）の代替アプローチを整理する

---

## 現状の課題
- **現行フロー**: Python分析 → 数値データ抽出 → Manusに渡してスライド化
- **Manusの利点**: グラフに注目ポイントの矢印・色分け・吹き出しが自然に入る（画像として出力）
- **Manusの欠点**: コストが非常に高い

---

#### 1. Python + Matplotlib/Plotly でグラフ画像を直接生成

| 項目 | 内容 |
|------|------|
| 概要 | Pythonでグラフを生成し、注釈（矢印・吹き出し・色分け）もコードで付与。PNG/SVGで出力 |
| ツール | Matplotlib (`annotate()`)、Plotly (`layout.annotations`)、Altair |
| コスト | **無料**（OSS） |
| 注目ポイントの表現 | `annotate()` で矢印+テキスト、色分け、ハイライト領域、カスタム吹き出し等が可能 |
| 長所 | 完全無料、再現性100%、自動化しやすい |
| 短所 | **デザインが機械的**。「どこに注目すべきか」のロジックを事前にコードで定義する必要がある。ビジュアルの調整に時間がかかる |
| 向いているケース | 毎週同じ観点で出す定型グラフ。注目ロジックが決まっている場合 |

#### 7. LLM API（Claude/Gemini） + python-pptx のハイブリッド

| 項目 | 内容 |
|------|------|
| 概要 | LLMにデータを渡して「どこに注目すべきか」「どんな注釈を入れるか」を判断させ、その指示をもとにpython-pptxで配置。判断はAI、描画はコード |
| ツール | Claude API / Gemini API + python-pptx |
| コスト | **月額 $5〜30程度**（LLM APIの利用量に依存。Gemini Flashなら月数ドル） |
| 注目ポイントの表現 | AIが「この曜日が要注意」「この数値が前月比で大幅減」等を判断し、JSON形式で注釈指示を出力 → python-pptxが矢印・吹き出し・色分けを配置 |
| 長所 | **「どこに注目すべきか」の判断をAIに任せつつ、描画コストはゼロ**。Manusの知的判断部分を低コストに再現。カスタマイズ性も高い |
| 短所 | 注釈の配置ロジック（座標計算等）の開発が必要。LLMの出力フォーマットの安定性確保が必要 |
| 向いているケース | **最もバランスが良い選択肢**。Manusの「賢さ」をAPIコストで代替し、描画はプログラムで行う |
| 参考コスト | Gemini 2.0 Flash: 入力 $0.075/M tokens → 1スライド分のデータ+指示で約1,000 tokens → 4店舗×30スライド=120,000 tokens → **月額約 $0.01〜0.10** |

## 推奨候補（コスパ重視）

### 第1候補: 7. LLM API + python-pptx ハイブリッド
- **理由**: Manusの「どこに注目すべきか」という知的判断を安価なLLM API（Gemini Flash: $0.075/M tokens）で再現し、描画はpython-pptxで行う。月額コストは数ドル〜数十ドル。PPTXなので後から手動微修正も可能
- **開発イメージ**: 分析データJSON → LLMが注釈指示JSON（「スライド3の水曜日バーを赤色に、矢印で"前月比-22%"と注記」）を出力 → python-pptxがPPTX生成

### 第2候補: 8. LLM API + HTML/SVG ハイブリッド
- **理由**: LLMにHTML/SVGコードを直接生成させる方式。デザインの自由度はpython-pptxより高い。ダッシュボード改プロジェクトとの親和性もある

### 第3候補: 11. Presenton（セルフホスト）
- **理由**: OSSでセルフホストすればLLM API費のみ。PPTX出力対応。開発コストは中程度だが、一度セットアップすれば運用は自動化可能
