# SMP Studio（Short Movie Pro） — アプリ要件整理

> **前提**: このアプリはクライアント（動画制作を依頼する企業の担当者）が使うアプリである。
> 制作会社が提供するサービスのクライアント向けポータル。

---

## 1. アプリ概要

**SMP Studio（Short Movie Pro）** — 動画制作を依頼するクライアントが、台本の作成から納品確認までを一つのアプリ上で完結できるWebアプリ。

**クライアントのユースケース**: 自社の商品・サービスのショート動画（リール/TikTok/YouTube Shorts等）を制作会社に依頼し、台本作成→制作進捗確認→納品物の確認・承認を行う。

---

## 2. 画面構成と機能要件（モックから読み取れるもの）

### 2.1 ダッシュボード

クライアントが自社の案件状況を一目で把握するホーム画面。

| 要素 | 内容 |
|---|---|
| ステータスサマリ | 全プロジェクト数、着手待ち/制作中/確認待ち/完了の件数をカード表示 |
| クレジット残高 | 動画制作に使えるポイント残高と今月の完了件数 |
| 最近のプロジェクト | プロジェクト名・ステータスのリスト（詳細への導線） |
| 新規プロジェクト作成ボタン | ヘッダーに配置。新規依頼への導線 |

### 2.2 プロジェクト一覧

自社が依頼した案件をすべて確認・検索する画面。

| 要素 | 内容 |
|---|---|
| 検索 | プロジェクト名・クライアント名でテキスト検索 |
| ステータスフィルタ | 「すべて」+ 各ステータスでフィルタリング |
| プロジェクトカード | プロジェクト名 / クライアント名 / 依頼日 / シーン数 / ステータスバッジ |

### 2.3 新規プロジェクト作成

クライアントが動画制作を新規依頼するためのウィザード形式フォーム。

**ステップ1: 基本情報入力**
| 入力項目 | 必須 | 説明 |
|---|---|---|
| プロジェクト名 | * | 動画のタイトル（例: 春のキャンペーン動画） |
| クライアント名 | | 自社名 |
| 参考動画URL | | YouTube/TikTok/Instagram Reels等のURL。作りたい動画のイメージ共有 |
| 商材情報（テキスト） | | 商品・サービスの特徴、ターゲット、訴求ポイント等 |
| PDF/資料アップロード | | 任意。既存の商品資料等を添付 |
| Slack通知設定 | | Incoming Webhook URL。進捗通知の受信設定 |

**ステップ2: シーン編集**
- AIが参考動画と商材情報からシーン構成を自動生成
- クライアントがシーンの内容・順番・秒数を編集

**ステップ3: 確認・発注**
- 最終確認して発注確定

### 2.4 プロジェクト詳細

個別案件の内容確認・進捗追跡を行う画面。

| タブ | 内容 |
|---|---|
| シーン構成 | シーン番号・秒数・演出効果の一覧 |
| シーン編集 | シーンの内容を編集 |
| 多言語指示書 | 海外編集者向けの指示書表示 |
| プレビュー | 制作物のプレビュー確認 |

**ヘッダー情報**: プロジェクト名 / クライアント名 / 依頼日 / ステータスバッジ

---

## 3. ステータスフロー

```
着手待ち → 制作中 → 確認待ち（Review） → 完了
```

| ステータス | クライアント視点の意味 |
|---|---|
| 着手待ち | 依頼済み。制作会社が着手するのを待っている |
| 制作中 | 制作会社が動画を作っている |
| 確認待ち | 動画が上がってきた。クライアントが確認・フィードバックする番 |
| 完了 | 納品完了。承認済み |

---

## 4. コア機能まとめ

| # | 機能 | 説明 |
|---|---|---|
| F-1 | ダッシュボード | ステータスサマリ、クレジット残高、最近のプロジェクト一覧 |
| F-2 | プロジェクト管理 | 一覧表示、検索、ステータスフィルタ |
| F-3 | 新規依頼（ウィザード） | 基本情報入力 → AIシーン構成生成 → シーン編集 → 確認・発注 |
| F-4 | AIシーン構成生成 | 参考動画URL + 商材情報からAIがシーン構成を自動生成 |
| F-5 | シーン編集 | シーン単位で内容・秒数・演出効果を編集 |
| F-6 | 多言語指示書 | シーン構成を多言語に変換して編集者向けに出力 |
| F-7 | プレビュー | 制作物の確認 |
| F-8 | クレジット管理 | ポイント残高の表示、発注時の消費 |
| F-9 | Slack通知連携 | Webhook URLによる進捗通知 |
| F-10 | 資料アップロード | PDF等の商品資料を添付して制作会社に共有 |

## 5. データモデル（画面から推定）

### プロジェクト
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 一意識別子 |
| name | string | プロジェクト名 |
| clientName | string | クライアント名 |
| status | enum | 着手待ち / 制作中 / 確認待ち / 完了 |
| createdAt | date | 依頼日 |
| referenceVideoUrl | string | 参考動画URL |
| productInfo | text | 商材情報 |
| attachments | file[] | PDF等の添付資料 |
| slackWebhookUrl | string | Slack通知用URL |
| scenes | Scene[] | シーン一覧 |

### シーン
| フィールド | 型 | 説明 |
|---|---|---|
| number | int | シーン番号 |
| duration | int | 秒数 |
| effect | string | 演出効果 |
| content | text | シーンの内容（台本テキスト等） |

### クレジット
| フィールド | 型 | 説明 |
|---|---|---|
| balance | int | 残高 |
| completedThisMonth | int | 今月の完了件数 |

## 6. 画面遷移

```
ダッシュボード
  ├─→ プロジェクト一覧（「すべて表示」）
  │     └─→ プロジェクト詳細（カードクリック）
  │           ├── シーン構成（タブ）
  │           ├── シーン編集（タブ）
  │           ├── 多言語指示書（タブ）
  │           └── プレビュー（タブ）
  └─→ 新規プロジェクト作成（「+ 新規プロジェクト」ボタン）
        ├── Step1: 基本情報入力
        ├── Step2: シーン編集（AI生成後）
        └── Step3: 確認・発注
```
