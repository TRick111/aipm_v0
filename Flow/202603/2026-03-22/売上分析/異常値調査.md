# 旗の台店（shiki-001）2024-06 異常値調査

## 概要

- 対象期間: 2024年6月
- 異常値: ランチデリバリー売上 **-2,468,710円（構成比 -35.1%）**
- 調査対象: デリバリーアカウント（customer_count <= 0）

## 調査結果

### 原因: POS赤伝（取消伝票）がデリバリーとして集計されている

2024-06のデリバリー判定アカウント（customer_count <= 0）102件のうち、**18件がマイナス売上**（赤伝/取消伝票）。

マイナス売上の合計: **¥-2,683,160**

### 最大の異常値: 1件の大口赤伝

| 項目 | 内容 |
|------|------|
| accountId | No:202400100002220 |
| entryAt (JST) | 2024-06-23 06:10:07 |
| totalAmount | **¥-2,500,000** |
| customerCount | -4 |
| 商品明細 | 「大将お任せコース」¥-5,000 × 500件 |

**この1件だけで異常値の約95%を占める。**

500個の「大将お任せコース」(単価¥5,000) を一括取消した赤伝。通常の取消ではなく、POS上の一括修正（テスト入力の取消、もしくは過去の累積誤差の調整）と推測される。

これらは通常の会計取消（赤伝）。実際にあった取引を取り消すためのPOS操作。customerCount が負数のため「デリバリー」(customer_count <= 0) として分類されてしまっている。

## 根本原因

### なぜ赤伝がデリバリーに分類されるか

現在の分類ロジック:
- `customer_count <= 0` → デリバリー
- `customer_count > 0` → Eat-in

赤伝は `customer_count` が負数（元の伝票の符号反転）のため、**店内飲食の取消でもデリバリーに分類される**。

### 影響

| 項目 | 値 |
|------|-----|
| 赤伝によるマイナス合計 | ¥-2,683,160 |
| うち大口赤伝(No:202400100002220) | ¥-2,500,000 |
| うち通常赤伝(17件) | ¥-183,160 |

2024-06の「ランチデリバリー」が -2,468,710円と表示された原因は、この赤伝群（特に¥-2,500,000の大口赤伝）が全てランチ時間帯のデリバリーとして集計されたため。

## 推奨対応

1. **分析時の除外**: 赤伝（totalAmount < 0 かつ customerCount < 0）は分析対象から除外するか、別カテゴリとして集計すべき
2. **大口赤伝の確認**: No:202400100002220 の ¥-2,500,000（大将お任せコース500個取消）は店舗に経緯を確認すべき
3. **分類ロジックの改善**: `customer_count < 0` を赤伝として識別し、`customer_count == 0` のみをデリバリーとする案
