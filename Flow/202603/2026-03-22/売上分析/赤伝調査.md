# 赤伝ペア調査レポート（H.元伝票番号ベース）

調査日: 2026-03-22

## 判定方法

- `H.元伝票番号` が空でない伝票を「元伝票番号あり」と判定
- そのうち金額 < 0 → 赤伝（取消伝票）
- そのうち金額 >= 0 → 再入力伝票（取消後の再登録）
- 赤伝の `H.元伝票番号` が `H.伝票番号` として存在 → ペアあり
- 存在しない → ペアなし（単独赤伝）

## 結論

全ての赤伝にペアが存在する。赤伝と元伝票は完全に相殺されるため、
**両方を含めて集計すれば正しい売上が算出される。**

## DB分析への適用方針

- `totalAmount < 0 かつ customerCount < 0` → 赤伝として扱い、|customerCount| をEat-in人数に使用
- `totalAmount >= 0 かつ customerCount == 0` → デリバリー
- `totalAmount >= 0 かつ customerCount > 0` → Eat-in
- 赤伝は元伝票と相殺されるため、全伝票を含めて集計する
