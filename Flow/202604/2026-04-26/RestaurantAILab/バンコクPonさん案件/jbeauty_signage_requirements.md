> 目的: JBeauty 店舗（バンコク／日本／その他想定）のデジタルサイネージで「来店客に合わせて商品をレコメンドする体験」を実現するために、機能・UI・データ連携・運用の各観点で要件を一覧化し、Pon さんチームと合意形成のための共通たたき台にする。

## 1. 機能要件

### 1.1 商品レコメンド
- **入力**: 顧客属性（性別・年代・肌タイプ・関心カテゴリ）、来店動機、滞在ステップ（入店直後 / カウンセリング後 / 会計前）
- **出力**:
  - メイン推し 1 商品（大画面）+ サブ推し 2〜3 商品（カルーセル）
  - 推奨理由テキスト（例: 「乾燥肌向け / 30 代女性に支持」）
  - 在庫切れ時は自動で同カテゴリの代替品に差し替え
- **アルゴリズム方針**:
  - フェーズ 1: ルールベース（属性 × カテゴリ × 在庫）でローンチ
  - フェーズ 2: 購買履歴ログを蓄積した上で、簡易協調フィルタ／LLM ベースのレコメンドへ拡張
- **NG ルール**: 同一顧客セッションで同じ商品を繰り返し出さない／在庫 0 は出さない／値引き率の表記揺れを抑制

### 1.2 顧客属性別の出し分け
- **属性軸（最小セット）**: 性別 / 年代帯（10s, 20s, 30s, 40s, 50s+）/ 肌悩み（乾燥・皮脂・敏感・エイジング・美白）/ 来店目的（自分用 / ギフト）
- **取得方法**:
  - スタッフ入力（タブレット）
  - 任意機能としてカメラベース推定（年代・性別の概算）— **プライバシー観点で Pon さん確認必須**
- **出し分けの例**:
  - 30 代女性 × 乾燥 → 高保湿ライン主訴求
  - 20 代女性 × 美白 → ブライトニング系・トライアルサイズ推し
  - 男性 → グルーミング・ギフト訴求
- **デフォルト**: 属性未取得時は「店舗おすすめ TOP5」を表示

### 1.3 多言語対応
- **必須言語**: 日本語 / 英語 / タイ語 / 中国語（簡体）
- **将来候補**: 中国語（繁体）/ 韓国語 / ベトナム語
- **切替方式**:
  - 自動: 顧客属性入力時にスタッフが言語を選択
  - 手動: 画面下に言語スイッチャー（フラグ + ISO コード）
- **コンテンツ管理**:
  - マスタは日本語、各言語は LLM 翻訳 + ネイティブ最終確認のフロー
  - タイ語は BL-0042（タイ語テキスト確認）と運用フローを共通化
- **音声**: 音声 ON/OFF、音声付き再生時は BL-0043（タイ語 TTS）と検証連携

## 2. UI 要件

### 2.1 画面サイズ・設置
- **想定**: 縦置き 43〜55 インチ（店頭・カウンター横）/ 横置き 32〜43 インチ（カウンセリング横）
- **解像度**: 1080×1920（縦）/ 1920×1080（横）を基準、4K 対応は将来
- **設置高**: 顧客の目線に近づく中央 = 床 +120cm 〜 150cm
- **照明影響**: 強い反射が想定されるため、ノングレア前提 / コントラスト比は最低 1000:1

### 2.2 レイアウト
- **基本ゾーニング（縦置き）**:
  1. ヘッダ: ブランドロゴ / 言語切替
  2. メインビジュアル: 動画 or 静止画（60〜70% 面積）
  3. レコメンド帯: 商品 1 + サブ 3
  4. フッタ: 価格 / QR（商品詳細・購入導線）/ ステッカー（NEW/SALE/限定）
- **状態表示**: スタッフ用にコーナー右上へ「接続 / バージョン / 言語 / セッション ID」を小さく常時表示

### 2.3 動画 / 静止画
- 動画: 15〜30 秒ループ（商品紹介・ハウツー）/ MP4 (H.264/AAC) / 1080p 以上
- 静止画: 商品単体カット + ライフスタイルカット 2 種を最低セット
- **モード**:
  - アイドルモード: ブランドムービー / シーズンキャンペーン
  - レコメンドモード: 顧客属性入力後、商品中心レイアウトへ遷移
  - カウンセリングモード: スタッフ操作で商品比較表示

### 2.4 アニメーション・遷移
- 画面遷移: 0.4〜0.6 秒のフェード or スライド（脳への過負荷を避ける）
- 強調アニメ: 「NEW」「SALE」バッジは 1 サイクル 2〜3 秒
- **アバター実装の検証 (BL-0071)**:
  - 候補 A: 実写人物の短尺動画ループ
  - 候補 B: AI 生成アバター（リップシンク・多言語）
  - 候補 C: 静止画 + テキストのみ（フォールバック）

## 3. データ連携要件

### 3.1 商品 DB
- **必須項目**: 商品 ID / 商品名（日英タイ中）/ カテゴリ / ブランド / 価格 / 画像 URL（複数）/ 動画 URL / 説明文 / 推奨属性タグ / 在庫フラグ
- **連携方式**:
  - フェーズ 1: CSV/Google Sheets を 1 日 1 回バッチ取り込み
  - フェーズ 2: Pon さん側基幹／POS と API 連携（REST/GraphQL）
- **マスタの正**: Pon さん側の店頭運用 DB に揃える方向で要協議
- 関連: BL-0044（実データに近い商品知識データの整備）/ BL-0072（実プロダクト連携）

### 3.2 在庫
- **更新頻度**: 5〜15 分間隔の差分同期（最小目標）/ 即時連携は将来
- **不整合時の挙動**: サイネージ側で「在庫切れ」と判定したら、その商品はレコメンド対象から除外（黒リスト方式）
- **店舗別在庫**: 店舗 ID 単位で表示制御（マルチ店舗運用前提）

### 3.3 価格更新
- **マスタは商品 DB 側**。サイネージはキャッシュ + バッチ更新
- セール価格 / 期間限定価格は「価格テーブル」とは別の「キャンペーンテーブル」で管理し、優先順位ルールを定義
- 表示通貨: THB を主、JPY/USD は併記オプション

### 3.4 顧客属性入力
- **入力 UI**:
  - スタッフ用タブレット（iPad 想定）/ 4〜6 タップで完了
  - 来店客自身の任意入力（QR 経由で Web フォーム）も将来検討
- **連携**: サイネージへ WebSocket もしくは BLE/ローカル LAN で属性ペイロードを push
- **保管**:
  - 個人特定情報は保持しない（属性のみ）
  - セッション ID は来店単位で発番、24h で破棄

## 4. 運用要件

### 4.1 コンテンツ更新フロー
1. 企画（Pon さん / Restaurant AI Lab）
2. 素材作成（動画・静止画・コピー）
3. 多言語化（日本語マスタ → 翻訳 → ネイティブ確認）
4. CMS 登録（商品 / キャンペーン / アバター映像）
5. ステージング配信 → 店舗で視認チェック
6. 本番反映（曜日・時間帯指定）
7. 振り返り（KPI: 滞留時間 / 購買率 / 言語別表示比率）

### 4.2 監視
- **常時監視**: 端末オンライン状態 / 最終ハートビート / コンテンツバージョン / 言語設定
- **アラート**: 30 分以上ハートビート途絶 → Slack/LINE 通知
- **ログ**: 表示イベント / 属性入力イベント / エラー
- **SLA 目標**: 営業時間内の稼働率 99% を目指す（ローンチ初期は 95%）

### 4.3 トラブル対応
- L1（店舗スタッフ）: 再起動 / ネット確認 / 紙のフォールバック POP
- L2（Pon さん側 IT 担当）: CMS / ネットワーク / 端末交換
- L3（Restaurant AI Lab 側）: ソフトウェア・データ連携・レコメンドロジック
- **エスカレーション**: 30 分復旧不可で L2、2 時間で L3 まで
- **手順書**: 1 ページの店舗向けクイックガイド + Pon さん側 IT 用詳細ランブックを別添（次フェーズで作成）
