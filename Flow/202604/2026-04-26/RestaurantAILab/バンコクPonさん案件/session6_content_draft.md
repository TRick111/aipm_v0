## Part 2：繰り返しタスクの設定 (25分)

### 2.1 説明スライド (10分)

#### スライド 2-1: なぜ「指示テンプレ」じゃダメなのか
> - 指示テンプレは **自分の手元のメモ** にしかならない
> - AIOSは「**ファイル＝知識**」なので、定型タスクも **AIOSの中にファイルとして** 置く
> - すると AI は **自分で仕様を読みに行く** → 起動が一言で済む

#### スライド 2-2: 定型タスクの構造
> ```
> Stock/定型作業/週次売上レポート/
>   ├── README.md       … 何のタスクか・トリガ・成果物
>   ├── 指示書.md        … AIに渡す手順 (プロンプト)
>   └── samples/        … 過去の出力例 (任意)
> ```

#### スライド 2-3: 起動コマンドはこれだけ
> ```
> 「定型作業/週次売上レポート を実行して」
> ```
> AIが README → 指示書 を読み、Flow/today/ に成果物を出す。

#### スライド 2-4: PONさんの想定タスク3種

## Part 3：iCloudカレンダー連携 (20分)

### 3.1 説明スライド (8分)

#### スライド 3-1: なぜカレンダー連携？
> - **「いつやるか」** を AI が知っていると、提案の質が変わる
> - 例: 「来週水曜の打合せ前に、要点まとめておいて」 ← AI が予定を見て、必要な準備物を提案
> - PONさんの予定本体は **iCloud カレンダー** (iPhone 標準) → ここに口を作る

#### スライド 3-2: 接続方法 3つの比較

> | 方式 | 仕組み | おすすめ | 備考 |
> |------|--------|:-------:|------|
> | **A. Apple MCP** | macOS 標準カレンダーDBを直接読む | ◎ | ローカル完結 / 認証不要 / Mac限定 |
> | **B. CalDAV CLI** | iCloud の CalDAV エンドポイントへAPP用パスワードで接続 | ○ | iCloud設定でAPP用パスワード発行が必要 |
> | **C. Google Cal 経由** | iCloud → Google Cal に同期して Google MCP で扱う | △ | 二重管理。非推奨 |

> **今日のデモは A (Apple MCP) で進める。**
> Mac前提だが PONさんは Mac なので問題なし。

#### スライド 3-3: Apple MCP の仕組み

> ```
> ┌───────────┐   Apple MCP   ┌──────────────────────┐
> │           │── 接続 ──────│ macOS 標準カレンダーDB │
> │  Cursor   │              │ (iCloud と自動同期)   │
> │  (AI)     │── 読み書き ──│                      │
> │           │              │ → iPhone にも反映     │
> └───────────┘              └──────────────────────┘
> ```
>
> - AIから読み書きすると、Mac標準カレンダーが書き換わり、**iCloud経由でiPhoneにも自動同期**
> - 認証不要 (Mac の OS が iCloud アカウントと紐づいているから)

#### スライド 3-4: 必要な権限
> macOS の **「フルディスクアクセス」** を Cursor に与える必要がある。
> システム設定 > プライバシーとセキュリティ > フルディスクアクセス > Cursor を ON。

### 3.2 デモ手順 (10分) — Mac/Cursor

#### Demo 3-A: Apple MCP を設定する

1. Cursor チャットで以下を依頼:
   ```
   iCloud のカレンダーを Cursor から扱いたいです。
   Mac 標準カレンダーを直接読む MCP はありますか？
   スター数が多くて安全なものを教えてください。
   ```

2. AI が「Apple MCP」を提案 → 設定を依頼:
   ```
   この Apple MCP を設定してください。
   今日はカレンダー機能だけ使えれば OK です。
   まずは読み取り専用で。
   ```

3. AI が `~/.cursor/mcp.json` に追記:
   ```json
   {
     "mcpServers": {
       "google-workspace": { "..." },
       "apple": {
         "command": "npx",
         "args": ["-y", "@dhravya/apple-mcp"]
       }
     }
   }
   ```
   ※ 実際のパッケージ名は当日 AI が確認 / 提案

4. macOS 設定 > プライバシー > フルディスクアクセスで Cursor を ON

5. **Cursor 完全再起動**

#### Demo 3-B: 読み取りデモ

```
来週の予定を一覧で見せて。日付・時間・タイトルだけでいいです。
```

→ AI がカレンダーを読み出し、表示。

```
4月29日（水）の予定だけ詳しく教えて。
```

→ 該当予定の詳細が出る。

#### Demo 3-C: 書き込みデモ (🟡 黄信号 — 確認してから実行)

```
来週月曜の朝9時に、「定型作業/週次売上レポート 実行」というタイトルで
30分の予定を入れてください。場所は不要、繰り返し設定で「毎週月曜」にしてください。
```

→ AI が予定追加 → Macカレンダーアプリで確認 → iPhone でも見えるか確認

> **これで Part 2 の「リマインド」が解決:**
> 毎週月曜 9:00 に iPhone がリマインド → Cursor を開いて「定型作業/週次売上レポート 実行」

### 3.3 iPhone 側はどうする (2分)

> | 操作 | iPhone での方法 |
> |------|----------------|
> | 予定を見る | 標準カレンダーアプリ (今まで通り) |
> | 予定を AI に頼んで追加 | Manus に「来週水曜18時に〇〇」と頼む (※Manus 側で iCloud コネクタ要設定) |
> | リマインド受信 | カレンダーの通知 |

> **Apple MCP は Mac 限定**。iPhone から AI 経由で書き込むなら別途 Manus か CalDAV を使う。

### 3.4 想定質問 + 回答テンプレ

| Q | A |
|---|---|
| カレンダーが複数ある (個人/仕事/家族) — AIはどれを見る？ | デフォルトでは全部見える。指示時に「仕事カレンダーだけ」と明示するか、設定で対象を絞れる |
| 予定を AI に入れさせて失敗したら？ | Mac カレンダーアプリで普通に削除できる。AIの操作も「元に戻して」で取り消せることが多い |
| プライベート予定を AI に見せたくない | 別カレンダー「Private」を作って Apple MCP の対象から外すか、その時だけ MCP を一時停止 |
| iPhone でも AI から書き込みたい | Manus + iCloud コネクタ、または CalDAV CLI を別途検討 (今日のスコープ外) |
| 認証画面が出ない / 動かない | フルディスクアクセスが OFF になっている可能性大。設定確認 → Cursor 再起動 |
