## Part 2 — 繰り返しタスクの設定 (25分)

### 2.1 なぜやるか
- 月次/週次/日次で **同じ依頼を毎回ゼロから書いていない？** を起点に説明
- ポイント: 「指示文をテンプレ化する」ではなく、「**AIOSに『定型タスク』として置く**」ことで、AIが自分で仕様を読みに来る

### 2.2 定型タスクの構造 (AIOSルール)

- 場所: `Stock/定型作業/<タスク名>/`
- 必要ファイル:
  - `README.md` — 何のタスクか・トリガ・成果物の定義
  - `指示書.md` (or `prompt.md`) — AIに渡す手順
  - `samples/` — 過去の出力例 (任意)
- AIへの起動: 「`定型作業/週次売上レポート` を実行して」だけでよい

### 2.3 PONさんの想定タスク3種

| タスク | 周期 | 成果物 | 起動方法 |
|--------|:----:|--------|----------|
| 週次売上レポート | 毎週月曜 | `Flow/.../週次売上_YYYY-MM-DD.md` | 「週次売上レポートを実行」 |
| 月次稼働サマリ (PONさん→Restaurant AI Lab 請求) | 月初 | 稼働時間集計 + 請求書下書き | 「月次稼働サマリを実行」 |
| Instagram 投稿アイデア出し | 隔週 | 投稿案3本 | 「Instagram 投稿アイデア出しを実行」 |

### 2.4 デモ手順 (PC/Mac, Cursor)

1. AIに「定型作業のフォルダを作って」と依頼 → `Stock/定型作業/週次売上レポート/` 作成
2. AIに「このタスクのREADMEと指示書を書いて」と依頼 (PONさんから具体的な仕様をヒアリング)
3. AIに「`定型作業/週次売上レポート` を試しに実行して」と依頼 → 成果物が `Flow/today/` に出る
4. 出力を確認・微修正 → 次回からは「実行して」だけでOK

### 2.5 リマインドはどこから来るか

- AIは時間で勝手に動かない (バックグラウンド常駐していない)
- **iPhone 標準リマインダー** or **iCloud カレンダー** で「毎週月曜 9:00」を仕掛けて、PONさんが Cursor を開いた瞬間に「定型作業/週次売上を実行」と打つ運用が現実的
- → **ここでPart 3 (iCloudカレンダー連携) に自然に繋がる**

## Part 3 — iCloud カレンダー連携 (20分)

### 3.1 何が解けるか
- 「来週の予定教えて」「火曜日の14時に空きある？」を Cursor / Manus 両方からAIに聞ける
- 第4回で設定した **Google Workspace MCP** の **iCloud 版**

### 3.2 PONさんの環境前提
- カレンダー本体は **iCloud** (iPhone の標準カレンダー)
- 連携相手: Mac (Cursor) と iPhone (Manus / 標準カレンダーアプリ)
- Mac の **macOS 標準カレンダーアプリ** に iCloud アカウントが追加されていれば、AIから扱う口は作れる

### 3.3 接続方法 (3方式比較)

| 方式 | 仕組み | おすすめ度 | 備考 |
|------|--------|:---------:|------|
| **A. Apple MCP** (推奨) | macOS 標準カレンダー DB を直接読む | ◎ | ローカル完結 / 認証不要 / Mac限定 |
| **B. CalDAV CLI** | iCloud の CalDAV エンドポイントへAPP用パスワードで接続 | ○ | iCloud設定でAPP用パスワード発行が必要 |
| **C. Google Cal 経由** | iCloud → Google Cal に同期して Google MCP で扱う | △ | 二重管理になりがち。非推奨 |

> 今回のデモは **A. Apple MCP** で進める。Mac前提だが PONさん環境は Mac なのでフィット。

### 3.4 デモ手順 (A. Apple MCP の場合)

1. AIに「iCloudのカレンダーをCursorから扱いたい。Mac標準カレンダーを直接読むMCPはある？」と相談
2. AIが Apple MCP を提案 → 「これを設定して」と依頼
3. AIが `.cursor/mcp.json` に Apple MCP を追記 (ファイル/カレンダー/メモが対象)
4. macOS の **「フルディスクアクセス」** を Cursor に許可 (システム設定 > プライバシーとセキュリティ)
5. Cursor 再起動 → 「来週の予定を一覧して」と聞く
6. 「来週月曜9時に『週次売上レポート実行』という予定を入れて」 → カレンダーアプリで確認

### 3.5 iPhone 側はどうする
- Apple MCP は Mac のみ。iPhone (Manus) からは **Manus コネクタ** か **Apple純正カレンダーアプリ** で直接見るのが現実的
- 「PCから AIに編集 → iPhone で確認」 が基本動線

## Part 4 — 会話履歴のインポート (25分)

### 4.1 何が解けるか
- ChatGPT で過去に話した「J-Beauty 戦略」「KINUJO 店販戦略」などをAIOSが参照できる
- すでに BL-0037 で田中チームが整理した **`pon-chatgpt-knowledge` リポジトリ** がある (PONさん招待待ち) → これをそのまま使う

### 4.2 履歴インポートの全体フロー

```
ChatGPT (web)
   ↓ ① エクスポート (公式機能)
conversations.json (大量)
   ↓ ② AI で分類・要約
Stock/<Program>/<Project>/ChatGPT履歴/  
   ↓ ③ ProjectIndex.yaml に登録
AIOSのAIが自然に参照できる状態
```

### 4.3 デモ手順

1. **エクスポート** (PONさん側で実施): ChatGPT > Settings > Data Controls > Export data
2. メールで届く zip → Mac にダウンロード → `Flow/today/PONさん/chatgpt_export/` に展開
3. AIに「この `conversations.json` を AIOS に取り込んで。プロジェクト分類は既存の `pon-chatgpt-knowledge` の分類体系に合わせて」と依頼
4. AIが分類スクリプト (BL-0037 の `step1_extract.py` 〜 `step4_synthesize.py`) を実行
5. 出力を `Stock/RestaurantAILab/バンコクPonさん案件/AIOS提供/ChatGPT移行/` 配下に配置
6. `ProjectIndex.yaml` に登録
7. テスト: 「J-Beauty の総合戦略について、過去ChatGPTで話した内容を要約して」 → AIが履歴から答えるか確認

### 4.4 既存資産の活用
- BL-0037 で作成済み: `pon-chatgpt-knowledge` private repo (PONさんを招待してもらう)
- スクリプト: `~/RestaurantAILab/Markdowns-1/Stock/バンコクPonさん案件/AIOS提供/ChatGPT分析/`
- 分類案: `確定_分類手順と分類案.md`

### 4.5 ChatGPT 以外 (Claude, Gemini) は？
- Claude: web から個別会話を Markdown で書き出す手作業 / または API export
- Gemini: 公式エクスポートあり (Google Takeout)
