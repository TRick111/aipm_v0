## 1. 概要

### 何のアプリか

田中さん 1 人で複数プロジェクトを並列運用するための、**モバイル一画面の AIPM ハブ**。

- AI が出した成果物 (markdown) を iPhone でレビュー
- AI への指示・フィードバックを音声で送信
- AI が代わりにバックログ更新・新規タスク起動を実行
- 朝に「今日のタスク案」生成、夜に「今日の振り返り」生成

**「rule-following AI」から「skill-calling AI」への転換** を Phase 4 で完了。エージェントが YAML を直接編集する代わりに、`mt` CLI で API を呼んで状態遷移する設計。

## 2. 設計哲学

| 原則 | 実装 |
|---|---|
| ファイルシステムが Source of Truth | AIPM `Stock/` と `Flow/` の YAML/Markdown |
| 中央 DB なし、ロックインなし | proper-lockfile + atomic rename、in-memory index は cache |
| Schema 厳守は API 層で強制 | zod 検証 + 400 reject (エージェントの prompt rule 遵守に頼らない) |
| Cockpit が AI 会話の Source of Truth | mini-tachyon は会話を duplicate しない |
| エージェントは skill-caller | 305 行の rule 文書 → `mt` CLI 呼ぶだけに |
| Tailscale 内に閉じる | 認証なし、安全境界は VPN |

created_by: ...

  ## タスク詳細
  ...

  - [x] ...

cockpit_task_ids:
  - "7d51e727"

  - id: q-...

## 7. HTTP API 一覧 (16 routes / Phase 4a)

**Base URL**: `http://localhost:3000`
**認証**: なし (tailscale 内 / localhost 限定)
**全書込**: zod validate + proper-lockfile + atomic rename
**エラー形式**: `{ error: <code>, message: <string>, detail?: <any> }`

### 7.1 BL 操作

| Method | Path | 用途 |
|---|---|---|
| `POST` | `/api/bl` | 新規 BL 作成 (per-project YAML、ID 自動採番可) |
| `GET` | `/api/bl?state=&project=&ids=&today_selected=` | BL リスト (filter 付) |
| `GET` | `/api/bl/:id` | BL 詳細 |
| `PATCH` | `/api/bl/:id` | 部分更新 (state/priority/title/description、optimistic lock 可) |
| `POST` | `/api/bl/:id/decisions` | decisions[] に append (`type/by/content` zod 強制) |
| `POST` | `/api/bl/:id/pending-questions` | pending_questions[] に append |
| `DELETE` | `/api/bl/:id/pending-questions/:qid` | 質問 consume |

### 7.2 Deliverable 操作

| Method | Path | 用途 |
|---|---|---|
| `GET` | `/api/deliverables/:id` | 詳細 (file / bl_id / review_log) |
| `PATCH` | `/api/deliverables/:id/state` | review_state 更新 (5値)、`comment` 指定で cockpit に [完了承認]/[要修正] prefix で送信 |
| `POST` | `/api/deliverables/:id/comments` | review_log に コメント append (cockpit 送信なし) |

### 7.3 Cockpit 連携

| Method | Path | 用途 |
|---|---|---|
| `POST` | `/api/cockpit/tasks` | タスク作成 (`related_bl_id` で `BL.cockpit_task_ids` に自動連携) |
| `POST` | `/api/cockpit/tasks/:tid/send` | 三段 fallback 送信 (send → resume_then_send → create) |

### 7.4 Daily Loop

| Method | Path | 用途 |
|---|---|---|
| `POST` | `/api/morning/start` | ☀️ 朝起動 (idempotent、`startDailyLoop` ラッパ) |
| `POST` | `/api/morning/finalize` | `[x]` 採択を atomic に処理 (BL状態更新 + クラスA spawn 一括) |
| `POST` | `/api/evening/start` | 🌙 振り返り起動 (`endDailyLoop` ラッパ) |

### 7.5 クエリ

| Method | Path | 用途 |
|---|---|---|
| `GET` | `/api/today/selected` | 今日の選択 BL リスト |
| `GET` | `/api/projects` | 全 project + active BL 数 |

### 7.6 HTTP ステータス / エラーコード

| Status | code | 意味 |
|---|---|---|
| 200 / 201 | — | 正常 (成功時 `{ ok: true, ... }`) |
| 400 | `validation` | zod schema 違反 |
| 400 | `invalid_json` | JSON parse 失敗 |
| 404 | `not_found` | BL / deliverable / question 不在 |
| 409 | `conflict` | optimistic lock conflict |
| 409 | `already_exists` | 既に同 ID の YAML 存在 |
| 500 | `io` | YAML 読書き失敗 |
| 500 | `internal` | その他例外 |
| 502 | `cockpit` | Cockpit subprocess 失敗 (3 段 fallback 全滅、`detail.attempts[]` 含む) |

## 8. `mt` CLI 一覧

**実体**: `~/.agi-tools/mini-tachyon/bin/mt` (Bun shebang script、実行権限付き)
**インストール**: `sudo ln -s ~/.agi-tools/mini-tachyon/bin/mt /usr/local/bin/mt`
**Env**: `MINI_TACHYON_URL` (default `http://localhost:3000`)
**Output**: デフォルト JSON、`--human` で整形
**Error**: stderr に JSON `{error, message, detail?}` + exit 1

### 8.1 BL コマンド

```bash
mt bl create --title <T> --project <P> --category <C> [--id <ID>] [--priority high|...] [--state pending|...] [--description <D>]
mt bl update <id> [--state ...] [--priority ...] [--title ...] [--description ...] [--expected-updated-at ...]
mt bl add-decision <id> --type answer|commitment|scope_change|deferred --by user|ai --content "..." [--answers-question-id <qid>]
mt bl add-question <id> --content "..." [--from-task <tid>]
mt bl consume-question <id> <qid> [--expected-updated-at ...]
mt bl list [--state X] [--project Y] [--ids id1,id2] [--today-selected]
mt bl get <id> [--human]
```

### 8.2 Deliverable コマンド

```bash
mt deliverable update-state <id> --state unreviewed|reviewing|done|needs_revision|generating [--comment "..."]
mt deliverable add-comment <id> --content "..." [--by name]
mt deliverable get <id> [--human]
```

### 8.3 Cockpit コマンド

```bash
mt cockpit spawn --instruction <I> [--directory <D>] [--agent-type claude|...] [--name <N>] [--bl <id>]
mt cockpit send <task_id> --text "..." [--fallback-instruction <I>] [--fallback-directory <D>]
```

### 8.4 Daily Loop コマンド

```bash
mt morning start
mt morning finalize <deliverable_id> --selections '[{"bl":"BL-0073","action":"approve","spawn_class":"A","next_action":"..."}]'
mt evening start
```

### 8.5 クエリコマンド

```bash
mt today selected [--human]
mt projects list [--human]
```

## 9. Daily Loop (朝・夜のフロー)

```
☀️ 朝 (8:30 までに開始)
  田中さん [☀️ 今日を始める] 
    ↓
  Server Action /api/morning/start
    - _orchestration/deliverables.yaml に generating placeholder upsert
    - cockpit task spawn (claude, AIPM_ROOT)
    - 固定プロンプト lib/prompts/daily-start.md (175行 / mt CLI ベース)
    ↓
  AI agent (Phase 4b prompt)
    - 中央 Backlog.md + 各 STATUS.md を読む
    - 4/24 形式で「今日のタスク案 — YYYY-MM-DD」を Project別テーブルで生成
    - mt deliverable update-state <id> --state unreviewed で完了登録
    ↓
  ホーム「📅 今日の運用」セクションに通常カード
    ↓
  田中さん iPhone でレビュー → [x] 採択 + ✍️ 回答 → done で承認 + コメント
    ↓
  /api/deliverables/:id/state (state=done, comment="...")
    → cockpit に [完了承認] prefix で送信、または create-fallback で新エージェント spawn
    ↓
  fallback agent (lib/fallback-instruction.ts)
    - mt morning finalize <deliverable_id> --selections '...'
      → 採択 BL 各々に対し cockpit spawn + state=in_progress (atomic)
    - 質問回答は mt bl add-decision
    - 新規 BL は mt bl create

🟢 日中 (既存通り)
  各 BL を AI 並列進行、田中さんレビュー、AI 質問 → 田中さん回答 (mt bl add-decision)

🌙 夜
  田中さん [🌙 今日を締める]
    ↓
  Server Action /api/evening/start
    - 固定プロンプト lib/prompts/daily-wrapup.md (mt CLI ベース)
    ↓
  AI agent
    - 当日 deliverables / decisions / 残課題集計
    - 4/24 風振り返り形式で「今日の振り返り.md」生成
    - lib/merge.ts で Flow → Stock STATUS.md マージ
    - mt deliverable update-state で完了登録
    ↓
  田中さんレビュー → 1日終了

🔄 翌朝
  朝の整理 agent が前日の振り返り読んで今日のタスク案 → ループ
```
