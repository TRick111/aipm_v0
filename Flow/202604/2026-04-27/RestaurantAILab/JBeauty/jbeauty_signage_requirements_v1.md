## 1. 目的・スコープ（v1 改訂）

### 1.1 目的
**展示会出展ブースに設置するデジタルサイネージ上で、来場者が実写のようなアバターと会話しながら、自分の悩みに沿って美容商品（スキンケア・ネイル・まつ毛など）をレコメンドされる体験**を提供する。

### 1.2 主要 KPI（候補）
- 🟡 1 体験（来場者→提案完了）の平均所要時間: 90〜180 秒
- 🟡 滞在ピーク時の同時稼働セッション数: 1 端末あたり 1（複数人視聴は OK）
- 🟢 1 日あたり対応できる来場者数: 200〜400 人/日（イベント規模次第）
- 🟢 提案商品の Web 詳細遷移率（QR 経由）: 計測可

### 1.3 スコープ外
- 決済・在庫予約: スコープ外（QR 誘導先で別途対応）
- 顔認識による個人識別: 倫理・各国規制リスクが高く v1 ではやらない（推定属性のみ）
- 化粧シミュレーター（AR メイクアップ）: 別 BL

## 2. 機能要件（v0.1 §1 改訂）

### 2.1 商品レコメンド体験
- 🔴 **入力**: 来場者の悩みヒアリング（音声 / タッチ）+ 任意属性（性別・年代・肌タイプ・関心カテゴリ・来場目的）
- 🔴 **出力**: メイン推し 1 + 関連推し 2〜3、推奨理由（自然言語）、商品詳細 QR
- 🔴 **絞り込み対話**: アバターが質問しながら 2〜4 ターンで候補を絞る（「乾燥が気になる？」「夜の保湿重視？」）
- 🟡 **比較表示**: 候補 2 商品の比較（成分・価格・推奨タイプ）
- 🟢 **セッション継続**: 「もっと見たい」と言われたら別カテゴリに横展開

### 2.2 ユーザー悩みヒアリングフロー（新規）
v1 で標準とする 4 ステップ:

1. **挨拶 / 関心ジャンル確認**（5〜10 秒）
   - 「肌・ネイル・まつ毛、どれが気になりますか？」
2. **悩み深掘り**（15〜30 秒）
   - スキンケア → 乾燥 / 皮脂 / 敏感 / エイジング / 美白
   - ネイル → 持ち / 折れ / 乾燥 / カラー
   - まつ毛 → 抜け / ボリューム / カール持ち / アレルギー
3. **絞り込み質問 1〜2 個**（10〜30 秒）
   - 例: 「日中も使う？」「香りは強い方が好き？」
4. **提案 + 理由 + 比較**（30〜60 秒）
   - メイン推し1 + 関連2、QR で詳細導線

### 2.3 顧客属性別の出し分け（v0.1 §1.2 踏襲）
- 🔴 **属性軸**: 性別 / 年代帯（10s〜50s+）/ 肌悩みタグ / 関心ジャンル（スキン・ネイル・まつ毛）/ 来場目的（自分用 / ギフト）
- 🟡 **取得方法**: スタッフ入力 or 来場者タッチ入力（カメラベース推定は v1 ではオフ）
- 🟢 **任意 v2**: カメラベース性年代推定（オプトイン UI / プライバシーポリシー必須）

### 2.4 多言語対応（v0.1 §1.3 改訂）
- 🔴 **必須言語（v1）**: **日本語 / 英語 / タイ語**（タイ展示会想定）
- 🟡 **推奨**: 中国語（簡体）
- 🟢 **将来**: 中国語繁体 / 韓国語 / ベトナム語
- 🔴 アバター TTS 音声 & 字幕 同時出力
- 🟡 言語切替: 来場者ボタン or アバターが「英語にしますか？」と提案
- ⏸ タイ語 TTS 品質は BL-0042 / BL-0043 と検証連携（**Pon さん側ネイティブ確認の運用**を要すり合わせ）

## 3. UI 要件（v0.1 §2 改訂）

### 3.1 画面サイズ・設置（展示会向けに更新）
- 🔴 **想定**: **縦置き 55 インチ程度**（来場者目線・全身〜上半身が映る寸法）
- 🟡 横置き 43〜55 インチ（カウンター型ブースの場合）
- 🔴 解像度: 1080×1920（縦）/ 1920×1080（横）以上、4K は 🟢
- 🔴 設置高: 床 +120cm 〜 150cm（顧客目線）
- 🔴 ノングレア前提（展示会照明強反射対策）
- 🟡 端末: PC（Windows/macOS）+ 大型ディスプレイ HDMI 接続。タッチパネル化は 🟡

### 3.2 レイアウト（縦置き 16:9 → 9:16）
🔴 基本ゾーニング:
1. ヘッダ: ブランドロゴ / 言語切替 / マイク状態
2. **アバター映像 50〜60% 面積**（全身 or バストアップの実写ループ動画）
3. レコメンド帯（30%）: メイン1 + サブ2〜3、QR、価格
4. フッタ（10%）: 字幕（音声同期）/ 操作ヒント

🟡 状態表示（運営向け、目立たない右上）: 接続 / バージョン / 言語 / セッション ID

### 3.3 アバター演出
- 🔴 **アイドルモード**: アバターが軽く手を動かす実写ループ（待機）
- 🔴 **対話モード**: リップシンク + 表情、AI 音声合成
- 🟡 「おすすめ商品はこちら」と画面下を指す動作
- 🟢 来場者の表情応答（笑顔→明るく、困り顔→丁寧に）

### 3.4 アニメーション・遷移
- 🔴 0.4〜0.6 秒のフェード or スライド（過剰演出回避）
- 🟡 「NEW」「SALE」「展示会限定」バッジ 2〜3 秒サイクル
- 🟢 商品比較画面のスワイプ遷移

## 4. データ連携要件（v0.1 §3 改訂、商品規模拡大に対応）

### 4.1 商品 DB（100〜200 アイテム規模）
🔴 **必須項目**:
- `product_id` / `product_name`（日英タイ）
- **`category` / `subcategory`** （3 階層: ジャンル → カテゴリ → 機能タグ）
- `brand` / `price` / `currency`
- `images[]`（最低 1 枚、推奨 3 枚: パッケージ・使用シーン・成分図）
- `description`（短文 50 字 + 詳細 200 字）
- **`concern_tags[]`**（乾燥, 皮脂, 敏感, 持ち悪い, 折れやすい... など、悩みタグ 30〜50 種を予定）
- **`recommend_attribute_tags[]`**（年代, 肌タイプ, ライフスタイル）
- `usage_text`（使い方 1〜2 文）
- `qr_url`（商品詳細 LP）
- `stock_flag`
- `is_exhibit_only`（展示会限定フラグ）

🟡 連携方式（フェーズ展開）:
- **Phase 1（展示会 v1）**: CSV / Google Sheets を起動時に取り込み（オフライン JSON にビルド）
- **Phase 2**: Pon さん側 POS / 基幹と REST API 連携（在庫リアルタイム）

🔴 マスタの正は Pon さん側の運用 DB に揃える方向で要協議（BL-0044 と紐付け）

### 4.2 在庫・展示会限定運用
- 🟡 展示会では「在庫切れで提案除外」よりも「**展示会限定セット**」の訴求を優先する想定
- 🟢 店舗常設用に切り替えるとき: 5〜15 分間隔の差分同期に切替

### 4.3 価格更新
- 🔴 マスタは商品 DB 側、サイネージはキャッシュ + バッチ更新
- 🟡 展示会限定価格 / セット価格は別テーブル（優先順位ルール）
- 🔴 表示通貨: 展示会開催地通貨を主、JPY/USD は併記オプション

### 4.4 顧客属性入力（v0.1 §3.4 踏襲）
- 🔴 タッチ入力（来場者）または スタッフ入力（タブレット）
- 🟢 任意 QR で個人スマホからフォーム入力
- 🔴 個人特定情報は**保持しない**（属性のみ、24h セッション ID）

## 6. 展示会オペレーション要件（v1 新規）

展示会という特殊環境ゆえに v0.1 で抜けていた要件を整理。

### 6.1 環境前提
| 項目 | 必須/推奨 | 内容 |
|---|---|---|
| 電源 | 🔴 | 100V/200V 規格確認、UPS 推奨（10 分維持） |
| ネット回線 | 🔴 | **有線推奨**（会場 Wi-Fi は混雑で不安定）、4G/5G ポケット Wi-Fi バックアップ |
| 帯域 | 🔴 | アバター WebRTC: **上り/下り各 2Mbps 以上**（Tavus 推奨）、4 ストリーム並列なら 10Mbps |
| 騒音 | 🔴 | 会場ノイズ 60〜80dB 想定 → **指向性マイク + ノイズキャンセリング** 必須 |
| 照明 | 🔴 | スポットライト直射対策（ノングレア + 設置角度） |
| 防犯 | 🟡 | 端末ロック・盗難対策ワイヤー・電源タップ固定 |

### 6.2 来場者導線
- 🔴 **声がけ役のスタッフ 1 名**: AI とのトーク誘導
- 🟡 アバターからの呼び込み発話（「ちょっと聞いてもいいですか？」） — 5 分以上人通りなしで自動起動
- 🟢 列が出来たときの順番待ち UI

### 6.3 デモトーク・スクリプト
- 🔴 1 体験 90〜180 秒で完結するシナリオを 3〜5 種準備
- 🟡 スタッフ向けトークスクリプト（声がけ・誘導・トラブル対応）
- 🟢 振り返り用ヒートマップ（よく出る悩みカテゴリ・時間帯）

### 6.4 法務・プライバシー
- 🔴 **音声収録の同意表示**（最低限のオプトイン UI）
- 🔴 個人特定情報を保存しない旨の表記
- 🟡 タイ PDPA / 日本個人情報保護法 / GDPR の最低ライン整理（出展国に応じて）
- ⏸ 録画・SNS 拡散の可否（Pon さん要確認 → §10）

## 7. 実写アバター技術選定（v1 新規）

### 7.1 候補比較マトリクス

「**実写のような** アバターと会話する」要件を満たすサービスを比較。**Voice Beauty Advisor の現状実装は Tavus（CVI）と Live2D の 2 タブ構成**で、Tavus が既に組み込まれているため、第一候補は Tavus とする。

| サービス | 実写品質 | 遅延（utterance→reply） | カスタムキャラ | LLM 差し替え | 主な価格 | リアルタイム対話 | 注釈 |
|---|---|---|---|---|---|---|---|
| **Tavus（CVI）** ★現状実装 | 高（実写 replica） | <500ms 〜 600ms | Stock 100+ / 自社 replica 作成可 | ◯ カスタム LLM | $59〜/月、$0.32/min（Growth）/ Enterprise 別 | ◎ | `voice-beauty-advisor/src/components/TavusAvatar.tsx` で iframe + Daily.co 統合済み |
| **HeyGen LiveAvatar** | 高（UGC 系最大手） | リアルタイム（公称値非公開） | ◯（Avatar IV $4/min） | △ 独自統合 | $99/月 + $1〜4/min | ◯ | 撮影・本人型アバター作成のリードタイム要確認 |
| **D-ID V4 Visual Agents** | 高（4K 対応） | <500ms | ◯ 動画から生成 | ◯ | API $5.9/min、Pro $48/月〜 | ◎ | 多言語 TTS と統合、コスト幅広い |
| **Hedra Live Avatars** | 高（Character-3） | **<100ms（最速）** | ◯（写真→アバター生成） | ◯ LiveKit Agents で自由 | **$0.05/min**（最安） | ◎ | コスパ・遅延ともに最強候補。実写 replica の安定性は要 PoC |
| **Anam** | 高 | <200ms | ◯ | ◯ | $12/月〜、$0.12/min | ◎ | SOC-2 / HIPAA、低コスト |
| **Synthesia Realtime（2026/01〜）** | 最高（240+ Studio Avatar） | 公称値リアルタイム | ◯（Studio Avatar $1000/年） | △ | Creator $89/月、Enterprise $20-100k/年 | ◯ | 法人最大手、コスト高、展示会単発には過剰 |
| **NVIDIA ACE（Audio2Face / Riva）** | 最高（カスタムスタジオ） | リアルタイム（GPU 依存） | ◎ 完全自社制御 | ◎ | NVIDIA AI Enterprise ライセンス | ◎ | 展示会単発には運用負荷大、ブランドアセット投資があれば検討 |

### 7.2 v1 推奨選定（AI 推奨案）

🔴 **第一候補: Tavus（CVI）** — 既に Voice Beauty Advisor で動作確認可能、Pon さん側で別契約せずにそのまま BL-0045（実写アバター動作確認）と接続できる。**60 秒以内のデモ用 replica を 1 体作成し、4/27 〜 5月初旬に POC**。

🟡 **第二候補: Hedra Live Avatars** — 0.05 USD/分の圧倒的コスパで展示会の数十時間運用が現実的。LiveKit ベースで OpenAI Realtime と組み合わせ、低遅延が必要なら切替候補。

🟢 **第三候補: D-ID V4** — 多言語 TTS と組み合わせやすい。タイ語の発話品質次第で再評価。

⏸ **選定の田中さん判断ポイント**:
- (a) 展示会 1 回あたりの想定運用時間（時間×日数）→ 価格レンジが決まる
- (b) アバターは「**Pon さん側のスタッフをモデル化**」したいか「**汎用 stock**」で良いか
- (c) ブランドコンプラ（SOC-2 / HIPAA / PDPA）の要件レベル

### 7.3 必要素材
- 🔴 アバターの **本人撮影（モデル）** または **stock 選定**
- 🔴 ブランド衣装 / 背景画像（ブースに合わせた緑系 or ブランドカラー背景）
- 🟡 商品紹介ジェスチャの素材ループ

### 8.2 タグ設計（悩み × 効能）

🔴 **悩みタグ（concern_tags）** 例:
- スキンケア: 乾燥, 皮脂, 毛穴, シミ, シワ, 敏感, ニキビ, くすみ, 紫外線, エイジング
- ネイル: 持ち悪い, 折れやすい, 二枚爪, 黄ばみ, 甘皮ケア, 形整え
- まつ毛: 抜けやすい, 短い, 細い, カール持ち悪い, ボリュームない, アレルギー

🟡 **属性タグ（attribute_tags）**: 年代, 肌タイプ（普通・乾燥・脂性・混合・敏感）, 性別, ライフスタイル（オフィス・スポーツ・夜遊び・育児中）, ギフト用途

### 8.3 索引化方式（実装方針）

| 規模 | 推奨実装 | 理由 |
|---|---|---|
| 〜50 件 | **JSON + tool-call**（現 Voice Beauty Advisor 方式） | 全件をプロンプトに同梱、追加コスト低 |
| **100〜200 件（v1）** | 🔴 **JSON カタログ + ベクトル検索 (RAG)** | 全件プロンプトはトークン無駄。悩み入力 → top-k 検索 → tool-call で確定 |
| 500+ 件（v2） | OpenAI vector store / Pinecone / Supabase pgvector | 在庫・店舗別の DB 化に発展 |

🔴 **v1 推奨**: 商品マスタを JSON 化（`products.json`）、起動時に OpenAI **Vector Store** へ upsert、質問テキストから top-5 を取得して tool-call の入力にする。

🟡 多言語対応: 商品名・description は **日本語マスタ + 各言語訳列** を持ち、検索クエリの言語に合わせて埋め込みを切替。

### 8.4 商品データ準備フロー（Pon さん側依頼）
1. 🔴 Pon さんから 100〜200 商品の **CSV / Google Sheets** を受領（最低限: 商品名・カテゴリ・価格・画像 URL・概要）
2. 🟡 RestaurantAILab 側で **悩みタグ・効能タグを付与**（AI 自動付与 → 田中さん最終確認）
3. 🟡 多言語化（日本語マスタ → 英タイ翻訳 → ネイティブ確認）
4. 🔴 v1 では「商品 30 件のサンプルセットで先に通す」フェーズ提案
