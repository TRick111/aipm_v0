# 定期実行（Cronタスク管理）

## 背景
- 1日のレポート作成など、毎日/毎週/毎月の定型処理を手作業で行うと漏れや遅延が発生しやすい
- AI（主にCursor CLI）を活用して、定期タスクを自動実行・監視できる運用基盤を整える必要がある

## 目的
- Cronを使った定期実行タスクの作成・管理を、AIOSプロジェクトとして一元管理する
- タスク定義、実行ログ、失敗時リカバリ手順を標準化し、運用コストを下げる

## ゴール（完了条件）
- Cronで動く定期実行ジョブの雛形（実行スクリプト + スケジュール + ログ）が運用可能
- 1日レポート作成などの実ジョブを再利用可能な形で追加できる
- 失敗検知と通知（例: LINE/メール/ログ監視）の基本導線が整っている

## 定期実行のルール
- まず手動実行で成功させてから、Cronなどの定期実行に登録する
- 1ジョブ1責務で小さく作る
- 実行ログを必ず残す
- 入力・出力・失敗時の対応をジョブ定義に明記する
- APIキーやトークンなどの秘密情報はコミットしない
- PCがスリープ/電源OFF中の場合、Cronは実行されない可能性がある前提で運用する

## 定期実行ジョブを作る際のルール
- `scripts/` に実行スクリプトや実行ラッパーを置く
- `ジョブ一覧/` にジョブ説明書をMarkdownで置く
- `logs/` にCronや手動実行のログを保存する
- `テンプレート/` にジョブ定義テンプレートを置き、新規ジョブ作成時はコピーして使う
- 新しいジョブを追加したら `ProjectIndex.yaml` と `log.md` を更新する

## フォルダの役割
| フォルダ | 役割 |
|---|---|
| `scripts/` | 実際に動くスクリプトや実行ラッパー |
| `ジョブ一覧/` | ジョブごとの目的・入力・出力・実行タイミングを説明するMarkdown |
| `logs/` | Cronや手動実行のログ保存先 |
| `テンプレート/` | 新しいジョブ説明書を作るための雛形 |
