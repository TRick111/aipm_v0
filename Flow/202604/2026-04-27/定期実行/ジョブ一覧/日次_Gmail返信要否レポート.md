# 日次 Gmail 返信要否レポート

## このジョブは何をする？

**毎朝、前日届いたメールを自動でチェックして「返信が必要 / 不要」に分類し、件数レポートを作成します。**

- 朝起きたら、このレポートを見るだけで「今日返信すべきメールが何通あるか」が分かる
- 自動通知メール（領収書、メルマガ等）は「返信不要」に振り分けられる
- 見逃し防止と、メール対応の優先順位付けをサポートする

## いつ動く？

- 毎日 **07:00**（朝）に自動実行
- PCが起動していればCronが自動で動かす

## 何を見る？ 何が出てくる？

| 項目 | 内容 |
|------|------|
| **見るもの** | あなたのGmail（前日受信分） |
| **出てくるもの** | Markdownレポート（当日の作業フォルダに保存） |
| **ファイル名** | `日次_Gmail返信要否レポート_YYYY-MM-DD.md` |
| **保存先** | `Flow/YYYYMM/YYYY-MM-DD/定期実行/` |

## レポートに書かれる内容

- 総メール数
- 返信必要：○件
- 返信不要：○件
- 返信必要メールの一覧（件名・差出人・判定理由）
- 返信不要メールの一覧

## 判定のしくみ（簡易ルール）

| 条件 | 判定 |
|------|------|
| 差出人が no-reply / noreply 等 | → 返信不要 |
| 件名が「領収書」「メルマガ」「認証コード」等 | → 返信不要 |
| 件名や本文に「ご確認」「返信」「至急」等 | → 返信必要 |
| 上記に該当しない | → 返信不要（念のため） |

## 動かないときの確認ポイント

1. **Gmailの認証切れ？**  
   → ターミナルで `gws auth status` を実行して確認
2. **手動で動くか試す**  
   → ターミナルで以下を実行:
   ```bash
   cd /Users/rikutanaka/aipm_v0
   bash "Stock/作業効率化/定期実行/scripts/run_gmail_daily_reply_report.sh"
   ```
3. **ログを見る**  
   → `Stock/作業効率化/定期実行/logs/cron_daily_gmail_reply_report.log`

## 注意事項

- この判定はあくまで「一次分類」です。最終判断は自分の目で確認してください
- 判定ルールを変えたい場合は、スクリプト内のキーワードリストを編集します

### 入力
- Gmail APIアクセス権（`gws` CLI認証済み）
- 対象期間: 前日 00:00〜当日 00:00
- 取得データ: メッセージID / 件名 / 差出人 / 受信日時 / スニペット

### 処理フロー
1. 前日受信メール一覧を取得
2. 各メールのヘッダー/スニペットを読み込み
3. ルールベースで返信要否を判定
4. 件数集計 + 明細をMarkdownレポート化
5. 当日のFlow作業フォルダに保存
