## エグゼクティブサマリ（一行）

> **「会話・会議・メール・ツイート・記事・PDF をエージェントに食わせると、人物／会社／案件／概念のページが Markdown で勝手に育ち、相互リンクされ、必要な時に引用付きで出てくる」** 個人用知識ベース。

加えて **21 本の cron が夜間に自律実行**され、寝てる間にブレインが賢くなる。これが README の "You wake up and the brain is smarter than when you went to bed." の正体。

## 1. データの保存先（ブレイン構造）

ブレインは Markdown ファイルの集合（Postgres にもインデックス済み）。

```
brain/
├─ people/        # alice-yamada.md, bob-smith.md ...
├─ companies/     # acme-corp.md, beta-fund.md ...
├─ deals/         # acme-series-a.md ...
├─ concepts/      # do-things-that-dont-scale.md ...
├─ meetings/      # 2026-04-28-acme-pitch.md ...
└─ originals/     # founder-mode-observation.md（ユーザ自身の発想）
```

各ページは以下を持つ構造化ドキュメント：

| 構成要素 | 中身 |
|---|---|
| **compiled truth**（State セクション） | 要約された真実 + 引用つき。ページ全体の「現時点での結論」 |
| **timeline** | 時系列の生エビデンス（append-only）。「いつ何があったか」 |
| **back-links** | 被参照リンク（誰がこのページを参照しているか） |
| **typed links** | `attended` / `works_at` / `invested_in` / `founded` / `advises` などの型付き関係 |

## 2. 代表ユースケース：記憶させる側（4 パターン）

### UC-1. 会議の議事録を投入する

**入力例:** Zoom / Google Meet の文字起こし
**呼ばれるスキル:** `meeting-ingestion`
**起こること:**
1. `meetings/2026-04-28-acme-pitch.md` を生成（要約・決定事項・アクション）
2. 出席者全員の `people/*.md` を作成 or 更新
3. 言及された会社の `companies/*.md` を作成 or 更新
4. 各人物・会社のタイムラインに「Attended Acme Pitch on 2026-04-28」を追加
5. ページ書込み時に **LLM コール 0 で** `attended` リンクが自動生成（self-wiring graph）
6. 完了後、各エンティティについて `enrich` スキルが web 検索で補強

**結果として何が手に入るか:** 会議1本投入するだけで「人物＋会社＋会議＋関係性」が一気にデータベース化される。次に同じ人物に会うとき、過去の文脈が全部 query で出てくる。

---

### UC-2. 「これメモっといて」と URL を投げる

**入力例:** ツイート URL / Substack 記事 / GitHub リポジトリ / YouTube
**呼ばれるスキル:** `idea-ingest`（記事系）または `media-ingest`（PDF・動画・スクショ）
**起こること:**
1. 内容を取得・要約・主要主張を抽出
2. 言及されたエンティティを既存ブレインと突合
3. 該当の `concepts/*.md` を新規作成 or 既存ページに timeline エントリ追加
4. `[Source: paulgraham.com, 2013-07-01]` 形式の引用が必ず付く

**結果として何が手に入るか:** 後で「あの記事誰が書いたんだっけ」「誰の主張だっけ」が引用つきで取り出せる。

---

### UC-3. ユーザの発言を黙ってキャプチャ（常時発火）

**呼ばれるスキル:** `signal-detector`（全メッセージで常時発火） + `brain-ops`
**起こること（明示コマンド不要）:**

ユーザが「今日 Bob と話したけど Acme は資金調達中らしい」と打つだけで:
- `people/bob-*.md` に「Mentioned Acme funding round on 2026-04-28」エントリ追加
- `companies/acme-*.md` に「Currently fundraising [Source: User, 2026-04-28]」を追記
- **User の直接発言は最高権威ソース**として `[Source: User, ...]` 引用付きで保存

**結果として何が手に入るか:** 普通に AI と会話してるだけで、後から検索可能な構造化データが裏で勝手に貯まる。記憶力の補完になる。

---

### UC-4. オリジナルの発想を保存

**入力例:** ユーザ自身の独自仮説・考察・気づき
**呼ばれるスキル:** `idea-ingest`（自分の発想モード） + `signal-detector`
**起こること:** `originals/` 配下に保存され、後の検索で他のページから自動的に参照される

**結果として何が手に入るか:** 自分の過去の発想と新しい状況が自動的に結びつく。「これ前に考えたな」を AI が思い出してくれる。

## 3. 代表ユースケース：記憶を取り出す側（4 パターン）

### UC-5. 「Acme のことなにか知ってる？」（事実検索）

**呼ばれるスキル:** `query`
**フェーズ:**
1. 質問を分解（キーワード／セマンティック／構造的検索）
2. 3層検索を並列実行（FTS / hybrid / 構造化クエリ）
3. 上位3〜5ページを読み込み
4. 引用付きで回答 + 「ブレインは X について情報を持っていない」とギャップ明示
5. **絶対にハルシネーションしない**（ブレイン内容のみが回答源）

**応答例:**
```
Acme は 2026-04-28 のミーティングで Bob 経由で言及。
現在シリーズA調達中 [Source: User, 2026-04-28]。
創業者は Jane Doe（people/jane-doe.md）。
ブレインには ARR や評価額の情報はありません。
```

**価値:** 引用付き・ギャップ明示・ハルシネーションなし。普通の ChatGPT との決定的な差。

---

### UC-6. 「Acme で働いてるのって誰？」（関係性検索）

**使うのはグラフ層**（typed link）
**コマンド:** `gbrain graph-query "people who works_at acme-corp"`
**起こること:** ベクトル検索では出てこない**関係性の直接走査**で確実に取れる

**応答例:**
```
3 人がブレインに登録されています:
- Jane Doe (CEO) [people/jane-doe]
- Bob Smith (CTO) [people/bob-smith]
- Carol Lee (Head of Sales) [people/carol-lee]
```

**価値:** GBrain がベクトル単独 RAG を **+31.4 pt 上回る最大の理由**がここ。「誰と誰が繋がっているか」「誰がどこに投資したか」が確実に取れる。

---

### UC-7. 「今日のブリーフィング」（朝の状況把握）

**呼ばれるスキル:** `briefing` + `daily-task-prep`
**起こること:**
1. 今日のカレンダーを読む
2. 該当人物のページを自動ロード
3. 直近のタイムラインを集計
4. 未完了タスクを review
5. ブレインを横断した「今日の空気」を要約

**応答例:**
```
🌅 2026-04-28 のブリーフィング

📅 今日の予定
- 10:00 田中さん 1on1 — 前回（4/15）の議題: BL-0053 G-Brain 統合方針
- 14:00 Acme ピッチ — Jane Doe (CEO), 創業6ヶ月, シリーズA調達中
- 16:00 Bob と週次

🔥 着手中
- BL-0053 (G-Brain 統合) — 4/28 v2 調査公開, BL-0002 検証待ち
- BL-0070 (J-Beauty Signage) — 要件定義フェーズ

💡 注目シグナル
- Jane Doe がブログ更新（昨日）— founder mode について言及
```

**価値:** 「朝起きたら今日やることが整理されてる」を AI が毎朝自動で生成。

---

### UC-8. 「Bob が紹介した投資先まとめて」（複合グラフ走査）

**コマンド例:** `gbrain query "investments referred by Bob"` または `traverse_graph`
**起こること:** `referred_by` / `invested_in` リンクをグラフ走査して回答

**価値:** 関係性 × 関係性の合成検索。手動 Notion / Obsidian では絶望的に時間がかかる種類の問い合わせが秒で返る。

## 4. 自律運用ユースケース（夜間の cron）

GBrain の最大の特徴は「**寝てる間に自分自身を賢くする**」こと。21 個の cron ジョブが標準装備：

| 時間帯 | ジョブ | 内容 |
|---|---|---|
| 深夜 | `enrich`（ローテーション） | 各人物・会社ページを web 検索で補強 |
| 深夜 | `citation-fixer` | 壊れた引用を全ページで修復 |
| 深夜 | `maintain` | グラフ再構築・孤立ページ検出・ベンチ実行 |
| 深夜 | `dream` | 既存ページから新しい関連性を発見し提案 |
| 早朝 | `briefing` | 翌朝のブリーフィング下書きを生成 |

朝起きたら「ブレインが昨夜より賢くなっている」状態が作られる。

> production 実績: Garry Tan 本人の OpenClaw / Hermes デプロイで **17,888 ページ / 4,383 人 / 723 社 / 21 cron jobs** を 12 日で構築。

## 5. AIPM × GBrain 統合時の典型フロー（イメージ）

AIPM（田中さんの mini-tachyon）のエージェントが MCP 経由で GBrain を呼ぶ場合、こうなる想定：

```
[ユーザ] 「来週の田中さんとのミーティング、何の話するんだっけ？」
   ↓
[AIPM Agent]
   ↓ MCP: gbrain.query("upcoming meeting with Tanaka")
[GBrain] meetings/ + people/tanaka-*.md + 直近 timeline を返却
   ↓
[AIPM Agent] 「BL-0053 の進捗共有がメイン議題です。前回 4/15 の議事録では
              ハイブリッド統合方針が未確定で、田中さんから "G-Brain の調査を
              先行" との依頼がありました [Source: meetings/2026-04-15-..]」
```

**価値:** エージェントは「ブレインに聞く」だけで、ハルシネーションなし・引用つき・関係性込みの回答が返る。AIPM の Stock/Flow と GBrain のブレインは**役割が補完的**：

| 領域 | AIPM | GBrain |
|---|---|---|
| **タスク・BL 管理** | ◎（mt CLI / mini-tachyon） | × |
| **議事録・人物・会社の知識** | △（Stock 散在） | ◎（自動構造化） |
| **横断検索（hybrid）** | × | ◎ |
| **関係性走査（typed graph）** | × | ◎ |
| **Daily Loop / 朝晩** | ◎ | △（briefing スキル） |
| **引用付き回答** | × | ◎ |

→ **ハイブリッド方針が筋が良い**。

## 6. AIPM への組込みの最小デモ案（第一回 or BL-0002 検証時）

1. AICore のエージェント MCP 設定に `gbrain serve` を追加
2. 過去の議事録・Slack ログ・関連ドキュメントを `gbrain import` で投入
3. **デモ A:**「BL-0070 J-Beauty Signage について何知ってる？」→ query スキルで引用付き回答
4. **デモ B:**「J-Beauty 案件に関わってる人物リスト」→ graph-query で関係走査
5. **デモ C:** 新しい議事録を投入 → ページが勝手に育ち、人物に back-link が貼られる様子をライブで見せる
6. **デモ D:** 翌朝 `briefing` スキルで「夜の間に GBrain が補強したサマリ」を体感

## 7. 何が嬉しいのか（一言でまとめ）

| 嬉しさ | 比較対象との差 |
|---|---|
| **会話するだけで知識が貯まる** | Notion / Obsidian: 手動で書く必要あり |
| **引用付き・ハルシネーションなし** | ChatGPT: 引用なし、捏造あり |
| **関係性で取り出せる** | RAG: ベクトル検索のみで関係性は取れない |
| **夜間に勝手に賢くなる** | 他の知識ベース: 手動メンテ必須 |
| **Markdown が source of truth** | DB-only な仕組み: ロックイン、移行困難 |
| **30 分でセットアップ** | 自前実装: 数ヶ月 |
