```python
import json, re
def parse_yen(s):
    if not s: return None
    s = str(s).replace(',','').replace('¥','').strip()
    if s in ('','#DIV/0!','#N/A','#REF!','#VALUE!'): return None
    try: return float(s)
    except: return None
def parse_pct(s):
    if not s: return None
    s = str(s).replace('%','').strip()
    try: return float(s)/100
    except: return None

vals = json.load(open('data/recipe_raw.json'))['values']
last_cat = None
records = []
for row in vals[1:]:
    pad = row + ['']*10
    cat = pad[1] or last_cat
    if pad[1]: last_cat = pad[1]
    name = pad[3] or ''
    if not name: continue
    records.append({
        '種別': pad[0] or '',
        'カテゴリ': cat,
        '位置付け': pad[2] or '',
        'メニュー名': name,
        '売価': parse_yen(pad[4]),
        '原価': parse_yen(pad[5]),
        '原価率': parse_pct(pad[6]),
    })
json.dump(records, open('data/recipe.json','w',encoding='utf-8'), ensure_ascii=False, indent=2)
```

## 4. 分析計算 (Phase 2)

### 4.1 入口スクリプト: `scripts/analyze2.py`

このスクリプトが **章立てに必要なすべての集計指標** を計算し `data/analytics2.json` に書き出す。実装の柱:

#### A. 月次サマリ (第0章・第1章・第3章)
- 売上 (account_total 合計)
- 客数 (customer_count 合計)
- 客単価 (売上 ÷ 客数)
- 1組人数 (客数 ÷ 会計件数)
- コース利用率 (`isCourse=true` の会計比率)
- 営業日数 (POS出現日のユニーク数)

#### B. 主因分解 (第1章 — 客数 vs 客単価)
営業日換算で寄与額を計算:
```python
visits_per_day_mar = mar['visits']/bd_mar
visits_expected = visits_per_day_mar * bd_apr
visits_diff = apr['visits'] - visits_expected
visits_contrib = visits_diff * mar['avg_per_visit']
unit_contrib = apr['visits'] * (apr['avg_per_visit'] - mar['avg_per_visit'])
driver = '客数減' if abs(visits_contrib) > abs(unit_contrib) else '客単価減'
```

#### C. 日次・曜日別・時間帯ヒートマップ (第1章・第6章)
- `entry_at[:10]` で日付別集計
- `entry_at` を JST に変換 (`+9h`) して時間帯バケット化
  - `~18時 / 18-20時 / 20-22時 / 22-24時 / 24時~`
- 曜日 × 時間帯のクロス集計

#### D. 商品ランキング (第2章) — **tablecharge 等を除外**

```python
EXCLUDE_FROM_RANKING = {'tablecharge', 'No charge', 'no charge', '丸氷', 'ドリンク'}
prod_apr_excl = [p for p in prod_apr if p['name'] not in EXCLUDE_FROM_RANKING]
```

#### E. 客単価分布 (第3章)
- 1人あたり単価のヒストグラム (1000円刻み)
- 平均/中央値/p25/p75/p90/min/max
- 中央値 < 平均 → 高単価会計が押し上げているサイン

#### F. PL集計 (第4章)
**1-3月はスプレッドシート、4月は本番DBから集計** (田中さん指示):

#### G. POS×レシピ マッピング (第4章B / 別Markdown) — **★重要**

田中さん指示: 4月リニューアルで商品リストが更新されたため、**3月/4月の出現パターン別に分類**:

```python
# 3月のPOS商品セット
mar_pos_names = {p['name'] for p in prod_mar}
apr_pos_names = {p['name'] for p in prod_apr}

# 各POS商品に appeared_in_mar / appeared_in_apr フラグを付与
for p in prod_apr:
    p['appeared_in_mar'] = p['name'] in mar_pos_names
    p['appeared_in_apr'] = True
    p['matched'] = (p['name'] in recipe_by_name)  # レシピマッチ判定
```

→ 6区分にクロス分類:

| 出現パターン \ レシピ | レシピあり (matched) | レシピなし (unmatched) |
|---|---|---|
| 両月 (continued) | 区分1a | 区分2b |
| **4月のみ ★** | 区分1b | **区分2a ★** (要確認) |
| 3月のみ | 区分1c | 区分2c |

### 6.3 グラフ実装の重要ノート

#### SVG aspect ratio 問題 (★必読)
D3 で `svg.attr('width', W).attr('height', H).attr('viewBox', `0 0 ${W} ${H}`)` した場合、CSS `svg{max-width:100%;height:auto}` で container 幅にスケールされ、**指定 height: <px> を上書きする**。

→ **W:H 比を slide-body 内幅 (~1208px) に合わせる**。例:
```javascript
// 横長棒グラフ
const W=1208, H=290, M={t:15,r:70,b:22,l:220};

// 月次PL (sales/expenses 並列棒)
const W=1208, H=230, M={t:32,r:30,b:38,l:65};
```

container div には `style="margin-bottom:12px"` を入れて下要素との余白を確保。

#### 縦に長くなりがちなページの圧縮
`.compact` CSS クラスを sec-title / dt / msg-bar / kpi-card に適用し、フォント・padding を1段階圧縮:

```css
.compact .sec-title{font-size:19px;margin:5px 0 4px}
.compact .msg-bar{padding:8px 36px;font-size:18px}
.compact .dt{font-size:15px}
.compact .kpi-value{font-size:30px}
...
```

**過去に見切れた実例**: 4 / 6 / 10 / 11 / 14 / 17ページ (BL-0082 v2-v5 で順次修正)。
