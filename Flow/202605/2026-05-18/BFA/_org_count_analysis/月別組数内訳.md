- 対象店舗: FIVE Arrows (bfa-001)

> **結論**: 3月以外は **「合計 ≤ ¥0 の会計（No charge 全額無料 ＋ 赤伝/取消）」が pipeline カウントに含まれていた** ことが「+1」の正体。
> 修正方針: **組数 / 客数 とも `合計 > 0` の会計だけをカウントする**（売上総額は従来通り全件合算でOK：赤伝は売上から正しく引かれる）。

## 1. 検出された「除外候補」会計の中身

### 1.1 2026-01 / `RB0120260116195904000`（合計¥0）

- 業務日: 2026-01-16 ／ 人数: 4 名
- 明細: メニュー「No charge」(¥0×4) + アラカルト/ご飯もの/カクテル多数（**全アイテム単価¥0**）
- → 全額コンプの招待会計。エアメイト集計外と思われる。

### 1.2 2026-02 / `000120260212200855436`（合計 ¥-10,890）

- 業務日: 2026-02-12 ／ 人数: 1 名
- 明細: tablecharge ¥-500×2, カルパッチョ ¥-1,700, 禁断パスタ ¥-1,800, Arrow Head ¥-2,000, Tommy's Verde Margarita ¥-1,800, Cafe Oaxaca ¥-1,600
- → **赤伝 / 取消会計**（同日 1 件の通常会計を打ち消すための符号反転入力）。組数として 1 件足してしまうのが「+1」の正体。

### 1.3 2026-03 — 該当なし

- 合計¥0 も赤伝も 0 件。
- それでもユーザー報告で「+1」が出ている場合は、**3月分のエアメイト実数値を共有してもらう必要あり**（別要因の可能性）。

### 1.4 2026-04 / 2件

- `RB0120260406195333000` 合計¥0（人数¥0、No charge ＋ カクテル¥0）
- `000120260429020445643` 合計¥-12,800（赤伝、tablecharge ¥-600×2 ほか）
- → 1月・2月の両パターンが両方とも出現。**実差は -2 件**になる。

## 2. 修正方針（恒久対応）

### 2.1 `monthly_report_pipeline.py` の修正

`calc_monthly_kpi()` の 組数 / 客数 計算ロジックを次のように変更:

```python
# 旧（問題）
accounts = len(hdr)
customers = int(hdr["人数"].sum())

# 新（修正後）
hdr_paid = hdr[hdr["合計"] > 0]
accounts = len(hdr_paid)
customers = int(hdr_paid["人数"].sum())
```

- **売上総額（`sales_total`）は変更しない**: 赤伝は会計上は売上のマイナスとして引かれるべきなので合算を維持。
- 客単価 = `売上総額 / 客数` で再計算するため、両方とも整合する。
- ボトル除外売上の分母（ボトル除外客数）も同じ「合計>0」フィルタを通す。
- 業務日基準（朝6時境界）は変更しない。

|---|---:|---:|---:|---:|---:|---:|---:|

（売上はボトル含む。客単価=売上÷客数）

### 2.3 ボトル除外指標への波及

- `kyakutanka`(ボトル除外) / `daily_avg_sales` / 週次/曜日/時間帯 の各セクションも `hdr_no_bottle` から `合計 > 0` を更にフィルタする必要あり（赤伝・全額無料はボトル外でも 1 件カウントされている）。
- 影響は 1〜2 件レベルなので KPI 表示への影響は小さいが、整合性を保つため同じフィルタを適用する。

## 3. 残課題

- **3月**: 合計¥0 も赤伝も無いのに「+1」と感じている要因が別にある可能性。エアメイト側で「予約あり/なし」「コース」など別カウント基準を持つ場合もあるので、3月の **エアメイト実数値** を入手してから再検証。
- **客数の扱い**: エアメイトが「No charge会計の人数」を客数に含めるかは未確認。今回は「組数と客数は同じ会計フィルタを通す」前提で除外。エアメイト挙動が違えば再調整。
