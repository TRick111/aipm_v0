#### ⚠️ April 差異の発生要因（要マスター判断）

prod DB の 4月日別を見ると：

- 2026-04-19, 2026-04-22 のデータが完全に欠落（DB側に entry_at が無い日）
- HTML はボトル購入 ¥256,500（構成比 8.5%）を含むが、「5/1朝入力分を補正」と注記あり
- prod DB はその補正分が未反映の可能性が高い

→ prod DB をそのまま正本にすると **「店主が 5/1朝に入力した 4/30 分のボトル ¥256,500 とその他 ¥? を消す」** ことになり、業務的にはむしろ後退する。

**選択肢**:
1. **prod DB を正として April を再生成**（タスク指示通り）→ 売上 ¥2,970,190 / 客数 551 に修正
2. **HTML を正として prod DB に Backfill** → 提供CSV を再 import して prod DB をHTMLに合わせる
3. **April 既存 HTML をそのままprod DBに登録**（差異は注記コメントで残す）
