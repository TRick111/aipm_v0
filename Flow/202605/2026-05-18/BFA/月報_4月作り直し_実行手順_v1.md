### 1.8 営業日基準への変換確認（6AM境界・v1.2据え置き）

```python
# datetime化 → 営業日変換
df['会計日時'] = pd.to_datetime(df['会計日'] + ' ' + df['会計時間'])

def to_business_date(dt):
    # ★6AM境界（v1.2で据え置き確定）
    if dt.hour < 6:
        return (dt - pd.Timedelta(days=1)).date()
    return dt.date()

df['business_date'] = df['会計日時'].apply(to_business_date)
df['business_month'] = df['business_date'].apply(lambda d: d.strftime('%Y-%m'))

# 4月帰属の行数を確認
print('business_month=2026-04 件数:', (df['business_month']=='2026-04').sum())
print('会計日(暦日)=2026-04の件数:', (df['会計日'].str.startswith('2026/04')).sum())
# ↑ 数字が一致しないはず（深夜帯のズレ：0〜4:59のみ前日扱い）
```

**DoD**: business_month=2026-04 の件数と、4/30深夜〜5/1早朝の会計の帰属が **エアレジ画面の営業日表記と一致**することを目視確認。

## 10. トラブルシューティング

| 症状 | 原因 | 対処 |
|---|---|---|
| business_month=2026-04 件数が会計日2026/04と大きく違う | 営業日変換が正しい証拠（4/30深夜0-4:59＝4月、5/1の5:00以降＝5月扱い） | 想定通り。エアレジ画面と件数一致を確認 |
| 過去レポート（1月／2月）と数字が微妙にズレる | 旧6AM境界 → 新5AM境界 への変更影響（4:00〜4:59 の会計が前日扱いから当日扱いに変わる） | 想定通り。過去レポートの遡及修正可否はユーザー判断 |
| 本番DB接続が確立できない | 接続情報（URL／認証）が未確定 | ユーザーへ確認（実行手順 §0 Q6） |
| gws CLI で予算が読めない | 認証切れ or スプレッドシート権限なし | `gws auth status` → 必要なら `gws auth login`。権限不足の場合はオーナーに共有依頼 |
| 客単価がエアメイトと一致しない | 分母が伝票数になっていないか、0円商品（tablecharge等）が含まれていないか | 共通ルール §2 の定義を再確認。tablechargeは含めるが0円商品は除外 |

| ボトル除外後の日次売上が低すぎる | 4/30の25万円が抜けているのが正しい | 月次の「うちボトル分」カラムで合計を確認 |
