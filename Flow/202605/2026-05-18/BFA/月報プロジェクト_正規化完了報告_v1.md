## 3. 5月以降の月報作成手順（サマリ）

5月以降は以下の手順で月報を生成可能：

```bash
# 1) エアレジから 5月の会計明細CSVをダウンロード（Shift-JIS）
#    例: ~/Downloads/会計明細_20260501-20260531 (1).csv

# 2) パイプライン実行
cd /Users/rikutanaka/aipm_v0/Stock/RestaurantAILab/月報/Scripts
python3 monthly_report_pipeline.py \
  --target-month 2026-05 \
  --store-code BFA \
  --sales-csv "/Users/rikutanaka/Downloads/会計明細_20260501-20260531 (1).csv" \
  --output-dir "/path/to/Flow/202606/YYYY-MM-DD/BFA/" \
  --compare-rawdata "/path/to/月報/1_input/BFA/rawdata.csv" \
  --daily-report-csv "/path/to/月報/1_input/BFA/DailyReport.csv"

# 3) HTML生成
python3 build_monthly_html.py \
  --analysis-json "/path/to/Flow/.../analysis.json" \
  --output "/path/to/Flow/.../202605_BFA_月次報告資料_v2draft.html" \
  --store-name "BAR FIVE Arrows"

# 4) Chromeで開いて目視確認 → PDF出力 → Dashboardへアップロード（06プロンプト参照）
```

**4ステップ・実行時間1〜2分**（CSV取得・確認時間除く）で月報が完成。

## 4. 残った確認事項・既知の制約

### 4.1 残課題

1. **対予算（P0-5）**: 「【BFA】本部サポート」PLシートに2026-04列が無く、引用元セルが特定できない。確定したら確認事項スライド以外のすべての売上額表示にも対予算行を追加可能
2. **日報の入力件数**: 4月は3件のみ。月内全体での入力リズム化が必要（P4論点）
3. **5月以降の比較データ**: `rawdata.csv` は現状2026-05-11時点のスナップショット。月次更新が必要

### 4.2 既知の制約

1. **3月／前年同月の比較**: 提供CSVがない月は `rawdata.csv`（ダッシュボード経由のエアレジ会計明細）を使用。提供CSVと数字に差が出る可能性（4月の場合は ¥123,270 の差があった）
2. **店舗追加時**: BBC／麻布しき／キーポイント等、別店舗の月報作成は `--store-name` 指定だけでは不十分（カテゴリ分類マスタ.md と P1抽出キー が店舗固有なため、共通ルール.md を店舗別に派生させる必要あり）
3. **AIによるメッセージ生成**: 現在のスライドのメッセージ・課題・施策案はBFA 4月実績ベースで `build_monthly_html.py` 内にハードコード。月次でこのテキストを更新する仕組みは未実装（将来的にはanalysis.jsonから自動生成 or 分離設定ファイル化）

### 4.3 次回改善候補

- メッセージ・課題・施策案のテキストを設定ファイル（YAML/JSON）に外出し化
- 比較データの取得を `dashboard_data_pipeline.py incremental-export` と自動連携
- BBC等の他店舗用カテゴリ分類マスタ追加
- 共通ルール.md のスクリプト側 lint（rule violation を自動検出）
