## 1. 営業日基準（最重要・6AM境界 ★v1.2で据え置き確定）

### 1.1 営業日の定義

- BFA／BBC等のバー業態は **18:00〜26:00（翌2:00）前後** が営業時間
- **JST 00:00〜05:59 の会計 → 前日の営業日**として集計する
- **JST 06:00〜23:59 の会計 → 当日の営業日**として集計する
- **★境界時刻は 6:00 AM**（既存スクリプト仕様継承 ／ v1.2で据え置き確定）

### 1.2 営業日算出の擬似コード

```python
def to_business_date(checkout_dt_jst):
    """
    会計日時（JST, datetime）から営業日（date）を算出する。

    ルール（6AM境界）:
      JST 00:00〜05:59  → 前日扱い（business_date = checkout日 - 1日）
      JST 06:00〜23:59  → 当日扱い
    """
    if checkout_dt_jst.hour < 6:
        return (checkout_dt_jst - timedelta(days=1)).date()
    else:
        return checkout_dt_jst.date()
```

### 1.3 月帰属の擬似コード

```python
def to_business_month(checkout_dt_jst):
    bd = to_business_date(checkout_dt_jst)
    return bd.strftime('%Y-%m')
```

### 1.4 ボトル購入の除外スコープ（★v1.2で確定）

| 集計対象 | ボトル購入の扱い |
|---|---|
| **月次総売上／月次の対前年・対予算比較** | **含める** |
| **客単価**（売上÷人数） | **除外** |
| **週次売上推移**（月内の週別） | **除外** |
| **日次売上推移**（営業日別） | **除外** |
| **曜日別売上分析** | **除外** |
| **時間帯別売上分析** | **除外** |
| **会計レベル分析（中央値・分布等）** | **除外** |

- 「月次総売上にだけ含めて、単位売上の傾向を見るものすべて除外」が大原則
- スクリプト側は `category1 == 'ボトル購入'` の明細行をフラグ化し、各集計関数に `exclude_bottle=True` オプションで適用
- 月次サマリーKPI では「売上（総額／ボトル除く）」の2系列を併記

### 1.5 会計日カラムの扱い

- エアレジCSVの `会計日` カラムは **暦日**ベース。**そのまま月帰属に使ってはいけない**
- 必ず `会計日 + 会計時間` を結合して上記擬似コードに通す
- BBC等の Dinii / USENレジでも同じ前処理を適用する

## 2. 用語定義（必須）

スライド・基礎資料・HTML出力のいずれにも、以下の定義で統一すること。各スライドフッターまたは付録で参照可能にする。

| 用語 | 定義 | 備考 |
|---|---|---|
| **売上** | 会計合計金額の合算（税込）。割引／割増は控除後 | ボトル購入を含むかは下記参照 |
| **会計件数（伝票数）** | 会計トランザクション数 | グルーピング会計の有無で変動 |
| **客数（人数）** | 会計1件あたりの「人数」カラムの合算 | 通し人数 |
| **客単価** | **（売上 − ボトル購入）÷ 客数（人数）** | ★伝票数ではない／★ボトル除外（v1.2） |
| **組単価** | 売上 ÷ 会計件数 | 補助指標 |
| **営業日数** | 1件以上の会計が成立した営業日のユニーク数 | 定休日・臨時休業は除外 |
| **日平均売上** | 売上 ÷ 営業日数 | 月内週比較で用いる |
| **フロア客単価** | （売上 − コース − イベント − ボトル購入）÷ 客数 | P4-2対応の補助指標 |
| **構成比** | 当該セグメントの売上 ÷ 月次総売上 | パーセント表記、小数1桁 |
| **入店時刻** | 会計の `会計時間` ではなく、伝票の入店時刻（取得不可な場合は `会計時間 − 滞在時間` で代用） | 時間帯別分析で使用 |
| **対予算（消化率）** | 月次売上 ÷ 月予算 | 引用元: 2026/3/15事業計画シート Cell `〇〇`（要特定） |

### 2.1 集計対象に含める／含めないの整理（★v1.2でボトル仕様確定）

| カテゴリ | 月次総売上 | 日次売上 | 曜日別 | 時間帯別 | 週別 | 客単価分子 | フロア客単価分子 |
|---|:-:|:-:|:-:|:-:|:-:|:-:|:-:|
| 通常会計（フード／ドリンク） | ◯ | ◯ | ◯ | ◯ | ◯ | ◯ | ◯ |
| コース&セット | ◯ | ◯ | ◯ | ◯ | ◯ | ◯ | × |
| イベント | ◯ | ◯ | ◯ | ◯ | ◯ | ◯ | × |
| **ボトル購入** | **◯** | **×** | **×** | **×** | **×** | **×** | × |

> ★ボトル購入は**月次総売上のみ計上**し、それ以外の「単位売上の傾向を見る」集計（客単価／週次／日次／曜日別／時間帯別）すべてから除外する。これにより「単発のボトル販売25万円」が曜日別・時間帯別の傾向を歪めない。
> ★客単価でもボトルを除外する（v1.2で変更）。客単価＝（売上−ボトル）÷人数 で算出する。

## 3. グラフ・数字表現の必須ルール

### 3.1 凡例（Legend）— 必須

- **すべてのグラフに凡例を必ず描画する**
- 棒の色・線の色が複数あるグラフ（系列が2つ以上）は **凡例なしでの作成を禁止**
- 円／ドーナツチャートはセグメントごとのラベル＋％の表示で凡例代替可
- D3.js では `<g class="legend">` を必ず1つ含めること

### 3.2 キャプション（一行で「このグラフが何を語っているか」）— 必須

- グラフの**下**（または右）に1行で「このグラフから何が読み取れるか」を書く
- 例：「ボトル購入を除いた日平均売上はW3に集中している」
- スライドタイトルとは別物（タイトルは指標名、キャプションは含意）

### 3.3 算出根拠の注記 — 必須

- **すべての主要数値**にフッター注記をつける。テンプレート：
  > *算式: 売上÷客数 ／ 期間: 2026-04-01〜2026-04-30（営業日基準）／ 除外: ボトル購入 ／ 出典: AirRegi 会計明細CSV*
- KPIカードはカード下に小さく、表は表下に注記行
- スライド最下部に「**注記行**」を必ず1行入れる

### 3.4 色だけで意味を運ばない

- 色違い（緑／赤、青／グレーなど）に意味を持たせる場合は、**必ず凡例または注記で意味を明示**
- 色覚多様性に配慮（青／オレンジを基本、赤／緑単独使用は避ける）

### 3.5 「達成・未達成」の使い分け

- **「達成」「未達」「達成率」は目標値（予算）に対してのみ使用**
- 前月比・前年同月比に「達成率」は使わない → 「前月比 +X%」「前年比 -X%」

### 3.6 日本語表記の原則

- カタカナ英語・英語表記は極力避ける
  - NG「サイズ」→OK「規模／人数」
  - NG「セールス」→OK「売上」
  - NG「トップ15」→OK「上位15商品」
- 固有名詞（商品名・店舗名）は英語のまま
- 一般化したもの（KPI、SNS等）は許容

### 6.2 ヘッダー行と明細行の見分け

```python
def is_header_row(row):
    # 合計または人数が空でない行をヘッダーとみなす
    return pd.notna(row['合計']) and row['合計'] != ''

def is_detail_row(row):
    return not is_header_row(row)
```

### 6.3 入店時刻の算出

```python
def to_entry_time(checkout_dt_jst, stay_duration):
    """
    会計日時 - 滞在時間 = 入店時刻（推定）
    時間帯別分析は入店時刻ベースで集計する。
    """
    return checkout_dt_jst - stay_duration
```
