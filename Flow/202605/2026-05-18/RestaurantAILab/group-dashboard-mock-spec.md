## 0. このドキュメントの位置づけ

`group-dashboard-requirements.md` (v0.1) のフィードバックを反映し、**モック制作にそのまま使える粒度** にまで具体化したUI仕様書。

### v0.1 からの変更点

| 項目 | v0.1 | v0.2（本ドキュメント） |
|---|---|---|
| 店舗比較ビュー | 画面あり | **削除** |
| 異常検知センター | 画面あり | **維持** |
| 日報横断フィード | 時系列＋アクション（タスク化／既読／コメント） | **時系列＋タブ強化、アクションは廃止**。カードは既存の店舗PL／店舗日報タブへディープリンク |
| グループ週報 | A4 1〜2枚／スライド6〜8枚 | **既存単店週報と同じ構造で約20〜30ページ**（合算ビュー＋店舗別の差し込み） |
| グループ月報 | 簡易構造 | **既存単店月報（9章18ページ）と同じ構造**を踏襲し、各章を「全店合算 → 店舗別」の二段構えで構成 |

### 既存の単店レポート構造（DBから実物を確認した結果）

- **単店 週報**: 全30ページ、6部構成（HTML; iframe表示; サンプル: `bfa-001/2026-W08`, 84KB）
- **単店 月報**: 全18ページ、9章構成（HTML; サンプル: `bfa-001/2026-M04`, 329KB）
- いずれも `<div class="slide">` 単位のページ送り。`section-tag`（第N部 / 第N章）でグルーピング
- `weekly_reports` / `monthly_reports` テーブルにメタを持ち、本体は Vercel Blob に HTML として格納

これと **同じテンプレ骨格** をグループ層にも踏襲し、章立てを変えずに「合算+店舗別」へ展開する。

## 1. ペルソナと目的（再掲・簡略）

- **グループオーナー**（10〜30店舗保有）: 全店健康診断・異常店の早期発見・経営会議資料
- **エリアマネージャー**: 担当店の日次チェック・店長対応
- **本部スタッフ**: 数値抽出・レポート配信

→ 「朝5分で異常店を見つけ、必要なら既存の店舗UIにそのまま潜って深掘りできる」体験を作る。

## 2. ルート構成（v0.2）

```
/group/[groupId]/
├── overview          グループサマリー（トップ）
├── stores            店舗一覧（ランキング）
├── anomalies         異常検知センター
├── daily-reports     日報横断フィード（タブ＋時系列）
├── pl                グループPL
├── weekly-reports/[weekId]    グループ週報（既存週報と同骨格・約20〜30P）
└── monthly-reports/[monthId]  グループ月報（既存月報と同骨格・約18〜25P）
```

**削除**: `/group/[groupId]/compare`（v0.1からドロップ）

### スコープ切替

ヘッダーに `グループ全体 ▼` のセレクタを置く。`グループ / エリア / 単店` を切り替えると、同じ画面の集計レベルが変わる。単店を選んだ場合は既存 `/store/[storeId]/...` にリダイレクト。

## 3. 画面仕様

### 3.1 `/group/[groupId]/overview` — グループサマリー

#### 目的
オーナーが朝一番に開く画面。3秒で「今日見るべき店」が分かる。

#### レイアウト

```
┌──────────────────────────────────────────────────────────────────────┐
│ ヘッダー: [ロゴ] グループ:◯◯フード  期間[昨日▼] 比較[前年同日▼]    │
│         スコープ: ● グループ全体  ○ エリア:関東  ○ 単店          │
├──────────────────────────────────────────────────────────────────────┤
│ ┌── KPIカード列（6枚） ─────────────────────────────────────────────┐│
│ │ 全店売上 / 客数 / 客単価 / 原価率 / 人件費率 / 営業利益          ││
│ │ 各カードに「前期比▲▼+x%」「予算差」を併記                       ││
│ └────────────────────────────────────────────────────────────────────┘│
│ ┌── 異常アラート（赤3/黄5/緑12） [全件→ /anomalies] ────────────────┐│
│ │ 🔴 渋谷店      売上 -28%（前年同曜日比） / 日報トラブルあり       ││
│ │ 🔴 池袋東口店  原価率 38.4%（基準+7pt）                            ││
│ │ 🔴 川崎店      人件費率 35.1%                                       ││

│ │ 🟡 ... （max 5件まで表示。全件は /anomalies）                       ││
│ └────────────────────────────────────────────────────────────────────┘│

│ └──────────────────────────┘ └──────────────────────────────────────┘│
│ ┌── 今日見るべき日報 TOP5 ───────────── [全件→ /daily-reports] ─────┐│
│ │ 5件、AIタグ付きカード（4.3節と同じ構造、リード3行＋遷移）         ││

│ [📄 PDF出力] [🖼 スライド出力] [📧 メール配信]                       │
└──────────────────────────────────────────────────────────────────────┘
```

#### UI要素

- **期間ピッカー**: 「昨日 / 今週 / 今月 / カスタム」。デフォルトは「昨日」
- **比較セレクタ**: 「前年同日 / 前週同曜日 / 予算」
- **KPIカード**: 数値 + 差分（色付き矢印） + 予算達成率（小さく）
- **異常アラート行**: クリックで `/anomalies?storeId=...` に遷移
- **ヒートマップタイル**: クリックで `/store/[storeId]/dashboard` に遷移（既存）
- **日報カード**: 3.4節と同じ構造。クリックで既存の `/store/[storeId]/dashboard?tab=daily-report&date=...` に遷移

#### モックに必要なダミーデータ

- 店舗 20件（実DB: bfa-001 / bbc-001 / shiki-001 / shiki-002 / bginza-001 などを利用、不足分は名称を補う）
- 直近30日分の売上系列（グループ計）
- 各店の予算達成率（一様乱数で60〜120%）
- 日報 直近7日×全店分（タグ付き）

└──────────────────────────────────────────────────────────────────────┘
```

#### タブ仕様

| タブ | 表示条件 | 数バッジ |
|---|---|---|
| すべて | 全件 | 直近7日件数 |
| 🔴トラブル | 「トラブル」タグ付き | 件数 |
| ⚠クレーム | 「クレーム」タグ付き | 件数 |
| 🛠設備 | 「設備」タグ付き | 件数 |
| 👥人員 | 「人員」「スタッフ」タグ付き | 件数 |
| 💡好事例 | 「好事例」「成功」タグ付き | 件数 |
| 📦在庫・仕入 | 「在庫」「仕入」タグ付き | 件数 |
| 💰会計・釣銭 | 「会計」「釣銭」「金銭」タグ付き | 件数 |
| 🗒その他 | タグなし／上記以外 | 件数 |

- タブはバッジに件数を表示（一目で量感が分かる）
- タブをタップで即フィルタ。URLにクエリ反映 (`?tag=trouble`)
- サブフィルタ（期間・店舗・キーワード）はタブと併用可能

#### カード仕様 ★

- ヘッダー: `日付時刻 / 店舗名 / 店長名 / タグ群`
- 本文: 日報本文の冒頭 3〜5 行（折り畳まずそのまま表示。長い場合のみ「もっと見る」）
- 関連指標: その日の `原価率`／`達成率` 等を 1 行ハイライト（自動異常店なら）
- フッターのリンク **（v0.2 でここを刷新）**:
  - `📊 店舗ダッシュボードへ` → `/store/[storeId]/dashboard`
  - `💴 店舗PLへ` → `/store/[storeId]/pl`
  - `🗒 日報タブへ` → `/store/[storeId]/dashboard?tab=daily-report&date=YYYY-MM-DD`
- **フィード上での「タスク化」「既読」「コメント」は撤廃**。アクションは店舗UI側で取る前提

#### モックに必要なダミーデータ

- 直近7日 × 20店舗 = 140件程度の日報レコード
- 各レコードに `tags: string[]`, `content: string`, `storeId`, `managerName`, `createdAt` を持たせる
- タグは上記 9 種類から 0〜3 個ランダム
- 文面は数パターンのテンプレ＋固有名詞ランダム

#### 日報タブへのディープリンク仕様（既存ページ側の確認事項）

`/store/[storeId]/dashboard` の既存ページに「日報タブ」が存在することを前提に、`?tab=daily-report&date=YYYY-MM-DD` で該当日の日報にジャンプできるようにする。**既存ページ側で `useSearchParams` を読んでタブ初期値を切り替える小改修が必要**（仕様確認事項として残す）。

### 3.6 `/group/[groupId]/weekly-reports/[weekId]` — グループ週報 ★大幅刷新

#### 設計方針

**既存単店週報（30P / 6部構成）と同じ章立てを踏襲する**。各セクションは「全店合算ページ → 店舗別ハイライトページ → 店舗別一覧ページ」の三段構えで構成。総ページ数は **約20〜30ページ**（店舗数で前後）。

#### 章立て（既存単店週報を踏襲）

既存 `bfa-001/2026-W08` の構成と完全に対応:

| 部 | タイトル | グループ版での扱い |
|---|---|---|
| 表紙 | 週次営業報告 | グループ名・期間・店舗数 |
| 第1部 | エグゼクティブサマリー | グループ合算 + 店舗別ハイライト |
| 第1部 | 売上トレンド（5週推移） | グループ合算チャート + 店舗別ミニチャート |
| 第1部 | 客数トレンド（5週推移） | 同上 |
| 第1部 | 客単価トレンド（5週推移） | 同上 |
| 第1部 | 前年同週比サマリー | 全店表 |
| 第2部 | 曜日別概況（要因分解） | 合算 |
| 第2部 | 不調曜日 - 基本/グループ人数/時間帯/商品/まとめ | 合算（不調日は全店共通の曜日 or 店舗別） |
| 第2部 | 好調曜日 - 基本/グループ人数/時間帯/商品/まとめ | 同上 |
| 第2部 | 曜日別評価まとめ | 全店表 |
| 第3部 | カテゴリ別売上構成比 | 合算 |
| 第3部 | 好調カテゴリの深掘り | 全店 |
| 第3部 | 商品分析まとめ | 全店 |
| 第4部 | 今週の総括：強み | 全店 |
| 第5部 | 店舗の様子（質的） | **店舗別カード（N枚）** |
| 第6部 | 施策一覧 | 全店 |
| 巻末 | 店舗別 1ページサマリー集 | **店舗数 × 1ページ** |

#### ページ構成例（20店舗の場合・約28ページ想定）

```
P1   表紙

## 7. ダミーデータ生成ルール（モック用）

- 店舗: 既存DBの 5店舗（bfa-001 / bbc-001 / shiki-001 / shiki-002 / bginza-001）に、ダミー15店を加えた20店構成
- 期間: 直近30日 + 過去5週
- 売上: 店舗ごとにベースライン（¥1.5M〜¥3.5M/週）に正弦波＋ノイズ
- 達成率: 60〜120% の一様乱数
- 原価率/人件費率: 業態基準 ± 7pt に揺らぎ
- 日報: 直近7日×20店 = 140件。タグは 9 種から 0〜3 個、本文はテンプレ 30 パターン
- 異常検知: 3.3節のルールで導出

## 8. 既存機能との関係（破壊しない原則）

- 個店UIは現状維持。グループUIは親として上から被せる
- グループUIの店舗カード／店舗名はすべて既存 `/store/[storeId]/...` にディープリンク
- 日報フィードは「読むだけ」。書き込み・タスク化はしない（既存日報フォームの責務を侵さない）
- グループ週報・月報の HTML テンプレは既存単店レポートを継承し、テーマカラーだけ変える
