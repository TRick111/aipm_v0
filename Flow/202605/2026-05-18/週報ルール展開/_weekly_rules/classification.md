**確定日**: 2026-05-19（マスター判断 K-1〜K-8 反映）  
**作成者**: Master Agent（claude）  
**ステータス**: **確定**
**入力**: `Stock/RestaurantAILab/月報/共通ルール.md` v2.3（全20セクション）  
**目的**: 月報の確立ルールを週報に展開するにあたり、各セクションを次の3カテゴリに分類する。

- **C. 月報固有**: 週報には適用しない（PL分析、27枚構成、前年同月比等）

## セクション別 分類詳細

### §1 データソース → **A（一部C）**

- **A適用**: 「出典表記の統一」「対象月CSV を正、過去月は rawdata.csv からの集約」「日報＝ダッシュボードAPI」「カテゴリ分類マスタを別ファイルで参照」
- **C（週報には不要）**:「予算（対予算）」行（週次PLを当面表示しないため）。週報側で扱わない
- 週報文脈に翻訳: 「対象週の売上＝ダッシュボード経由の rawdata.csv（PL日報は DailyReport.csv）」へ置換

### §2 集計軸の使い分け（業務日基準） → **A**

- 朝6時境界、`to_business_date()`、暦日基準の使用禁止 などは全店舗共通の基本仕様
- 週報側もすでに analysis_script.py で `business_date` を採用済（互換性確認済）
- そのまま週次文脈で適用（「月帰属」は「週帰属（ISO週）」に翻訳）

### §3 ボトル購入の取扱い → **B**

- 「POSカテゴリ "ボトル購入" 全件で抽出」「客単価分子から除外」「会計レベル分析から除外」など、BFA の業態（バー）に固有
- BBC / 麻布しき / キーポイントには「ボトル購入」カテゴリは存在しない or 意味が異なる
- → `BFA固有ルール.md` に集約。週報側は「BFA の場合のみ§3を適用」と注記

### §4 客単価2系統 → **B**

- 「ボトル除く（経営判断用）」「全部込み（エアメイト比較用）」の併記は、ボトルカテゴリがある BFA のみ
- 他店舗は単一の客単価で OK
- → `BFA固有ルール.md` に集約。週報側はBFAのみ2系統表示

### §5 P1抽出キー定義 → **B**

- メモ欄「海外」、メニュー名 `CLP` / `その他CLPコース`、カテゴリ「コース&セット」「イベント」「ボトル購入」など、ほぼ全項目がBFA固有
- 唯一P1-7/P1-8（フード／ドリンクランキング）は他店舗にも応用可能だが、「カテゴリ分類マスタ.md」自体がBFAのカテゴリ実態に紐づく
- → `BFA固有ルール.md` に「抽出キー全項目」を切り出し。月報・週報の双方から参照

### §6 3点セット原則（入客数 / 売上 / 構成比） → **A**

- 全切り口分析に「入客数 / 売上 / 構成比」を必ず付ける、という分析方針は全店舗に有効
- 週報の曜日別・時間帯別・カテゴリ別すべてに適用可能
- 週次文脈: 「該当売上 ÷ 週次総売上」に置換

### §7 ランキング（TOP15）→ **A**

- フード／ドリンク TOP15、列構成（出数 / 入客数 / 売上 / 平均販売価格 / 構成比）、★マーク（平均販売価格±5%乖離）
- 全店舗に応用可能。週次データでも TOP15 が読み取りやすい
- BFA固有部分は「カテゴリ分類マスタの個別カテゴリ名」のみ → §5 と一緒にBFA固有ファイルへ。ランキングの「型」自体は§7としてA

### §8 グラフ・数字表現の必須ルール → **A**（最重要）

- §8.1 凡例必須／§8.2 キャプション／§8.3 メッセージ帯（オレンジ縦線×薄黄）／§8.4 分類条件帯（緑薄色帯・箇条書き）／§8.5 算出根拠フッター／§8.6 集計範囲注釈
- すべて店舗を問わず適用したい UI ルール
- 週次文脈: 「集計対象: YYYY年MM月の業務日」 → 「集計対象: YYYY-Www（業務日基準 開始日〜終了日）」

### §9 色だけで意味を運ばない → **A**

### §10 達成・未達成 の使い分け → **A**

- 「前月比 / 前年同月比に達成率は使わない」→「前週比 / 4週平均比 / 前年同週比 に達成率は使わない」と翻訳して週報共通ルールへ
- 既存の週報03/05プロンプトに同等内容が散在しているので、共通ルールに集約

### §11 日本語表記の原則 → **A**

- 既存の週報03/05に同等記述あり。共通ルールに集約してDRY化

### §12 デザイン規約（カラーパレット） → **A**（重要・差分大）

- **現状の週報**: ダークネイビー `#1E3A5F` 主基調＋アクセント緑/オレンジ/赤/青
- **月報の新基調**: ダークグリーン `#1B4D3E` ＋ゴールド `#C9A961` ＋暗緑/暗赤茶
- 月報側は「派手色（青 #3B82F6・橙 #F97316・鮮緑 #22C55E・鮮赤 #EF4444）廃止」を明文化済
- → 週報も同じダークグリーン基調に揃える方針が望ましい（ブランド一貫性）
- ただし「現行週報HTMLの大幅な見た目変更が良いか」は **マスター判断**（→ questions_to_user.md）

### §13 スライド構成（27枚 / PL分析を含む） → **C**

- 週報は別の枚数（現行 28枚＋日報3-5枚）。月報固有の構成のため適用不可
- 週報側のスライド構成は 03_スライド構成案作成プロンプト で別途定義
- 週次でも「KPI / トレンド / 曜日別 / 時間帯別 / カテゴリ / 商品 / 強み課題 / 施策」の枠は保持

### §14 絵文字使用ガイドライン → **A**

- 「1スライドに3個まで」「メッセージ帯冒頭✅⚠️💡」「KPIラベル先頭の単発絵文字」「装飾目的禁止」
- 全店舗・週月共通で有効

### §15 比率ページの円グラフ必須化 → **C**（A も含む）

- 「客層分析①②③・商品構造①②③」というスライド名はBFA月報固有 → C
- ただし「比率ページにはドーナツチャート必須・サイズ180-200px・中央%表示・色＝意味の凡例」というデザイン原則は A としても流用可能
- → 週報共通ルールには「比率を扱うスライドはドーナツチャートを併設すること（180〜200px・中央%・凡例必須）」を一般原則として継承

### §16 集計対象スライドの注釈（再掲）→ **A**

- 月帰属の注釈は§8.6で扱い済。週次は別文言に翻訳：

  ```
  集計対象: YYYY-Www の業務日（YYYY/MM/DD 6:00 〜 翌週月曜 5:59 JST、朝6時境界）
  ```

### §17 CSVカラム参照（エアレジ会計明細） → **B**（条件付き）

- AirRegi 仕様の特定カラム名（取引No / 会計時間 / 滞在時間 / カテゴリー名 / メニュー名 / メモ）
- BFA・キーポイントは AirRegi、BBC は Dinii、麻布しきは USEN
- → 「AirRegi系店舗（BFA / キーポイント）」用のCSV仕様として `BFA固有ルール.md` に格納（または将来 `共通/POS_CSV仕様.md` 等に分離）
- Dinii / USEN の rawdata 仕様は別途存在（週報の analysis_script.py がそれを吸収）

### §18 PL分析の取り扱い（v2.1 新設） → **C**

- 「KPI下段にPL系」「PLサマリー / 費用推移 / PLからのインサイト」スライド3枚等は月次運用
- 週次でPLを表示するか？は **マスター判断**（→ questions_to_user.md）
- 当面は週報に含めない（C扱い）

### §19 エアメイトとの組数差異について（v2.2 新設） → **B**

- 「取引No `0001` プレフィックスの会計の扱い」「月報の正本は prod DB」など、ほぼBFA固有
- ただし「月報＝週報の正本ソース」「rawdata.csv は前月以前比較用」「月次は prod DB から直接抽出」の方針は週報側にも適用可能
- → BFA固有部分（0001系の話・実例表）は `BFA固有ルール.md` に格納。データソース原則は週報共通ルールに記述（A 寄り）

### §20 インバウンド比率（P1-1）の取り扱い（v2.3 新設） → **B**

- メモ欄「海外」運用が 2026年4月開始 → BFA固有運用ルール
- → `BFA固有ルール.md` に格納。週報側もBFAの場合のみ適用
