## 2. 合計>0 フィルタ（赤伝・無効会計の除外） — **要追加検討**

### 月報側の仕様
- `monthly_report_pipeline.py` は「業務日基準＋合計>0 フィルタ」で集計（BFA固有ルール §B-6 参照）
- エアメイト差異の主因 `0001` プレフィックスや赤伝1件等を、合計>0 で扱う

### 週報側の現状
- `analysis_script.py` には `account_total > 0` または `subtotal > 0` の明示的フィルタは見つからない
- ただし `sum()` 系の集計なので赤伝（マイナス額）が混入すると総売上が小さく出る可能性

### 提案

- (b) `account_total` カラムが負値の場合の扱いをログに残す
- いずれも **マスター判断待ち**。週報の組数定義と月報の組数定義を一致させる場合は (a) を推奨

### 影響箇所（提案差分）

## 3. BFA固有: ボトル購入除外（曜日別／時間帯別／会計レベル） — **要新規実装**

### 月報側の仕様（BFA固有ルール §B-1）
- 月次総売上は **ボトル含む**
- 曜日別・時間帯別・客単価分子・会計レベル分析は **ボトル除外**

### 週報側の現状
- `analysis_script.py` の曜日別・時間帯別集計はカテゴリで除外していない（混在）
- BFA 店舗の場合、ボトル購入の単発計上が「金曜売上の異常値」として現れ得る

### 提案
- 週報も BFA 限定で同様のロジックを実装する
- 実装方針:
  - `--exclude-bottle-categories` フラグ（`['ボトル購入']` をデフォルト）を `analysis_script.py` に追加
  - 曜日別・時間帯別・会計分布の集計時のみ除外
  - 週次総売上は除外しない（月報と整合）
  - 客単価は 2系統（除く／込み）算出（BFA固有ルール §B-2）

### 影響箇所（提案差分）
```python
# 例: 曜日別集計の前にボトル除外サブセットを作る
EXCLUDE_BOTTLE_CATS = {'ボトル購入'}
sales_df_nobottle = sales_df[~sales_df['category1'].isin(EXCLUDE_BOTTLE_CATS)].copy()

# 曜日別・時間帯別・会計レベルの集計は sales_df_nobottle を使う
# 週次総売上は sales_df を使う
```

> **マスター判断待ち**: 既存BFA週報スライドとの数値差が発生する可能性あり。週次総売上は変わらないが、曜日別・時間帯別の各値が変動する。比較しやすい移行案も同時に提示すること。
