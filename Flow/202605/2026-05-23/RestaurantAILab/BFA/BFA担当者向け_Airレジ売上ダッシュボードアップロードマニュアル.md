# BAR FIVE Arrows 様向け｜売上データ アップロードマニュアル

Airレジからダウンロードした売上 CSV を、Restaurant AI Lab のダッシュボードにアップロードして、月別の売上分析グラフに反映するまでの手順です。

- 所要時間: 約 5〜10 分（CSV ダウンロード含む）
- 必要なもの: AirID（メールアドレス）と Airレジパスワード、ダッシュボードの店舗コードとパスワード
- 推奨環境: PC（Google Chrome / Edge / Safari の最新版）

---

## 全体フロー

```
[1] Airレジで売上 CSV をダウンロード
        ↓
[2] ダッシュボードにログイン
        ↓
[3] アップロード画面で CSV を選択し「Airレジ形式」でアップロード
        ↓
[4] 売上ダッシュボードで反映を確認
```

---

## ステップ 1｜Airレジから売上 CSV をダウンロード

> 弊社内部でも同じ手順で毎週ダウンロードしています（参照: `週報データ準備スキル`）。

### 1-1. Airレジにログイン

1. ブラウザで **<https://airregi.jp/>** を開く（または「Airレジ ログイン」で検索）
2. ログイン画面で **AirID／メールアドレス** と **パスワード** を入力 → **［ログイン］** をクリック

![Airレジ ログイン画面](./screenshots/airregi-01-login-annot.png)

3. 店舗選択画面が表示されたら **「BAR FIVE Arrows」** をクリック

![店舗選択画面](./screenshots/airregi-02-store-select-annot.png)

ログインに成功すると、Airレジのバックオフィストップ画面が表示されます。

![バックオフィストップ](./screenshots/airregi-03-top-annot.png)

### 1-2. 日別売上ページに移動

1. 左メニューから **［売上・分析］** をクリックして展開
2. **［日別売上］** をクリック

### 1-3. 集計対象の期間を選択

1. 画面上部の **集計対象** プルダウンで対象期間（例: `2026年5月`）を選択
2. **［表示する］** ボタンをクリック

![日別売上ページ - 期間選択とCSVダウンロードボタン](./screenshots/airregi-04-daily-sales-annot.png)

> ⚠️ **会計明細 CSV は「前月1日から当日分まで」しか出力できません。** 月初に前月分をダウンロードする運用がおすすめです（例: 6月1日に 5/1〜5/31 をダウンロード）。

### 1-4. 会計明細 CSV をダウンロード

1. 画面中央の **［CSVデータをダウンロードする］** ボタンをクリック
2. 表示されたメニューから **「会計明細」** をクリック

![CSVダウンロードメニュー](./screenshots/airregi-05-csv-download-menu-annot.png)

3. ダウンロードが始まります（標準では `~/Downloads/` フォルダに保存）

> 「売上集計」と「会計明細」の 2 つが出ますが、必ず **「会計明細」** を選んでください。ダッシュボードに取り込めるのは会計明細 CSV です。

ダウンロードしたファイルは、わかりやすい名前にリネームしておくと管理しやすいです。

| 例 | 形式 |
|---|---|
| `20260501-20260531-BAR FIVE Arrows.csv` | `{開始日}-{終了日}-{店名}.csv` |

> **複数月分まとめてアップロード可能です。** ダッシュボード側で自動的に月別に振り分けて保存されます。Airレジ側の「前月1日まで」の制限内であれば、2 ヶ月分を一度に取得してまとめてアップロードしても問題ありません。

### 1-5. Airレジからログアウト（任意）

画面右上のアカウント名 → **［ログアウト］** をクリック。

## ステップ 2｜ダッシュボードにログイン

### 2-1. ダッシュボードを開く

ブラウザで以下の URL にアクセスしてください。

```
https://<ダッシュボードのURL>/login
```

> ※ 本番 URL は別途共有します（ブックマーク推奨）。
> 検証用の見た目を以下に示します。

![ログイン画面](./screenshots/01-login-annot.png)

### 2-2. 店舗コードとパスワードを入力

| 項目        | 入力値          |
| --------- | ------------ |
| **店舗コード** | bfa-001      |
| **パスワード** | 別途お渡ししたパスワード |

入力後 **［ログイン］** をクリック。

> パスワードが分からない場合は Restaurant AI Lab 担当（吉田または田中）までご連絡ください。

### 2-3. アプリ一覧が表示されることを確認

ログインに成功すると、店舗のアプリ一覧画面が表示されます。

![アプリ一覧](./screenshots/02-store-apps.png)

このうち今回使うのは、以降のステップで開く **データアップロード画面** です（メニューには出ないので URL から直接開きます）。

---

## ステップ 3｜CSV をアップロード

### 3-1. アップロード画面を開く

ブラウザのアドレスバーに以下を直接入力するか、ブックマークから開いてください。

```
https://<ダッシュボードのURL>/store/bfa-001/upload
```

> URL の `bfa-001` の部分が BAR FIVE Arrows 用の店舗コードです。**店舗ごとに固有なので、他店舗の URL を使わないようにご注意ください。**

開くと、以下のようなアップロード画面（ステップ 1: ファイル選択）が表示されます。

![アップロード画面 - 初期状態](./screenshots/03-upload-step1-empty-annot.png)

### 3-2. CSV ファイルを選択

1. 画面中央の **「クリックしてファイルを選択 またはドラッグ＆ドロップ」** エリアをクリック
2. ステップ 1 でダウンロードした Airレジの CSV ファイル（例: `20260301-20260331-BAR FIVE Arrows.csv`）を選択
3. ファイル名・サイズが青文字で表示されたら OK

### 3-3. CSV フォーマットを選ぶ

ファイルを選ぶと、フォーマット選択ラジオボタンが表示されます。

- ✅ **「Airレジ形式」を選択してください**
- Airレジからダウンロードした会計明細 CSV は、列名や並びが Airレジ独自の形式です
- 「Airレジ形式」を選ぶと、列マッピング（売上日・合計金額・メニュー名…etc）が自動適用されます

![Airレジ形式を選択](./screenshots/04-upload-step1-filled-annot.png)

> 標準形式や他レジ形式（USEN／スマレジ／Dinii）を選ぶとうまく取り込めません。必ず **Airレジ形式** を選んでください。

### 3-4. プレビューで内容を確認

**［次へ（プレビュー）］** をクリックすると、CSV の中身が表として表示されます。

![プレビュー画面](./screenshots/05-upload-step2-preview-annot.png)

ここで確認するポイント:

- **検出されたヘッダー** に `取引No` `会計日` `合計` `人数` `カテゴリー名` `メニュー名` などが入っていること
- **サンプルデータ** の `会計日` `合計` `メニュー名` などが正しく読めること

問題なければ **［次へ（列マッピング）］** をクリック。

### 3-5. 列マッピング（自動設定の確認）

「列マッピング」画面に切り替わります。左側に「ダッシュボードの標準列」（売上日・合計金額・メニュー名 等）、右側に「CSV の列」が並ぶ画面です。

Airレジ形式を選んでいれば、必要な列はすべて自動で割り当てられた状態になっています。左側の標準列に緑色の `✓ {列名}` バッジが付いていれば OK です。

確認したら、画面下部の **［アップロード実行］** 青いボタンをクリックしてください。

> 万が一、左側の標準列にオレンジの「⚠️ {列名}」バッジが出ているものがあれば、その列名が CSV 内に見つからなかったことを示します。その場合は Restaurant AI Lab 担当（田中）までスクリーンショット付きでご連絡ください。

### 3-6. アップロード完了

数秒〜数十秒のアップロード処理後、画面上部に以下のようなメッセージが表示されます。

```
✓ Xヶ月分のデータをアップロードしました！ ダッシュボードにリダイレクトしています...
```

その後、自動的に **売上ダッシュボード** にリダイレクトされ、最新月の集計画面が表示されます。

---

## ステップ 4｜売上に反映されているか確認

### 4-1. 売上ダッシュボードを開く

リダイレクトされなかった場合は、店舗トップから **「売上ダッシュボード」** カードをクリックして開きます。

```
https://<ダッシュボードのURL>/store/bfa-001/sales-dashboard
```

### 4-2. 対象月を選んでグラフを確認

- 月セレクタで、アップロードした月（例: `2026年3月`）を選択
- 売上推移・客数・客単価・カテゴリ別・メニュー別ランキングなど、各グラフに今回アップロードしたデータが含まれていることを確認

> 同じ月のデータを再アップロードした場合、**既存データは上書きされます**（重複しません）。誤って同じファイルを 2 回アップロードしても問題ありません。

## トラブルシューティング

| 症状 | 想定される原因 | 対処 |
|---|---|---|
| ログインで「店舗コードまたはパスワードが正しくありません」と出る | 店舗コード／パスワードの入力ミス | 店舗コードは `bfa-001` です。パスワードは半角でコピペ推奨 |
| ファイルを選んでも何も起こらない | CSV 以外のファイルを選択 | 拡張子が `.csv` のファイルを選択してください |
| プレビュー画面で「ヘッダーが検出できません」と出る | Excel で開いて保存した CSV の文字コード破損 | Airレジからダウンロードした **元の CSV** をそのままアップロードしてください（Excel で開かない） |
| アップロード後に「0ヶ月分」と表示される | 対象期間に有効な明細が無い／日付列がうまく読めていない | ファイル名と中身（会計日が入っているか）をご確認のうえ、担当までご連絡ください |
| グラフに新しい月が表示されない | 月セレクタで対象月が選ばれていない | 月セレクタを切り替えてください |
| 「Login failed」「サーバーエラー」が出る | サーバー側の一時的な問題 | 1〜2 分待って再試行。改善しない場合は担当までご連絡ください |

## 付録 A｜ダッシュボード側のフォーマット仕様（参考）

Airレジ形式を選ぶと、以下のマッピングが自動で適用されます（BAR FIVE Arrows 様で操作の必要はありません）。

| ダッシュボードの標準列        | Airレジ CSV の列名  |
| ------------------ | -------------- |
| 取引 ID（accountId）   | `取引No`         |
| 売上日時（exitDateTime） | `会計日` + `会計時間` |
| 曜日（dayOfWeek）      | 売上日から自動計算      |
| 合計金額（totalAmount）  | `合計`           |
| 来客数（customerCount） | `人数`           |
| 商品点数（itemCount）    | `商品点数`         |
| カテゴリ（category1）    | `カテゴリー名`       |
| メニュー名（menuName）    | `メニュー名`        |
| 単価（price）          | `価格`           |
| 数量（quantity）       | `注文数量`         |
