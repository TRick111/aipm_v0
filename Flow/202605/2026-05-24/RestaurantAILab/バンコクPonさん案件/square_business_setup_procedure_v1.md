# Square 事業者開設 — PONさん月次請求 完全手順 v1（Wise からの切替版）

- 作成日: 2026-05-24
- 関連 BL: BL-0068 PONさんへの請求方法の検討（Wise→Square切替）
- 切替前提: 旧 Wise ルート（Pon さん個人 Wise → 田中さん個人 Wise → みずほ）はアーカイブ。Square 一本に統一
- ゴール: 田中さん側で Square Japan に事業者アカウントを開設し、Square Invoices / 支払いリンクで 250,000 JPY を発行 → Pon さんへ LINE 送付 → Pon さんがタイのクレジットカードで決済 → 田中さんみずほ口座へ自動入金、までを再現可能にする

## 1. 事前準備（着手前にこれを揃える）

- マイナンバーカード（推奨。表裏両面の写真）
- 運転免許証（表裏両面）
- パスポート（顔写真ページ + 現住所が記載された補助書類）

→ アップロード用に **スマホで撮影 → iPhone の写真ライブラリに置いておく**。

### 1.2 事業者情報

| 項目 | 田中さんの値（暫定） |
|---|---|
| 事業形態 | 個人事業主（法人化していなければ） |
| 屋号 / 事業名 | 未確定 → 開業届の屋号 or 「田中利空」個人名で開設 |
| 事業内容（カテゴリ選択） | 「コンサルティング / プロフェッショナルサービス」が無難 |
| 業種詳細 | 「AI・IT コンサルティング / システム導入支援」 |
| 事業所住所 | 田中さんの自宅住所（個人事業主の場合） |
| 連絡先 | 田中さんの個人携帯 + setup@restaurantailab.core-driven.com |
| 想定月商 | 25 万 〜 100 万円（Pon さん案件のみ）→ 「10万〜50万円」レンジを選択 |

> **注意**: 屋号は後から変更しづらいので最初に確定させる。RestaurantAILab 関連であれば「RestaurantAILab」or 「合同会社 RestaurantAILab（仮）」のいずれか。法人化前なら個人名 or 屋号「RestaurantAILab（田中利空）」推奨。

### 1.3 入金先銀行口座（みずほメイン）

- 銀行名: みずほ銀行
- 支店名 + 支店コード
- 口座種別: 普通
- 口座番号: 7 桁
- 口座名義: 田中利空（カナ: タナカ リク）

→ Square 側で **口座名義の文字数・カナ・全角半角ミスは却下原因 No.1**。
事前にみずほ通帳/キャッシュカード/オンラインバンキングのスクショで正確な表記をコピペできる状態にしておく。

### 1.4 開業届（強く推奨だが必須ではない）

- 個人事業主として開業届を出していなくても Square は開設可能だが、屋号利用時は届出があると審査が通りやすい
- 未提出なら freee 開業 / マネーフォワード開業で 5 分で作成 → 税務署提出（電子申請可）
- BL 進行を止めない判断: 個人名義のみで開設して走り、屋号変更は後日。先に開設を優先

## 2. Phase A — Square アカウント開設

### 2.1 申込フロー

1. `https://squareup.com/jp/ja/signup` を開く
2. メールアドレス: `setup@restaurantailab.core-driven.com`
3. パスワード設定（1Password に保存）
4. 事業の種類: 「個人事業主」 or 「法人」
5. 事業カテゴリ: 「プロフェッショナルサービス」→ 「ビジネスコンサルティング」
6. 受付方法: 「オンライン決済（リンク・請求書）」を主に選択（対面 POS は不要）
7. 月間想定売上: 「10〜50万円」
8. 電話番号: 田中さん個人携帯（SMS 認証用）

### 2.2 本人確認 (KYC)

- 氏名（漢字 + カナ）
- 生年月日
- 自宅住所（事業所と同じなら同住所チェック）
- 本人確認書類アップロード（マイナンバーカード推奨、表裏）
- セルフィー（一部ケースで要求）

### 2.3 審査

- 通常 **1〜2 営業日**（最短当日）
- 審査中ステータスはダッシュボード上部に表示
- 追加書類要求が来た場合は登録メール宛て → ダッシュボード「アカウントと設定」→ 「アカウントの確認」から再アップロード

> **着手判断**: KYC アップロードまで完了できれば、その後は Square 側の審査待ち。並行して銀行口座登録（Phase B）と Invoice テンプレ作成（Phase C / E）を進められる。

## 3. Phase B — みずほ口座を入金先に登録

1. ダッシュボード →「アカウントと設定」→「銀行口座」
2. 「銀行口座を追加」
3. 入力項目:
   - 銀行コード: みずほ銀行 = **0001**
   - 支店コード: 3 桁（みずほオンラインで確認）
   - 口座種別: 普通
   - 口座番号: 7 桁
   - 口座名義（カナ）: タナカ リク
4. マイクロデポジット認証（1 円入金で確認）または通帳写しアップロード
5. ステータス「認証済み」になれば完了

### 入金サイクル（重要）

| 銀行 | 入金タイミング |
|---|---|
| みずほ・三井住友・三菱UFJ・ゆうちょ・楽天 | **翌営業日**（自動入金） |
| その他都市銀・ネット銀 | 翌営業日 |
| 一部地方銀行 | 週 1 回（水曜入金） |

→ みずほは翌営業日で着金するため、Pon さんが月初に決済すれば月初〜中旬には田中さん手元に入る。

## 4. Phase C — 決済設定（カードブランド・機能）

### 4.1 受付カードブランド

Square Japan は標準で以下を受付:

- Visa
- Mastercard
- American Express
- JCB
- Diners Club
- Discover
- 中国 UnionPay（一部）

→ Pon さんのタイ発行クレジットカードが上記いずれかに該当すれば決済可能。
**事前に Pon さんに「お手持ちのクレジットカードのブランドを教えてください（VISA/Mastercard 等）」を確認**（→ §8 LINE テンプレ参照）。

### 4.2 手数料

| 受付方法 | 手数料率 |
|---|---|
| オンライン Invoice / 支払いリンク | **3.6%** （JCB は 3.95%） |
| Square Reader（対面） | 3.25% |
| 海外発行カード | 上記 + 為替変動 / カード会社海外利用手数料 1.6〜2.2%（Pon さん側負担） |

→ **250,000 JPY × 3.6% = 9,000 JPY/月** が田中さんの実コスト。
4 ヶ月遡及まとめ発行（1,000,000 JPY）の場合 **36,000 JPY** が差し引かれる。

> **注意（要確認 §10 参照）**: 9,000 JPY の手数料を (a) 田中さん側が吸収 / (b) 請求額を 259,332 JPY に上乗せして Pon さんに負担 / (c) 別建てで請求、のいずれにするか決める。BL の既存 decision は「250,000 JPY 固定」なので、手取り 250,000 を維持するなら (b) の上乗せ請求が必要。

### 4.3 Square Invoices 機能の有効化

- ダッシュボード →「Invoices」→「最初の請求書を作成」
- 機能は標準で有効。追加料金なし

## 6. Phase E — 本番運用（Pon さんへの請求発行）

### 6.1 発行する Invoice

旧 Wise 設計の遡及スコープを継承し、以下を Square Invoices で発行:

| # | 稼働期間 | 金額（手取り 250,000 JPY 維持の場合） |
|---|---|---|
| 1 | 2026-02-19 〜 2026-03-18 | 250,000 JPY（要 §10 決定: 上乗せ請求なら 259,332 JPY） |
| 2 | 2026-03-19 〜 2026-04-18 | 同上 |
| 3 | 2026-04-19 〜 2026-05-18 | 同上 |
| 4 | 2026-05-19 〜 2026-06-18 | 同上（5 月稼働分まで含めるか §10 で確認） |

### 6.2 Invoice 作成手順（1 件あたり 3 分）

1. ダッシュボード →「Invoices」→「請求書を作成」
2. 顧客:「Pon さん」を新規追加（氏名 + メール + 国＝Thailand）
3. 項目:「AI コンサルティングサービス 2026 年 2 月稼働分（2/19-3/18）」
4. 金額: 250,000 JPY（or 259,332 JPY）
5. 支払い期限: 発行日 + 14 日
6. 「支払いリンクを生成」を選択（メール送付は不要）
7. 発行 → 生成された URL（`https://squareup.com/u/xxxxxxxx`）をコピー

### 6.3 LINE 送付

Pon さんに LINE で送るテンプレ（日英併記）:

- BL-0068 に commitment 記録: 「2026-MM-DD: N月稼働分 入金確認、手取り XXX JPY」

## 7. 想定リスクと対処

| リスク | 対処 |
|---|---|
| Pon さんのカードが Square で reject される | カード会社に「日本のオンライン決済を許可する」設定をしてもらう / 別カードで再決済 |
| 高額決済（25万円）で 3D セキュア要求 | Pon さん側で SMS / アプリ認証 → 通常 1〜2 分で通る |
| Square 審査で追加書類要求 | 開業届 / 事業内容説明書 / 取引先（Pon さん側）情報を提出 |
| 為替手数料の Pon さん側負担が想定外に高い | Pon さんから事前確認、必要なら金額調整 |
| Square の月次入金上限 | 個人事業主は標準で月 100 万円上限の場合あり → 大型決済前に上限引き上げ申請 |

## 11. 旧運用との対応関係（Wise → Square 切替）

| 旧 Wise 運用 | 新 Square 運用 |
|---|---|
| Pon さん個人 Wise → 田中さん個人 Wise → みずほ | Pon さんクレカ → Square → みずほ |
| 田中さん側 Wise KYC（マイナンバー、1〜3 日） | 田中さん側 Square KYC（マイナンバー、1〜2 営業日） |
| Pon さん側 Wise 開設（必須、KYC 必要） | **Pon さん側手続きなし**（手持ちクレカで決済するだけ） |
| 請求書 PDF を LINE で送付、Pon さんが Wise で送金 | 支払いリンク URL を LINE で送付、Pon さんがクレカで決済 |
| 手数料: Wise の送金手数料 1,000〜2,000 JPY/件（Pon さん負担） | 手数料: Square 3.6% = 9,000 JPY/件（田中さん負担、要 §10 確認） |
| 為替: Wise 中値レート + Wise 手数料 | 為替: Pon さんのクレカ会社レート + 海外利用手数料 |
| 着金: Wise → みずほ 数時間〜1 営業日 | 着金: Square → みずほ 翌営業日 |

**Pon さん側の手間は大幅に減る**（Wise 口座開設不要）。一方、手数料は田中さん側に発生するため §10-2 の決定が必要。
