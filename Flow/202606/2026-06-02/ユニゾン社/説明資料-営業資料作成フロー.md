# Claude Code でホームページから営業資料 (PowerPoint) を作る流れ

> 今回 UNI'SON 社の営業資料を作った実例を題材に、Claude Code から PowerPoint を生成する全体フローを説明します。飯竹さん向け。

---

## 1. 全体イメージ (1 枚絵)

```
┌─────────────────┐
│  ① 指示を出す    │   「営業資料をホームページをベースに作って」
└────────┬────────┘
         ▼
┌─────────────────┐
│ ② 情報を読み込む │   ・ホームページ (uni-s-on.com)
│                  │   ・(必要なら) PC 上の他の営業資料
│                  │   ・(必要なら) 社内のテンプレ .pptx
└────────┬────────┘
         ▼
┌─────────────────┐
│ ③ 構成案を作る   │   ← ここで人間が一度 OK/NG を判断 ★
│  (Markdown)      │     「10 枚でこういう流れにします」
└────────┬────────┘     という設計書を先に作る
         ▼
┌─────────────────┐
│ ④ スライドを作る │   Anthropic の PPTX スキル + Python
│  (.pptx)         │   検証用に JPEG / PDF も自動生成
└────────┬────────┘
         ▼
┌─────────────────┐
│ ⑤ Drive に納品   │   指定フォルダに自動アップロード
└─────────────────┘
```

ポイントは **③ と ④ が分かれていること**。「いきなりスライドを作る」のではなく、**まず構成案 (設計書) を作って、それから PowerPoint に起こす** という 2 段階で進めます。

---

## 2. 何をインプットにできるか

Claude Code は、スライドの素材として複数のソースを同時に参照できます。

| 種類 | 例 | 今回使った? |
| --- | --- | --- |
| **田中さんからのプロンプト** | 「10 店舗以上の FC オーナー向け、配布資料として営業資料を作って」 | 🟡 使った |
| **外部のウェブサイト** | uni-s-on.com の本文・画像・色・フォント | 🟡 使った |
| **PC 上の他の営業資料 (.pptx / .pdf / Word)** | 過去案件の提案書、製品カタログ等 | ⬜ 今回は未使用 (使うことは可能) |
| **PC 上の生データ** | Excel の売上、CSV のアンケート結果 | ⬜ 今回は未使用 |
| **社内のテンプレ .pptx** | 会社ロゴ・カラー・標準レイアウトが入った空テンプレ | ⬜ 今回はなし (あれば次回から使える) |

→ **「ホームページ + PC 上の過去資料 + 会社テンプレ」を同時に渡せば、もっと精度の高い資料が作れる** ということです。今回はホームページだけでしたが、次回からは過去資料や会社テンプレも一緒に渡すと、ぐっと品質が上がります。

## 3. 実際の流れ (ステップ別)

### ステップ ① 「営業資料を作って」と指示する

田中さんから一言:

> 「ユニゾン社のホームページをベースに、**10 店舗以上を経営する FC オーナー向け**の営業資料を作ってください。
> 用途は商談での**口頭プレゼンではなく、相手に渡して後で読んでもらう「配布資料」**を想定。1 枚で意味が完結するように、各スライドに必要な情報を入れてください。
> デザインはホームページのカラーに合わせてください。」

これだけです。あとは Claude Code 側で必要な情報を取りに行きます。

**ポイント**: 「誰向け (10 店舗以上の FC オーナー)」「用途 (配布資料、プレゼン用ではない)」を最初に伝えることで、各スライドの情報量・トーン・1 枚あたりの密度が大きく変わります。
- プレゼン用: 1 枚 1 メッセージ、画像と数字中心、口頭で補足する前提
- 配布資料: 1 枚で意味が完結、本文をしっかり書く、後で読み返せる

---

### ステップ ② 外部ソース + PC 上の情報を読み込む

Claude Code は **ホームページを自動で訪問**※ して、以下を取得します:

- 会社のキャッチコピー (「いいもの」で世界がつながる)
- 提供サービスの一覧 (グルメ大学+PLUS / Consulting / GBP)
- ブランドカラー (#7297A9 ブルーグレー / #126D64 ティール / #F09D01 オレンジ)
- フォント (Noto Sans JP)
- 画像 (ロゴ、ヒーロー画像 7 枚、サービス紹介画像 等)

> ※ ここで使っているのが **Playwright** という仕組み。次のコラムで詳しく説明します。

> ### 🔍 コラム: Playwright というウェブ取得ツールの話
>
> 「ホームページから情報を自動で取ってくる」を裏で支えているのが **Playwright** という仕組みです。
> Claude Code から呼べる **MCP (=外部ツール接続規格)** として用意されており、田中さんが「ホームページを見てきて」と頼むと、内部で Playwright がブラウザを起動して、
>
> - 本文の抽出
> - 画像 URL の収集
> - 色や使用フォントの取得 (`getComputedStyle`)
> - スクリーンショット撮影
>
> までを 1 分以内で済ませます。
>
> **何が便利か**:
> - ログイン不要のサイトなら、URL を渡すだけで本文・画像・配色を構造化して取得
> - 動的サイト (JavaScript で内容が変わるサイト) でも、レンダリング後の最終状態を見られる
> - スクリーンショットも撮れるので「見た目の雰囲気を再現したい」用途に強い
>
> **業務でこういうときに便利**:
> - 競合分析資料: 競合 5 社のサイトを巡回して「サービス比較表」を自動生成
> - SEO 監査: 自社サイトのページを全部見て、メタ情報や見出し構造をチェック
> - 営業先調査: 商談前に相手企業のサイトから事業概要・最新ニュースを要約

同じ仕組みで、PC 上のファイル (過去の .pptx, PDF, Word, Excel) も読み込ませることが可能です。次回もし手元に過去の営業資料があれば、それも渡してください。「過去資料のトーンに合わせて」「過去資料で使ったこのフレーズは流用して」といった指示ができるようになります。

---

### ステップ ③ スライドの構成案を Markdown で作る (1 段階目)

ここが **2 段階アプローチの 1 段階目**。いきなり PowerPoint を作るのではなく、**先に「構成案」を Markdown で作ります**。構成案は単なるタイトル一覧ではなく、**1 枚ごとの目的・レイアウト・配置・本文・配色まで指定した「設計書」** です。

#### 構成案の全体像 (10 枚)

```markdown
# UNI'SON 営業資料 (10 店舗以上の FC オーナー向け配布資料) — 構成案

1. 表紙 — フルブリード画像 + キャッチコピー
2. 課題提起 — 多店舗 FC が抱える集客の 3 つの課題 (3 カラム)
3. 解 — UNI'SON の 3 サービス紹介
4. サービス① グルメ大学+PLUS (インフルエンサーキャスティング)
5. サービス② グルメ大学Consulting (TikTok × Instagram 比較)
6. サービス③ グルメ大学GBP (MEO 一括管理)
7. ロードマップ — フェーズ別の使い分け
8. 他社サービスとの比較表
9. お客様の声 (引用カード 2 枚)
10. 会社概要 + 次のステップ
```

#### 1 ページの構成例: スライド 4「サービス① グルメ大学+PLUS」

これだけだと「ふーん」となるので、**1 ページの中身がどこまで詳細に決まっているか** の例を示します。

```markdown
# Slide 4 — サービス① グルメ大学+PLUS

**目的**: 新店オープン局面での即効性を訴求
**レイアウト**: 左 35% ヒーローパネル / 右 65% 解説

## 左ヒーロー (35%)
- 背景: ホームページから取った飲食シーン画像 (slide-02.jpg)
- 上に半透明ブルーグレー (#7297A9 70%) をかぶせる
- 中央に大きな数字バッジ (角丸正方形 オレンジ #F09D01):
    「3 ヶ月」 (白, 48pt bold)
- バッジの下: 「新店オープン期の認知獲得を圧縮」 (白, 14pt)
- 一番下に注釈: 「※ 撮影前にインフルエンサーと企画を擦り合わせ
                   → 高い再現性」 (白, 11pt)

## 右ペイン (65%)
- 上部小タイトル: SERVICE 01 (オレンジ, 14pt)
- 大見出し: グルメ大学+PLUS — インフルエンサーキャスティング
           (ダーク, 26pt bold)
- 本文 (3 行):
    メガインフルエンサーを中心に、業態と相性の合う発信者を選定。
    グルメ大学運営で築いた人脈から、本部側で 1 件ずつ営業せず、
    本部担当者は最終 GO だけ。

- セクション見出し: 「こんな本部に効く」 (ティール #126D64, 16pt bold)
- チェックリスト 4 つ (オレンジ ✓ + 14pt):
    ✓ グランドオープン期の認知獲得
    ✓ 季節キャンペーン / 新商品の一斉告知
    ✓ 既存店リブランド後の再認知
    ✓ 多店舗一斉投入で話題化
```

この粒度を **10 枚分書いておく** ことで、PowerPoint 化のときに迷いがなくなります。

#### なぜここまで詳細に書くか

このタイミングで田中さんが目を通すと、**構成レベルで全部見える** ので:

- 「3 番のサービス紹介順は逆にしたい」 → 順番を入れ替えるだけ
- 「9 番の引用はまだ実例がないから削って、8 番の比較表を大きくして」 → セクション差し替え
- 「サービス① の本文がちょっと弱い、もう少し『本部の手間が減る』感を出して」 → 文言だけ修正

**PowerPoint を作り直すより、Markdown を直す方が圧倒的に速い**。これが 2 段階に分ける最大の理由です。本筋に対して**早い段階で「設計レビュー」が入る** ことで、最終 PPT の手戻りがほぼゼロになります。

### ステップ ④ 構成案をもとに PowerPoint を作る (2 段階目)

構成案が固まったら、Claude Code が **Anthropic の PPTX スキル** を使って PowerPoint ファイルを生成します。

- Anthropic 公式の「PPTX スキル」をベースに、**Python でコードを書いて pptx を生成**します (細かい仕組みは省略、「Python でやっている」程度の理解で OK)
- 構成案 Markdown の指示通りに、10 枚分のスライドを 1 ファイルとして組み立てます
- ホームページから取得したカラー (#7297A9 等) と画像が、ちゃんとスライドに反映されます

**中間生成物 (作業ファイル)**:

| ファイル | 何のため | 残るか |
| --- | --- | --- |
| Python コード (build.py) | スライドを作るための作業用コード | ❌ 実行後に削除 (再現は構成案から再生成すれば良い) |
| **検証用 JPEG (1 枚ずつ画像化)** | 田中さんが目視で品質チェックできるよう、各スライドを画像でも出す | ❌ 検証後に削除 |
| **検証用 PDF (10 枚通し)** | PowerPoint がなくても全体を確認できる形式 | ❌ 検証後に削除 |
| **最終 .pptx** | 納品物本体 | ✅ 残る |

→ Python コードは「1 回作って実行して捨てる」運用です。「同じ営業資料をもう一度作って」と頼めば、構成案 (Markdown) からまた新しいコードが生成されます。

---

### ステップ ⑤ Google Drive にアップロード

完成した .pptx を、指定された Google Drive フォルダに自動でアップロードします。

今回の例:
- `マイドライブ/Uni'son社/PPT例/` フォルダを自動作成
- `.pptx` をアップロード
- リンクを返す → そのまま Slack や Teams で共有可能

「ローカルに保存して、エクスプローラーで Drive にドラッグ&ドロップして…」という手作業が不要です。

## 4. 社内テンプレートがある場合 (次の一歩)

今回はテンプレなしで作りましたが、もし会社で**ロゴ・カラー・標準レイアウトが入った .pptx テンプレ** をお持ちなら、それも一緒に渡せます。

```
インプット:
  ├─ 田中さんからの指示
  ├─ ホームページ (外部ソース)
  └─ corporate-template.pptx  ← ★ ここを追加するだけ
```

これを渡すと:
- **会社カラー・ロゴが自動で全スライドに入る**
- **既存のスライドレイアウト (2 カラム比較、表紙テンプレ等) が活用される**
- **ブランドの統一感が出る** (今回のように「ホームページのカラーを推測して再現」より精度が高い)

PowerPoint アプリで「スライドマスター」表示にして、必要なレイアウトを登録した .pptx を 1 本作っておけば、以後すべての営業資料がそのテンプレベースで作られます。

## 5. 全体のまとめ (覚えてほしい 4 つ)

| # | ポイント |
| --- | --- |
| **①** | **2 段階で作る** — まず構成案 (Markdown) を作って合意、それから PowerPoint 化 |
| **②** | **インプットは複数 OK** — プロンプト + ホームページ + PC 上の過去資料 + 会社テンプレ |
| **③** | **検証も自動** — 完成 pptx と並べて、JPEG / PDF も自動生成して目視チェックできる |
| **④** | **納品まで自動** — Google Drive へのアップロードまで一連の流れで完結 |

## 6. もし社内で本格運用するなら次にやるべきこと

1. **会社テンプレ .pptx を 1 本作る** — PowerPoint で「スライドマスター」を編集して、ロゴ・カラー・標準レイアウト (表紙 / 中扉 / 2 カラム / 比較表 / 数字 / 引用) を登録
2. **過去の良かった営業資料を 1 つ用意** — 「このトーンで」と渡せる参考資料
3. **想定 Drive フォルダを決める** — どこに納品されるかを事前に決めておく
4. **田中さんが Claude Code に投げる指示テンプレを作る** — 「○○社向け、××業界、ターゲットは△△」というフォーマットがあると毎回楽になる
