## 1. エグゼクティブサマリ（IT 担当者向け・1ページ）

> 本資料は PON さん（Rio's Innovation 社長）が自社売上データを Cursor + AI で分析するための技術設計です。**IT 担当者の懸念（データ漏洩・流出）を、追加ハードウェアなし・既存 Power Platform の標準機能のみで解決する案**を提示します。

### 1.1 提案の骨子（30 秒で読める版）

| 項目 | 内容 |
|---|---|
| **誰が使う** | PON さん本人のみ（業務 PC 1 台に限定。スタッフには開放しない） |
| **何のデータ** | Power Platform（Dataverse / SharePoint）上の売上データのみ |
| **どう繋ぐ** | Microsoft 公式 Dataverse MCP / PAC CLI 経由（追加 SaaS なし） |
| **誰の権限で** | 専用サービスプリンシパル `svc-ai-readonly-pon`（**読み取り専用・PII 列除外**） |
| **どこに送る** | Anthropic Claude API（ZDR 契約・学習取込なし）。Cursor は Privacy Mode |
| **追加コスト** | 月 5,000〜10,000 円程度（API + Cursor ライセンス）。ハード追加なし |
| **追加工数** | サービスプリンシパル発行 + ロール作成 = IT 工数 **半日〜1 日** |

### 1.2 IT 懸念に対する 3 段防御サマリ

```
[端末層]   PON さん業務 PC のみ・FileVault/BitLocker・MDM 配下
   ↓
[認証層]   サービスプリンシパル・証明書認証・Conditional Access（IP/デバイス制限）
   ↓
[DB 層]    セキュリティロール 「AI Read-Only」（Read のみ・PII 列除外）
   ↓
[LLM 層]   Anthropic ZDR + Cursor Privacy Mode + PII リダクション
```

→ どの層が破られても次の層で止まる多層構造。**「サービスプリンシパル乗っ取り → 全件流出」は DB 層で物理的に不可能**（書き込み権限なし、PII 列見えない）。

### 1.3 IT 担当者に決めていただきたいこと（5 分で答えられる）

§2 の即答質問票（Q1-Q3 + C1-C8）に答えていただければ、本構成の細部が確定し、サービスプリンシパル発行に進めます。**全項目「答えられない」「IT 判断に委ねる」を許容**しており、デフォルト案で進めます（§3 参照）。

## 3. デフォルト前提一覧（IT 回答が無くても進める場合）

> **IT 担当者が回答を返せない / 回答に時間がかかる場合**、田中さんは以下のデフォルトで設計を進めて差し支えありません。後から IT が回答した時点で差分を反映します。

| 質問 | デフォルト | 理由 |
|---|---|---|
| Q1 バックエンド | **(c) Dataverse + SharePoint 併用想定** | 両対応で準備 → 実際にどちらか不要だった場合、該当 MCP を Cursor から外すだけ。設計の手戻りなし |
| Q2 認証方式 | **(b) クライアントシークレット 90 日ローテ** | 証明書はキーチェーン格納の運用合意が必要。シークレットなら 1Password 等で即開始可。IT が証明書希望なら後で切替可 |
| Q3 Cursor 利用 | **(c) 未利用前提でセットアップ手順を別添** | 安全側で「未利用」と仮定し、設定ガイドを最初から含める。利用中ならスキップ |
| C1 テナント情報 | 田中が PON さんから直接ヒアリング | IT 経由を待たない |
| C2 Entra ID 管理 | テナント管理者は PON さん自身（個人事業主規模想定） | PON さんに直接確認・委任状不要 |
| C3 Conditional Access | **無効前提**で開始 | P1 ライセンス未契約でも開始可。シークレット 90 日ローテ + Dataverse 監査ログで多層防御 |
| C4 Purview Audit | **E3 標準（90 日保管）想定** | 月次 CSV エクスポートで 1 年保管を補完。E5 ならネイティブで OK |
| C5 MDM | **未配下前提** | 遠隔ワイプ不可 → FileVault + 自動ロック 1 分 + 紛失時即 SP 失効で代替 |
| C6 認証情報保管 | **1Password Business** | 田中さん側で同等運用中、推奨できる |
| C7 スタッフ展開 | **PON さん 1 名のみ** | multi-user は明示的な追加要件として後日設計 |
| C8 Cursor 契約 | **Cursor Pro 個人契約**で開始 | Privacy Mode は Pro でも有効化可。組織展開時に Business へアップグレード |

### 3.1 デフォルト案で進めた場合のリスクサマリ

| リスク | 影響度 | 緩和 |
|---|---|---|
| Conditional Access なし → 別 PC からの SP 利用余地 | 中 | シークレット 90 日ローテ + 監査ログ月次レビュー |
| MDM 未配下 → 紛失時の遠隔ワイプ不可 | 中 | FileVault 必須 + 1 分自動ロック + Entra ID 即時 SP 失効（5 分以内伝播） |
| Purview E3 → 監査ログ 90 日のみ | 低 | 月次 CSV → Google Drive バックアップで 1 年保管 |

→ いずれもセキュリティの致命傷にはならない（v2 §2 リスク R3/R8/R9 の代替策で網羅済）。
