1. このマニュアルの使い方

このマニュアルは「新橋店のスタッフが、開発者の手を借りずに自分でできること」をまとめた手順書です。

  • 章立て: 「日常運用」「LP メンテナンス」「トラブル対応」「エスカレーション」の 4 領域
  • 粒度: スマホ / PC のブラウザだけで完結する操作を中心に説明
  • 想定読者: SQL / Git / CLI を触らない店舗運用担当者
  • 「これは開発側」と書いてある操作: 田中 (むしめがね) に依頼
  • このマニュアルに無い操作が出てきた場合: 自己判断で本番に触らず、まず連絡
Note

開発者向けの技術手順 (CLI / SQL / Vercel CLI 等) は docs/operations.md に別途あります。本マニュアルとは別物です。

2. システム全体像 (1 分で理解する)

システム同士の関係

お客様
  │
  ├─→ ① 予約 LP (https://c5med-shimbashi-lovat.vercel.app/)
  │      └─ コース選択 → 日時選択 → 連絡先入力 → 予約確定
  │
  └─← ④ 予約完了メール (店舗 Gmail から自動送信)

店舗 (新橋店)
  │
  ├─→ ② Google Calendar (c5clinic.tokyo.hokuseikai@gmail.com)
  │      └─ 自動で予約イベントが入る (本予約 + 前後 30 分の準備時間)
  │
  ├─→ ③ 予約通知メール (店舗 Gmail 受信箱に届く)
  │
  └─→ ⑤ 管理画面 (/admin)
         └─ 営業時間 / 定休日 / コースの編集
役割システムスタッフが触るか
空き枠の真実Google Calendar (店舗 Gmail)はい (手動で予定を入れれば空き枠から外せる)
店舗 / コース情報管理画面 /adminはい (営業時間・コースを編集)
予約 LP の見た目デプロイ済みサイトいいえ (文言変更は開発側)
顧客への自動メールGmail SMTPいいえ (送信文面は固定)
覚えておく重要ポイント
  1. Google Calendar が真実。店舗 Gmail のカレンダーに何か手動で予定を入れると、その時間はお客様から見て予約不可になる。逆に、システム経由の予約は Calendar に自動で入る。
  2. 管理画面で営業時間を変えると、最大 60 秒で LP に反映される (即時ではない)。
  3. コース写真や文言の差し替えは、コースは管理画面で OK / LP 全体は開発側。

4. 管理画面 (/admin) へのログイン

4.1 手順

  1. ブラウザで https://c5med-shimbashi-lovat.vercel.app/admin にアクセス
  2. 「Google でサインイン」をクリック
  3. 許可された Gmail アカウント (店舗 Gmail or 開発側で登録済みアカウント) を選ぶ
  4. 認可されると /admin のトップ「店舗一覧」が表示される

5. 日常運用

5.1 予約の確認 (今日 / 今週の予約を見たい)

方法 1: Google Calendar (推奨)

  1. 店舗 Gmail でログインしたカレンダーを開く
  2. 「日」「週」表示で予約状況を確認
  3. 1 件の予約に対して 3 つのイベントが並ぶ仕様 (これは正常):
    • 「準備時間 (前)」… 開始 30 分前から (薄色 / 透明)
    • 「本予約」… 実際の施術時間
    • 「準備時間 (後)」… 終了 30 分後まで (薄色 / 透明)
  4. 本予約イベントを開くと「お客様名 / メール / 電話 / コース」が記載されている
補足

薄色の準備時間イベントが消えない / 見えないという場合: それは正常です。お客様向けの「空き枠」には影響しません。

方法 2: 店舗 Gmail の通知メール

新規予約が入るたびに c5clinic.tokyo.hokuseikai@gmail.com に「【新規予約】〜」という通知メールが届きます。Gmail の検索窓で subject:"【新規予約】" と入れると一覧できます。

5.2 営業時間を変える

例: 火曜日を 11:00-19:00 に変更したい / 水曜は午前のみ営業にしたい。

  1. /admin にログイン → 「店舗」→ 「新橋店」をクリック
  2. 「営業時間」セクションで該当曜日の [+ 時間帯追加] を使って入力。1 日に 2 つ以上の時間帯 (例: 10:00-13:00 と 15:00-19:00) も設定可能 (休憩を挟む場合)
  3. 画面下部の「保存」をクリック
  4. 最大 60 秒後に LP の空き枠表示に反映 (キャッシュが切れるまで)
注意

当日分の営業時間は変えない。すでに当日分の予約を受け付けてしまった場合、整合性が崩れる。前日までに変更してください。

5.3 定休日 / 臨時休業を追加する

例: 「年末年始 12/30-1/3 を休業にしたい」「9 月 15 日臨時休業」

  1. /admin → 「新橋店」→ 「定休日」セクション
  2. カレンダーで休業日をクリック (複数日連打で OK)
  3. 「保存」
  4. その日は予約フォームのカレンダーに表示されなくなる
注意

既に予約が入っている日を定休日にしても、その予約は自動キャンセルされません。個別連絡が必要です (5.4 参照)。

5.4 予約をキャンセルする (店舗側都合 / お客様希望)

必ずこの順番で実施します:

  1. Google Calendar で該当の 3 イベントを削除: 本予約 / 準備時間 (前) / 準備時間 (後) の 3 つ全部 (同じお客様名 / 同日のもの)。本予約 1 件だけ消しても空き枠としては復活するが、見た目の混乱を避けるため 3 件まとめて削除
  2. お客様にキャンセル連絡 (メール or LINE)。専用テンプレなし。営業文面で対応
  3. (任意) お客様の事情で全額返金などが必要な場合は、店舗運用ルールに従う
DB について

DB から予約レコードを消したい場合は開発側に依頼。スタッフは Calendar 削除までで OK。

5.5 コースの追加 / 編集 / 写真差し替え

新しいコースを追加

  1. /admin → 「コース」→ 「+ 新規作成」
  2. 以下を入力:
    • コース名 (例: 初回体験 60 分コース)
    • 説明 (任意)
    • カテゴリ (facial / body 等、既存の値を使う)
    • 所要時間 (分単位、例: 90)
    • 価格 (税込円、例: 13980)
    • タグ (任意、カンマ区切り)
    • 並び順 (数字、小さい方が先頭)
    • 画像 (PNG/JPG、推奨 1080x720px 程度)
  3. 「保存」
  4. 重要: 続けて「店舗」→「新橋店」→ 「コース紐付け」で新コースをチェック → 保存しないと LP に出ない

既存コースを編集 (価格変更・説明変更など)

  1. /admin → 「コース」→ 該当コースの「編集」
  2. 内容を変更 → 「保存」

コース写真を差し替え

  1. /admin → 「コース」→ 該当コース → 「編集」
  2. 既存画像の下の「画像を変更」→ 新ファイル選択
  3. 「保存」(古い画像は自動的に削除される)

コースを一時非公開にする (一覧から消したい)

「編集」画面の isActive のチェックを外して保存。再公開はチェックを戻すだけ。

7. トラブル対応 FAQ

7.1 お客様「予約完了メールが届かない」と連絡があった

  1. 迷惑メールフォルダを見てもらう。Gmail から Gmail 宛は通常届くが、他社メーラー (Outlook / iCloud 等) は迷惑フォルダ行きの可能性
  2. 店舗側の Gmail で予約通知メールが届いているかを確認 (件名: 【新規予約】〜)
    • 届いている → 予約自体は成立済み。お客様には口頭/別途メールで日時を再案内すれば運用上問題なし
    • 届いていない → システム障害の可能性 → 7.5 へ
  3. メールアドレスのスペルミスがないかお客様に確認 (画面入力ミス)
  4. それでも未着 → 開発側にエスカレーション

7.2 お客様「予約をキャンセル / 変更したい」と連絡があった

  • キャンセル: 5.4 の手順で対応
  • 日時変更: 「現予約をキャンセル + LP から新規予約取り直し」が確実
  • コース変更だけ: Google Calendar イベントを開いて備考欄に変更内容を記載 + 当日対応 (DB を直接変える必要なし)

7.3 お客様「予約フォームでエラーが出る / 進めない」

  1. お客様にブラウザ名と画面のスクリーンショットを聞く
  2. スタッフ自身のスマホで LP を開いて再現を試す
    • 再現する → 開発側に即連絡 (7.6 障害テンプレ)
    • 再現しない → お客様側のブラウザ問題の可能性。「別のブラウザ (Chrome / Safari)」「シークレットウィンドウ」「最新版に更新」を案内
  3. お客様が急いでいる場合は電話 / LINE から手動受付 → スタッフが Calendar に直接予定を入れて対応

7.4 スタッフ「店舗 Gmail に予約通知メールが来ない」

  1. Gmail の迷惑メール / すべてのメール / ラベル振り分けを確認。送信元が自分自身 (c5clinic.tokyo.hokuseikai@gmail.com) のため、Gmail のフィルタによっては「送信済み」に分類されている場合あり。検索: subject:"【新規予約】" newer_than:7d
  2. 1 件も見つからない場合は、テスト予約 (自分のメールで予約)
    • 通知が来る → 単発の取りこぼし、運用上問題なし
    • 通知が来ない → 開発側に連絡 (STORE_NOTIFY_EMAIL 未設定の可能性)

7.5 スタッフ「Google Calendar に予約予定が入っていない」

  1. お客様の「予約完了画面のスクリーンショット」 or 「お客様宛の予約完了メール」があるか確認
  2. ある → システムは予約を受け付けたが Calendar 反映に失敗した可能性 → すぐに開発側に連絡 (お客様にはまず手動で電話 / メール確認を入れる)
  3. ない → そもそも予約が成立していない可能性。お客様に「予約完了メールの有無」を確認

9. 運用 FAQ

Q. 同じ時間帯に手動で 2 件目を入れたい

A. Google Calendar に直接イベントを追加すれば、システム経由の予約はその時間を避ける。ただし「準備時間 (前)」「準備時間 (後)」までは自動でブロックされないので、施術 30 分前後の余裕を手動で確保する必要あり。

Q. 1 日 1 件だけ特別枠 (VIP 等) を作りたい

A. 該当時間帯を Google Calendar で「VIP 枠」イベントとしてブロックしておく → 当日にお客様を別途案内。

Q. 営業時間外の問い合わせをどう捌くか

A. お客様の問い合わせは LINE 相談ボタン (https://lin.ee/y4yB6lD) に集約されている。営業時間内にまとめて返信。

Q. お客様から「コースの所要時間が思っていたより短い」と言われた

A. 管理画面 → コース編集 → 「所要時間 (分)」を変更。ただし変更後の所要時間は新規予約から適用。既存予約の時間枠は Calendar 上で別途調整が必要。

Q. 予約画面のテスト方法

A. スタッフ自身のメールアドレスで実際に予約を入れて、メール / Calendar まで通しで確認する。テストが終わったら 5.4 の手順でキャンセル。

10. 月次チェックリスト

毎月 1 日に以下を確認しておくと事故が減ります:

  • ☐ 翌月の定休日 / 臨時休業を /admin に登録済みか
  • ☐ 翌月のキャンペーンに合わせたコース価格の更新は済んでいるか
  • ☐ 店舗 Gmail の予約通知が直近 1 週間で 1 件以上届いているか (届いていない場合、システム障害の可能性)
  • ☐ テスト予約 1 件 → メール 2 通 (顧客 + 店舗) → Calendar 反映を通し確認
  • ☐ 予約 LP / 管理画面の URL が変わっていないか (本マニュアルの URL を再確認)
