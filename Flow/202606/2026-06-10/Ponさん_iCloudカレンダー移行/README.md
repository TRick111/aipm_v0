# ICSファイル分割スクリプト（PONさん向け）

iCloudカレンダーからエクスポートした `.ics` ファイルが大きすぎて Googleカレンダーに
インポートできない場合に、ファイルを安全に複数に分割するためのスクリプトです。

## 何ができるか

- 1つの大きな `.ics` ファイルを、N件ずつの小さな `.ics` ファイルに分割します
- 分割後のファイルはそれぞれ単独で有効な ICS ファイルなので、Googleカレンダーに
  そのままインポートできます
- カレンダーのヘッダ情報やタイムゾーン定義 (VTIMEZONE) は全ての分割ファイルに
  正しく複製されるため、時刻がズレることはありません

## 動作環境

- macOS（標準で入っている `python3` を使うので、追加インストールは不要です）

念のため、ターミナルで以下を実行して python3 が入っているか確認してください:

```bash
python3 --version
```

`Python 3.x.x` と表示されればOKです。
もし「コマンドが見つかりません」と出た場合は、ターミナルに `xcode-select --install` と
入力して、出てきたダイアログでインストールしてください（10分ほどかかります）。

## 使い方

### 1. ファイルを用意する

このフォルダ（`split_ics.py` が入っているフォルダ）に、iCloudからエクスポートした
ICSファイルをコピーします。ファイル名は何でもOKですが、ここでは
`calendar.ics` とします。

### 2. ターミナルでこのフォルダを開く

Finder でこのフォルダを開き、フォルダのタイトル部分を右クリック →
「フォルダに新規ターミナル」を選ぶのが一番簡単です。

または、ターミナルを開いて `cd ` の後にフォルダを Finder からドラッグしてEnter。

### 3. スクリプトを実行する

```bash
python3 split_ics.py calendar.ics
```

これで、デフォルトで **500件ずつ** に分割されます。

分割数を変えたい場合:

```bash
# 1ファイルあたり 300件 にする
python3 split_ics.py calendar.ics --events 300

# 1ファイルあたり 1000件 にする
python3 split_ics.py calendar.ics --events 1000
```

実行すると以下のような出力が出ます:

```
入力ファイル: /Users/.../calendar.ics
サイズ:       12.3MB
出力フォルダ: /Users/.../calendar_split

読み込み & 解析中...
  VEVENT:          4823 件
  VTIMEZONE:       2 件（全分割ファイルに複製）

分割中 (500 件/ファイル, 計 10 ファイル)...
  calendar_part01_of_10.ics  (500 件 / 1.4MB)
  calendar_part02_of_10.ics  (500 件 / 1.4MB)
  ...
  calendar_part10_of_10.ics  (323 件 / 0.9MB)

完了しました。
```

### 4. Googleカレンダーにインポートする

1. https://calendar.google.com を開く
2. 左サイドバーの「**他のカレンダー**」の横にある **＋** をクリック
3. 「**インポート**」を選択
4. `calendar_split` フォルダの中の `.ics` ファイルを **1つずつ**選んでアップロード
5. インポート先のカレンダーを指定して「インポート」

これを分割ファイルの数だけ繰り返してください。

## トラブルシューティング

### Googleカレンダーで「不明なエラー」が出る場合

`--events` の値を小さくしてください（例: 500 → 300 → 200）。
1ファイルあたりのサイズが 1MB を超えていると失敗しやすくなります。

### 「同じ予定が重複した」

同じ ICS ファイルを2回インポートすると、同じ予定が重複して入ります。
Googleカレンダー側で重複削除する機能はないので、もし重複してしまった場合は、
インポート先のカレンダー（テスト用に新規作成しておくのが安全）ごと削除して、
やり直してください。

**おすすめの進め方:**
1. まずGoogleカレンダーで「iCloudインポート用」という空のカレンダーを新規作成する
2. その新規カレンダーに分割ファイルをインポートしていく
3. うまくいったら、メインカレンダーと統合するか、そのまま使う

### 1ファイルだけテストしたい

最初は `part01_of_XX.ics` だけインポートしてみて、件数が合っているか、
時刻がズレていないかを確認してから残りを進めると安全です。

## このスクリプトについて

- ICSファイルを物理的に分割しているだけで、内容は一切変更しません
- 行折り返し（line folding）や CRLF も RFC 5545 に準拠して再構成しています
- 元の `calendar.ics` には触らないので、何度でも実行し直せます
