# 飲食店売上分析ダッシュボード｜課題 → 要件 → 機能 マッピング

営業資料インプット用。現行仕様から逆引きで「どの課題を、どの要件として咀嚼し、どの機能で解決しているか」を一覧化。

- 作成日: 2026-06-12
- 対象プロダクト: 飲食店売上分析ダッシュボード（ぽっくんPa / Airレジ / Dinii 対応）
- 想定読者: 営業資料作成者（その先の顧客は飲食店オーナー／多店舗経営者／コンサル）

---

## A. 売上分析の「自社カスタマイズ」課題

### A-1. POS標準ダッシュボードは「全社平均値」しか出ない

| 課題                                               | 生まれた要件                     | 実装機能                                                              |
| ------------------------------------------------ | -------------------------- | ----------------------------------------------------------------- |
| 客単価にお土産・物販・デリバリーが混ざり、店内体験の実態が見えない                | 客単価計算から特定カテゴリを除外できること      | **客単価除外カテゴリ設定**（設定タブ、店舗単位、全タブに自動反映） `docs/01_design/ダッシュボード設計.md` |
| POSのデフォルト分類では自店の「ランチ／ディナー／コース／アラカルト」を分けられない      | 売上カテゴリを店舗ごとに自由定義・並び替え可能にする | **PL設定：売上カテゴリ管理**（追加/編集/削除/D&D並び替え） `app/store/[id]/pl/settings/` |

### A-2. 曜日別／時間帯別の動的な分析ができない

| 課題                                   | 生まれた要件                            | 実装機能                                            |
| ------------------------------------ | --------------------------------- | ----------------------------------------------- |
| POSは「日次合計」止まりで、何曜日の何時にピークがあるか可視化できない | 曜日 × 時間帯のヒートマップで売上／来店／客単価を切り替え可能に | **曜日・時間帯分析タブ**（24h × 月〜日、15–85パーセンタイル正規化で外れ値除外） |
| 月の中で週ごとの動きが違うのに均してしまう                | 月全体／第1〜6週で絞り込み                    | **週フィルター**（月曜開始週）                               |
| 「ピークに人を寄せたい」が、勤怠と突き合わせができない          | 同じヒートマップ上に当番スタッフ数（社員／バイト内訳）を重ね表示  | **勤怠オーバーレイ**                                    |

### A-3. 「売れているもの／支えているもの」が分からない

| 課題                     | 生まれた要件                       | 実装機能                                  |
| ---------------------- | ---------------------------- | ------------------------------------- |
| 構成比上位を感覚で把握しているだけ      | 売上累積構成比でA/B/Cに自動仕分け          | **ABC分析**（A:0–70%／B:70–90%／C:90–100%） |
| セット提案・並び替えの根拠データがない    | カテゴリ × カテゴリ、商品 × 商品の同時購入率を出す | **クロスセル分析**（カテゴリ間ヒートマップ＋商品ペアTop20）    |

## B. 「ダッシュボードはあるけどレポートにならない」課題

### B-1. 経営会議で配れる体裁の資料がない

| 課題 | 生まれた要件 | 実装機能 |
|---|---|---|
| ダッシュボードのキャプチャを貼って体裁を整える作業が毎週発生 | 週次の経営レポートをそのまま会議資料として配布可能に | **週報機能**：自己完結HTML（CSS/JS inline）をBlobに保管、年×週で一意管理、ブラウザ／iframeで配信 `docs/weekly-reports-spec.md` |
| 数値が出ていても「今週何を見るべきか」がない | 注目すべき指標・コメントを織り込んだ"読み物"形式に | **HTML週報のフォーマット化**（管理者がCLI経由でアップロード→店舗オーナー閲覧） |
| 週次レビューが残らない | 過去週報のアーカイブと振り返りが必要 | **年別グループ表示／過去週の再閲覧** |

### B-2. 月次の経営会議向けまとめ

| 課題 | 生まれた要件 | 実装機能 |
|---|---|---|
| 月次の売上／PL／客層を別ツールで毎回作っている | 月次レポートを単一画面に集約 | **月次レポート画面**（`/store/[id]/monthly-reports/`） |
| 前年同月比など"会議で必ず聞かれる切り口"が手元で出ない | 月次推移・前年比較 | **月次推移タブ（trends）** |

## C. 店長のコスト意識・利益意識が薄い課題

### C-1. 「今日の発注がいくら利益を削るか」が現場で見えない

| 課題 | 生まれた要件 | 実装機能 |
|---|---|---|
| 売上だけ見ていて、コスト側は感覚で発注している | 売上と同じ画面で経費を日次入力させ、当月の利益着地を毎日見せる | **PL日次入力画面**（売上カテゴリ／経費カテゴリ別、経費タイプ＝定常 or 例外） `docs/01_design/PL管理設計.md` |
| 月末まで利益が分からない | 経過営業日から「着地予想売上」「着地予想経費」「見込み利益」をリアルタイム算出 | **PLダッシュボード**：着地予想売上 ＝（累計÷経過営業日）×営業日数／経費は定常経費を線形補外／利益＝差分 |
| 定休日や月の日数差で着地予想が毎月狂う | 営業日設定（定休曜日＋例外休日）を店舗ごとに登録 | **営業日カレンダー設定** |
| カテゴリ別の予算超過が"事後"でしか分からない | 月次目標を売上カテゴリ別・経費カテゴリ別に立てて、達成率と達成見込みを並列表示 | **月次目標設定画面 + 達成率／達成見込み%** |

## D. 「日報が形骸化していて店舗の雰囲気が分からない」課題

### D-1. 同じテンプレで全員に書かせるから内容が均質化する

| 課題 | 生まれた要件 | 実装機能 |
|---|---|---|
| スタッフの役割が違うのに同じ質問では深い情報が出ない | スタッフを役割別に分け、それぞれに刺さる質問を出し分け | **スタッフ別質問テンプレート**（A: 店舗責任者／B: 外国人スタッフ／C: 一般、英語/日本語切替） `docs/01_design/日報スタッフ設定.md` |
| 「今日も忙しかったです」で終わってしまう | 1問目の回答に対し、AIが文脈に応じた深掘り質問（最大2回） | **AI深掘りプロンプト**（役割×質問別に体系化、疲労配慮・選択肢提示） |

### D-2. オーナーが現場の空気を取りに行けない

| 課題                       | 生まれた要件                        | 実装機能                                                                                                           |
| ------------------------ | ----------------------------- | -------------------------------------------------------------------------------------------------------------- |
| 全員の日報を毎日読むのは現実的でない       | スタッフ全員分を1本に統合した「店舗まとめ日報」を自動生成 | **AI店舗サマリー**（`dailyReportSummaryPrompt`／個別回答も要約）                                                               |
| 経営会議に毎日のサマリーを取りに行く動線がない  | PLダッシュボードと同じ画面に最新日報サマリーを併載    | **PLダッシュボード内 日報セクション**（未締め時は直近締めまとめ＋個人日報フォールバック）                                                               |
| オーナー・本部にプッシュで届かない        | 「締め」操作で自動的にLINEへ配信            | **LINE通知連携**：①日次売上速報（本日／今月見込み／達成見込%）＋②AI一言コメント（40字以内、gpt-4o-mini）＋③日報サマリー `docs/03_dev-notes/0217_LINE連携機能.md` |

## E. 「指標が多すぎて何を見ていいか分からない」課題

| 課題                                   | 生まれた要件                       | 実装機能                                               |
| ------------------------------------ | ---------------------------- | -------------------------------------------------- |
| ダッシュボードに指標が並んでいるだけで「今週何を見るべきか」が示されない | "今週見るべきこと"を編集者がまとめた読み物として配信  | **週報機能**（B-1と同じ。"自動可視化"ではなく"今週の論点を絞ったレポート"として位置付け） |
| 売上速報の中で「目標達成見込み」だけは毎日見たい             | LINEに見るべき1〜2指標を絞って毎日送る       | **LINE売上速報**（売上／今月見込／達成見込%／一言コメント）                 |

## F. 機能側から見えた追加課題（営業で刺さりやすい論点）

### F-1. 複数店舗オーナー／エリアマネージャーが管理しづらい

| 課題                        | 生まれた要件                     | 実装機能                                                                                                                          |
| ------------------------- | -------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| 店舗ごとに別アカウント／別ログインで運用が破綻する | 1アカウントで複数店舗にアクセス、店舗選択画面を経由 | **グループアカウント**（`store_groups` / `store_group_members` / `store_group_sessions`、`/group/select`） `docs/01_design/店舗グループ認証設計.md` |

### F-2. 店舗ごとに運用ルール・必要機能が違う

| 課題                                          | 生まれた要件                           | 実装機能                                                                               |
| ------------------------------------------- | -------------------------------- | ---------------------------------------------------------------------------------- |
| 店ごとに「日報のポジション入力欄」「LINE通知」「売上控除」のON/OFFを変えたい | 機能フラグを店舗単位で管理、店長は値だけ／管理者はフラグごと操作 | **店舗別機能フラグ＋カスタム設定**（`featureFlags` / `customSettings`） `docs/01_design/店舗別機能設定.md` |
