## 0. 背景と要望の構造

竹矢さんが既存ダッシュボード（売上分析・PL）に対して「マーケ／集客系のデータを束ねたい」という追加要望をしているのが今回のヒアリングの主旨。背景には以下がある。

1. **CMOプラン／CFOプランの外注化**を今月中に開始予定。外注コンサルにクライアントの食べログ／ホットペッパー等の **ログインID・パスワードを直接渡したくない**（支払情報など機密情報まで見えてしまう）。
2. **マーケティングパネル（自社の改善ロジック）** はあるが、現在は各媒体に手作業でログインして数字を集計しており、コンサル品質と運用負荷が律速になっている。
3. **アウトプットはダッシュボード単体ではなく、スライド形式**で進行したい（既に週報・月報の納品形態がスライド中心へ移行している流れと一致）。
4. **更新頻度は月1回のアップロードで十分**（マーケはデイリーで動かす世界ではないため）。
5. **店舗ごとに契約媒体が異なる**ため、全店舗で同じUIを強制するより「各店舗が契約している媒体だけ表示」する柔軟性が必要。

→ つまり今回のスコープは **「各媒体のレポートを月次CSV等で取り込み、横断ビューにし、スライド形式で出力する」マーケ向け拡張**。

## 1. 既存ダッシュボードの位置づけ（前提整理）

| 領域                                            | 既存状態                               | 今回の要望との関係                 |
| --------------------------------------------- | ---------------------------------- | ------------------------- |
| 売上分析（ホーム／曜日・時間帯／メニュー／商品詳細／月次推移／全体サマリー／客数／客単価） | 実装済（POS： Airレジ / Dinii / URegiなど）  | 実装済み。今回は触らない              |
| PL管理（日次入力・月次予実・経費カテゴリ）                        | 実装中（律速：客単価計算修正・予実UI・経費カテゴリ・PC日次入力） | 実装済み。今回は触らない              |
| 認証（管理者／店舗ユーザー／店舗グループ）                         | 実装済                                | **外注コンサル用ロール**を新規追加する必要あり |
| アップロード（CSV、Vercel Blob）                       | 売り上げについてのみ実装済　                     | **マーケ系CSV取り込み**に枠組みを流用    |
| 日報・LINE連携                                     | 実装済                                | 今回は触らない                   |
| アウトプット                                        | ダッシュボードUI中心                        | **スライド／PDF出力**を新規追加       |

→ 既存の認証・アップロード・月単位データ管理の枠組みを活かし、**「マーケ媒体取込→集計→スライド出力」レーンを増築**する形になる。

## 2. 要件一覧（カテゴリ別）

ID は M（マーケ取込）、A（分析）、O（アウトプット）、S（セキュリティ）、X（横断）で分類。優先度は仮置きで、最終的に竹矢さんに決めてもらう前提（推奨優先度は §3 で示す）。

### 2-1. データソース取り込み（M：マーケ媒体）

| ID | 要件 | 内容 | 取り込み方式 | 仮優先度 |
|---|---|---|---|---|
| M-01 | 食べログ レポート取り込み | 月次PV、予約数、アクセスレポート、ページ別タップ数 | 月1回CSV/スクショアップロード（最低でもCSV手動DL→アップ） | ★★★ |
| M-02 | 食べログ ランキング情報 | エリア内ランキング・自店ポジション・トップ100位置 | 食べログレポートに含まれる数値を抽出 | ★★ |
| M-03 | 食べログノート（予約管理） | 予約数、フラット来店数、予約数とフラットの比率 | 月次CSVアップロード | ★★★ |
| M-04 | ホットペッパー レポート取り込み | 月次PV、予約数、流入数 | 月次CSVアップロード | ★★★ |
| M-05 | Google ビジネスプロフィール（GBP） | 検索キーワード、ビジネス検索／マップ検索別アクセス、周辺シェア | GBP API（既存PoCコードあり：口コミ取得Node.jsツールを拡張）／CSV手動も可 | ★★★ |
| M-06 | Instagram（投稿ごとエンゲージメント） | 投稿別の閲覧数・いいね・保存・コメント | 当面は**クライアントから受領するスクショ＋手入力 or CSV**（API直叩きはコスト高） | ★★ |
| M-07 | （将来）Google マップ口コミ取り込み | 既存PoC（GBP OAuth全件取得）を本ダッシュボードに統合 | 既存Node.jsツール統合 | ★（将来） |
| M-08 | 媒体ごとアップロード画面 | 媒体別タブ＋ファイル形式ガイド＋プレビュー | 既存Upload画面を媒体別に拡張 | ★★★（M-01〜04の土台） |

> **取り込み方式の方針**: ヒアリングでは「人手で月次CSVをダウンロードできる範囲なら手動アップロードでよい」と明言されている。よって自動スクレイピングは今回スコープ外。GBPだけ既存PoCがあるので段階的にAPI化を狙う。

### 2-2. データ集計・分析（A：分析）

| ID | 要件 | 内容 | 既存資産との関係 | 仮優先度 |
|---|---|---|---|---|
| A-01 | 媒体横断KPIサマリー | 各媒体の月次主要指標を1枚に集約（PV／予約数／コンバージョン／単価） | 売上ダッシュボード「全体サマリー」タブの拡張枠 | ★★★ |
| A-02 | マーケティングパネル準拠の数字並び | 竹矢さんのマーケパネル（再生数→フック→技術→Aquisition→Lost等）の順で数値を縦に並べる | パネル定義をJSONで持って画面定義に流す | ★★★ |
| A-03 | おかしな数字検出（アラート） | パネル各指標の閾値と改善打ち手をマスタ化して、月次で「弱い指標＋打ち手」をハイライト | 週報の監査スキル（25項目）と同じ思想で実装可 | ★★ |
| A-04 | 近隣店舗比較（口コミ平均・単価・PV） | 食べログ等のエリア内ランキング由来の数値で「自店の相対位置」を出す | 食べログレポートに含まれる範囲のみ初期実装 | ★★ |
| A-05 | （将来）周辺店舗データ統合ビュー | エリア内クライアント店舗が増えてから、近隣店舗データを横断 | **エリアの7割の店舗を保有するまで実装しない**（竹矢さん明言） | ★（保留） |
| A-06 | （将来）エリア人口・通行人数連動 | 第二段階要望。住所から徒歩圏人口・通行量を引いて表示 | 外部API（estat／人流データ）連携。コスト要評価 | ★（将来） |
| A-07 | Instagram コンテンツ別分析 | 投稿テーマ別の閲覧数・保存数を時系列で見える化 | 入力がスクショ手打ち or CSVのため、入力さえあれば集計は軽い | ★★ |

## 3. 推奨優先順位（叩き）

竹矢さんに最終確定してもらう前提の **田中側の推奨順序**。判断基準は「（a）一番効くマーケ媒体から始めて成果が見える、（b）既存資産で安く実装できる、（c）セキュリティのボトルネック解消が外注化のゴールである」。

### Phase 1（〜4週間：外注化スタートに最低限間に合わせる）
- **S-01／S-02／S-03（コンサル用ロール＋ACL＋ログイン情報集中管理）** ※ 外注化が始まる前にここが無いと意味がない
- **M-08（媒体別アップロード画面の枠）**
- **M-01／M-03／M-04（食べログ・食べログノート・ホットペッパー の月次CSV取り込み）**
- **X-01（店舗ごとの契約媒体フィーチャーフラグ）**
- **A-01（媒体横断KPIサマリー：1枚ビュー）**
- **X-02（運用フロー定義書）**

### Phase 2（〜6週間：マーケパネル準拠と外部出力）
- **A-02（マーケティングパネル準拠の並び）**
- **A-03（おかしな数字検出＋打ち手提示）**
- **M-05（GBP取り込み：CSV→既存PoCのAPI拡張）**
- **O-01（スライド形式エクスポート）**
- **A-04（食べログレポート由来の近隣比較）**

### Phase 3（〜8週間：Instagram と運用磨き込み）
- **M-06／A-07（Instagram スクショ取込＋投稿別分析）**
- **O-02／O-03／O-04（PDF出力／コメント／スナップショット）**
- **S-04（監査ログ）**

### Phase 4（将来 / 条件付き）
- **A-05（エリア内クライアント店舗が増えたら近隣店舗統合ビュー）**
- **A-06（人口・通行人数）**
- **M-07（口コミPoC統合）**
- **X-03（SNS分析SaaSの内製/外部の最終判断）**
