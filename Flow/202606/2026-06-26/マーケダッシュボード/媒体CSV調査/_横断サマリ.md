## 1. 一目で把握する媒体別ステータス

| 媒体 | 公式 CSV 出力経路 | 公開情報の確証度 | Phase 1 自動化の現実解 | 2FA |
|---|---|:---:|---|:---:|
| **食べログ（owner.tabelog.com）** | 来店指標・アクセス解析・月間ページレポート — **公式ヘルプに CSV ボタンの明記なし** | 低 | スクレイピング前提（規約注意）／Rears 等の中間 SaaS 検討 | 不明 |
| **食べログノート（owner.tabelog.com/tbnote）** | 「分析機能」配下の **9 レポート** が公式に CSV 出力対応 | **高** | Playwright 自動化が成立する見込み（他社台帳経由の店舗は不可） | 公式記載なし |
| **一休本体（restaurant.ikyu.com）** | 公開情報ゼロ。CSV ボタン有無は **実地確認必須** | 低 | RESZAIKO 連携経由が現実解 | 不明 |
| **ホットペッパー（HPG）管理画面** | 直接 CSV DL の公式裏取り不可。代理店記事は HPB と混同の疑い | 低 | マンスリークライアントレポート（営業経由 PDF/Excel）が現実解 | **2FA 導入済** |

→ **「自社で完結する自動取得スキル」が実装しやすいのは 食べログノート と レストランボード のみ**。他媒体は (a) 実機確認で隠れた CSV ボタンを発見する、(b) RESZAIKO 等の予約管理 SaaS 経由で横取りする、(c) マンスリーレポートを OCR/手入力で扱う — の 3 方針を組み合わせる必要がある。

## 2. 媒体ごとの主要発見（個別ファイルへのリンク付き）

### 2.1 食べログ — [`食べログ_CSV出力項目.md`](./食べログ_CSV出力項目.md)（317 行）

- **管理画面 2 系統**: `owner.tabelog.com`（PV / アクセス解析）と食べログノート（予約 / 来店）
- 公式に CSV 明記なのは食べログノート側のみ
- 来店指標（PV / 050 通話 / ネット予約組数 / 地図印刷 PV）は **PR・ネット予約・集客サービス契約店舗限定**
- エリア内ランキング上位 100 は無料店舗でも閲覧可
- **PR 月次マーケットレポート**は営業担当からの月次 PDF 送付
- **規約**: 「事前同意なき複写・複製禁止」条項あり → 本番運用前に法務確認推奨
- **代替案**: **Rears（リアーズ）が食べログ／ぐるなび／HPG 横断のレポートを PDF/CSV 提供** → 自前 Playwright の代替候補

### 2.2 食べログノート — [`食べログノート_CSV出力項目.md`](./食べログノート_CSV出力項目.md)（345 行）

- **CSV 出力可能な 9 レポート**: ダッシュボード／予約数・総客数／卓稼働率／予約コース／予約経路／新規・リピート／予約キャンセル／人数割合／リードタイム
- **重大制約**: ebica / Resty / RESZAIKO / レスラク経由で他社台帳と連携している店舗は分析機能・CSV ダウンロード自体が使えない → **店舗ごとに「食べログノート利用 / 他社台帳経由」フラグが必須**
- **日次境界 06:00〜30:00**（独特仕様）→ 社内 0:00〜24:00 集計と 1 日ズレ → 取り込み時の正規化必要
- 月初に前月分が確定し、確定後は予約編集が過去 CSV に反映されない（=月次スナップショットとして安定取得可）
- TOP20 制約は CSV では外れて全件出力可
- 卓稼働率 ≠ 席稼働率（CSV からは卓ベースのみ）
- 文字コード公式明記なし（Shift_JIS の可能性）→ UTF-8 変換層を初手で組む
- 料金は無料（食べログネット予約契約必須）

### 2.3 一休 — [`一休_CSV出力項目.md`](./一休_CSV出力項目.md)（291 行）

- **一休本体は CSV ダウンロード機能の公開情報ゼロ** → 要・店舗管理画面の実地確認
- **RESZAIKO が実質的なデータソース**:
  - 「予約集計」が公式に CSV 出力対応（**最大 1 年間**、媒体別・時間帯別・リードタイム別）
  - 一休加盟店は **RESZAIKO Lite が無償付帯**
  - 客単価・売上は RESZAIKO Pro（有償）の「オーダー集計」が必要
- 2025/04 にメール取り込み機能リリース／TableCheck 連携有償化など直近変更あり
- Yahoo! JAPAN ID 連携は一般会員側、店舗管理画面側の SSO/2FA は公開情報なし
- 落とし穴: トレタ／TableCheck 移行用 CSV は「当月から 2 ヶ月前まで」のみ

### 2.4 ホットペッパー — [`ホットペッパー_CSV出力項目.md`](./ホットペッパー_CSV出力項目.md)（323 行）

- **公式に明文化された CSV はレストランボード（RB）の「予約リスト CSV」のみ**（20+ 項目、10,000 件 / PC 版限定）
- HPG 管理画面側の CSV/Excel 直接 DL は公式 FAQ で裏取り不可 → 代理店記事は HPB（サロンボード）と混同の疑い → **実機確認必須**
- PV / コール / 流入 FW / お気に入り数 / ページ別タップは画面参照可能だが CSV DL 経路は未確証 → **マンスリークライアントレポート（営業経由 PDF/Excel/PowerPoint）が現状の現実解**
- マンスリーレポートは Salesforce 経由でメール添付/対面配布 → **店舗側からの DL URL なし、自動取得不可** → Phase 1 では OCR/手入力で月次値だけ Sheets 投入推奨
- **2FA / ワンタイムパスワード導入済** → 単純な ID/PW 自動ログイン RPA は破綻リスク高
- 2026/01/29 以降「おすすめレポート」→「口コミ」へ仕様変更

### 2.5 オズモール — [`オズモール_CSV出力項目.md`](./オズモール_CSV出力項目.md)（210 行）

- **OZmall ネイティブ管理画面（`restaurantweb_client`）の CSV 公式エクスポートは公開一次情報から確認不可**
  - 画面パスとして `reserve/List.aspx`（予約一覧）、`report/review/ListNew.aspx`（レビュー一覧）、「連携先管理」の存在は確認
- **CSV 取得の現実解は RESZAIKO 連携経由**
  - RESZAIKO 公式ヘルプ「予約集計」で媒体別予約件数／人数／時間帯／曜日／リードタイム／予約経路を期間指定 CSV 出力（直近 1 年）
  - OZmall「連携先管理」で API 認証キーを発行 → RESZAIKO に貼る運用
- ebica / トレタ / レストランボード（Air・メール連携）/ TableCheck も OZmall 公式連携対応
- **編集部企画／特集枠経由の予約 ID 分解レポートは公開機能として存在せず** → プラン名→特集名マッピングで近似
- 2FA 状況は公開情報なし
- 成果報酬モデル（月額固定・特集掲載すべて無料、来店確定ベースの送客手数料）

### 3.2 §5.4 自動取得スキル（M-14）への影響

要件 v1.1 §5.4 の前提（ローカル Claude Code + Playwright スキル）はそのまま有効。ただし、本調査により媒体ごとの**自動化難易度**が明確になった:

| 媒体 | 自動化難易度 | 主な障壁 |
|---|:---:|---|
| 食べログノート | **低**（実装容易） | 公式 CSV、他社台帳経由店舗のみ要除外 |
| レストランボード（HPG 予約） | **低〜中** | 2FA／10,000 件上限／PC 版限定 |
| 食べログ本体 | **中〜高** | CSV ボタン未確認、規約注意、スクレイピング前提 |
| 一休本体 | **高** | CSV 経路不明 → RESZAIKO 経由に切り替え |
| ホットペッパー（管理画面） | **高** | 2FA 導入済、CSV 直 DL 未確認 |
| マンスリーレポート（HPG） | **不可** | 営業経由メール、店舗側 URL なし → OCR/手入力 |
| オズモール | **高** | CSV 経路不明 → RESZAIKO 経由に切り替え |

### 3.3 §5.4 への新たな提案

- **「RESZAIKO 経由」を Phase 1.1 の自動取得スキルの主流ルートとして位置付ける**
  - 一休・オズモール・（食べログノート連携可）を 1 スキルでカバーできる
  - 加盟店は RESZAIKO Lite が無償付帯される媒体もある → 顧客への追加負担なし
  - ただし RESZAIKO アカウントが必要 → §13 確認事項 / T-17 でローカル管理対象に追加
- **「中間 SaaS（Rears 等）を採用するか自前実装するか」を別途意思決定**
  - Rears 採用なら自動取得スキルの大幅短縮になるが、月額コスト発生
  - 田中・吉田で投資判断要
