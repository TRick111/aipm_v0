## 1. サマリ

ホットペッパー周辺で「自動取得を狙えるレポート」と「人手で月初に営業から受け取る前提のレポート」は、以下のように整理できる。

### 1.1 媒体ごとの“データ層”整理

| 層 | 媒体 | 主要データ | 出力形式 | 自動化難易度 |
|---|---|---|---|---|
| A. 予約管理データ | **レストランボード（RB）** | 予約 1 件 1 行（来店日／人数／コース／予約経路／ネット予約フラグ／要望／決済 等） | CSV（明示） | **中**（管理画面 → ボタン操作 → 非同期生成 → DL の 4 ステップ。10,000 件上限） |
| B. 集客アクセス系 | **HPG 店舗管理画面** | 月別 PV／コール数／ネット予約数／ページ別タップ／流入フリーワード／お気に入り（保存）数／クチコミ | 画面表示（CSV／Excel／PDF DL 可否は記事ベースで明示なし。「自動でレポートDL可」「Excel形式で出力」との二次記述あり＝**未確証**） | **中〜高**（公式 FAQ で CSV DL の手順記載が確認できなかった。画面スクレイピングか営業経由が現実解） |
| C. マンスリーレポート | **クライアントレポート／分析カルテ** | PV／予約／コール／流入キーワード上位／予約ニーズ／客数分析／**エリア平均との比較**／同プラン平均との比較 | **PDF／Excel／PowerPoint（営業担当が作成して配布）** | **高**（営業担当からのメール／対面配布が基本。自動取得不可と判断するのが安全） |

### 1.2 マーケダッシュボード Phase 1 への含意（推奨）

- **RB の「予約リスト CSV」**は唯一公式に明文化された CSV 出力で、ここが Phase 1 の自動取得の中核になる。
- **PV／コール／流入フリーワードの月次値**は、CSV 直接 DL の公式手順が確認できなかった。
  - 短期: 営業から受領するクライアントレポート（PDF/Excel）から OCR or 手入力で抽出
  - 中期: 店舗管理画面の HTML スクレイピング（要 2 要素認証突破・ToS 確認）
  - 長期: リクルートに API／データ連携を直接打診（個別契約）
- **2 要素認証／ワンタイムパスワード**が導入済みのため、自動ログインを前提とした単純 RPA は破綻リスクがある。

## 2. ホットペッパーグルメ管理画面の各レポート

> ログイン URL: <https://www.cms.hotpepper.jp/CLN/login/>
> ID／パスワード忘れ時は <https://www.cms.hotpepper.jp/CLN/mgr/remind/remindId/> で再通知可（認証キーは登録メール宛、有効期限 30 分）。

### 2.1 アクセス解析（PV／コール／ネット予約数）

- **出力形式**: 画面表示が基本。CSV／Excel ダウンロードの可否は **公式ヘルプで明確な記載が確認できず**。代理店記事（株式会社レスパ／サロンリザーブ等）では「自動でレポートをダウンロード可能」「Excel 形式で出力」と書かれている一方、ホットペッパー**ビューティー**（サロンボード）側の機能と混同されている疑いが強い。→ 信頼度「低」。**実機確認推奨**。
- **主要項目（画面で確認できる指標）**:
  - 月別 / 週別 / 日別 PV（ページビュー）
  - コール数（電話予約発信数）
  - ネット予約数（成立件数）
  - 来店人数（ネット予約経由）
  - ページ別タップ（コース／クーポン／写真／メニュー 等）
  - お気に入り（保存）数
- **取得手順（推定 / 業界標準ベース）**:
  1. <https://www.cms.hotpepper.jp/CLN/login/> でログイン
  2. グローバルメニュー「アクセス分析」または「レポート」へ遷移
  3. 期間（日別／週別／月別）を選択
  4. （該当する場合）「CSV ダウンロード」「Excel 出力」ボタンを押下
- **契約プラン要件**: 基本掲載プラン以上で参照可。**詳細レポート（分析カルテ等）はラ S 以上の有料プラン**が条件と複数の代理店記事に記載（信頼度: 中）。
- **取得頻度**: 月次集計は前月分が翌月初〜中旬には参照可能。日次更新の指標もあり。
- **備考**: ホットペッパー**ビューティー**側「SALON BOARD」では「月次／日別の PV・予約数が DL なしで参照可」と明示。グルメ側も類似 UI と推察されるが、**公式 FAQ にグルメ管理画面の CSV/Excel DL ボタンの明示記述は見つからなかった**。

### 2.2 流入フリーワードランキング

- **出力形式**: 画面表示が基本。CSV 公式 DL は確認できず。
- **主要項目**: 自店ページに流入する直前のユーザー検索ワードランキング（駅名／個別店名／無効ワードはリクルート集計時には除外。店舗側画面では生データが見える想定）。
- **取得手順**:
  1. 管理画面ログイン
  2. 「アクセス解析」→「フリーワードランキング」または「キーワード分析」
  3. 期間（月別が一般的）を選択して表示
  4. 画面スクリーンショット／手動コピーが現状の主取得方法
- **契約プラン要件**: ラ S 以上で詳細ランキングを閲覧可能との代理店記事複数あり（信頼度: 中）。基本プランでは TOP の一部のみ。
- **取得頻度**: 月次更新。リクルート公式サイト「ホットペッパーグルメ外食総研」では**全国版**の TOP30 が毎月公開されており、店舗別ローカルランキングは管理画面でしか見られない。
- **備考**: ホットペッパー**ビューティー**ではラ S 以上の契約店に対し、営業担当がメール（「ホットペッパー通信」）で FW ランキングを別途送付する運用が存在。グルメ側にも類似の運用がある可能性あり（信頼度: 中）。

### 2.3 クチコミ（旧「おすすめレポート」）

- **出力形式**: 画面表示。CSV／Excel の公式 DL は確認できず。
- **背景情報**: 2026 年 1 月 29 日に口コミ機能リニューアル、2026 年 3 月 1 日以降の投稿から名称が「おすすめレポート」→「口コミ」に変更。審査を満たせばネガティブも掲載される方針へ変更。
- **主要項目**: 投稿日／投稿者名（ニックネーム）／総合評点／カテゴリ別評点（料理／接客／雰囲気 等）／本文／写真／店舗からの返信
- **取得手順**:
  1. 管理画面ログイン → 「クチコミ管理」「お客様の声」相当のメニュー
  2. 期間 / 評点でフィルタ
  3. 一覧をスクリーン／印刷（CSV ボタンは未確認）
- **契約プラン要件**: 全プランで閲覧可。返信機能の差分は要確認。
- **取得頻度**: 投稿は随時。月次サマリは管理画面 or マンスリーレポートで集計される。
- **備考**: ダッシュボード KPI として件数／平均評点／返信率を扱う場合、件数増分は手動カウント or スクレイピングで取得することになる。

### 2.4 予約データ（HPG 経由ネット予約）

- HPG ネット予約は予約台帳 **「レストランボード」へ自動連携される**ため、HPG 管理画面側で個別予約の CSV 出力をする必要は基本的にない。RB 側 CSV を使うのが正攻法（§3 参照）。
- HPG 管理画面の「ご予約状況のお知らせ」「ご予約管理」は、レストランボード未契約店舗向けに最低限の予約情報（予約日／コース／人数）を表示する位置づけ。

## 3. レストランボード（RB）の各レポート

> 公式 FAQ: <https://faq.restaurant-board.airregi.jp/>
> RB は予約台帳 SaaS。HPG 掲載店向け機能で、CSV 出力は**パソコン版限定**。

### 3.1 予約リスト CSV 出力（公式に明文化された唯一の CSV）

- **出力形式**: CSV（UTF-8 想定 / 改行コード未確認）
- **主要項目（FAQ ベース・確認済み）**:
  - 予約受付日
  - ステータス（要連絡 / 仮予約 / 来店待ち / 来店済み / キャンセル 等）
  - 予約経路（ホットペッパーグルメ / 電話 / 自社サイト 等）
  - 店舗名（複数店舗権限の場合）
  - 来店日
  - 開始時間 / 終了時間
  - コース名 / 金額
  - 人数（大人 / 子供 / 合計）
  - ネット予約フラグ
  - テーブル（割当）
  - 要望（顧客からの自由記述）
  - メモ（店舗側メモ）
  - 決済情報（スマート支払い利用有無 等）
  - お客様氏名 / 電話番号 / メール（一部マスクの可能性あり）
  - 合計 20 項目以上
- **取得手順**:
  1. <https://restaurant-board.com/> へ PC ブラウザでアクセスしログイン
  2. 左サイドメニュー「予約台帳」→「予約リスト」または「スケジュール」を開く
  3. 画面右上の「CSV 出力」ボタンをクリック
  4. カレンダーから出力対象の開始日／終了日を選択 →「絞り込む」
  5. 複数店舗の権限がある場合は対象店舗を選択
  6. 「CSV ファイルを出力する」→ 確認画面「作成する」
  7. 数分待機後、左上「メッセージ」画面に**完成通知**が届く
  8. 通知から「ダウンロード」ボタンで CSV 取得
- **契約プラン要件**: RB は基本料 0 円から利用可。ただし**ホットペッパーグルメに掲載中の店舗向け機能**との注記あり。
- **取得頻度**: 任意（オンデマンド）。期間は日付指定で自由。月次運用も可。
- **既知の制約・落とし穴**:
  - **出力上限 10,000 件**: 超過時は期間／店舗を絞って分割実行
  - **PC 版限定**（iPad / iPhone アプリでは不可）
  - **複数コースが同一予約に紐づく場合、1 コース分のみ表示**（コース別集計時は注意）
  - 2024-05 に一時的に CSV DL が失敗する障害発生 → 解消済（FAQ #32487450513561）
  - 2024-10 に「繁忙時間帯の利用制限」が解除（FAQ #16141613389849）

### 3.2 集計分析（来店・予約コース・経路・新規/リピート・キーワード）

- **出力形式**: **画面表示・印刷が基本。CSV エクスポートの可否は公式 FAQ に明確記述なし**（信頼度: 中〜低）。
- **主要項目（FAQ「集計画面について」「項目別」シリーズ）**:
  - コース別予約件数／売上構成
  - 予約経路別件数（HPG ネット / 電話 / 自社サイト / 他媒体 等）
  - 新規 / リピート / 常連 / 不明 / 未入力の割合
  - キーワード分析（顧客カルテに入力したキーワード集計）
  - キャンセル件数 / キャンセル率
- **取得手順**:
  1. RB PC 版ログイン
  2. メニュー「集計・分析」 → 「集計画面」または「項目別」
  3. 期間（今日／今週／今月／カスタム）を指定
  4. グラフ・表を画面で確認、必要に応じてブラウザ印刷
- **契約プラン要件**: PC 版での詳細分析や印刷は HPG 連携店向け機能と注記あり（信頼度: 中）。
- **取得頻度**: 任意（オンデマンド）。月次の振り返り用途で運用可。
- **備考**: ダッシュボード側で「新規 / リピート比」「予約経路シェア」を扱いたい場合、現状は §3.1 の予約リスト CSV を Python で集計し直すのが現実的。

### 3.3 顧客一括登録 CSV（出力／入力）

- **出力形式**: CSV
- **用途**: 既存顧客マスタの一括登録／更新が主目的。出力は「アップロードファイル出力」ボタンから取得。
- **主要項目**: 顧客 ID／氏名／フリガナ／電話／メール／タグ／来店回数／最終来店日 等（FAQ「顧客一括登録 CSV ファイルの出力項目」<https://faq.airregi.jp/hc/ja/articles/33881299664665> 参照）
- **取得手順**: 顧客台帳 → 顧客一括登録 → 「アップロードファイル出力」
- **制約**: 5,001 件以上は 5,000 件単位で自動分割される。
- **備考**: マーケダッシュボードのリピート率算出には使えるが、PII を含むため取り扱いに注意。

## 4. マンスリークライアントレポート（営業経由）

### 4.1 概要

ホットペッパーグルメの**店舗営業担当**または**コンサルティング窓口**が、毎月（または契約期に応じて）店舗向けに作成・配布する詳細分析レポート。
業務サポートパック契約店および一定プラン以上で提供されるとされる（信頼度: 中。公式 FAQ には明確な提供条件の記述なし）。

### 4.2 入手フロー

1. **契約締結時 or 毎月初**に営業担当から「メール添付（PDF or Excel）」または「対面で印刷物を持参」して提供される。
2. 一部の店舗は MTG（月次同行）でレポートを画面共有しながら説明を受ける。
3. **店舗側から能動的にダウンロードする URL は公開されていない**（営業の Salesforce / 社内ツールで生成 → 店舗へ配信）。
4. 受領した PDF / Excel は **手動で保管 / OCR / 手入力**で社内システムへ取り込み。

### 4.3 含まれる項目（代理店ブログ・公式ヘルプ・口コミから集約）

- **アクセス推移**: 月別 PV、ユニーク数、デバイス別（PC / SP）構成比
- **予約数推移**: ネット予約件数、コール数、合計予約数、月別グラフ
- **流入フリーワード ランキング Top N**: 店舗単位のローカルランキング
- **予約ニーズ分析**: 利用シーン（宴会／デート／接待 等）、人数帯別、時間帯別
- **客数分析**: 新規 / リピート、年代、性別、エリア
- **頻出キーワード分析**（自店ページ内）: ページ内コンテンツ最適化用
- **エリア平均との比較**: 同エリア・同ジャンル・同プラン店舗の平均値と自店の差分
- **競合店比較（一部レポート）**: 営業判断で含まれる場合あり
- **施策提案 / アクションプラン**: 営業からの定性的コメント

### 4.4 自動取得不可・人手依存の理由

- 配布チャネルが営業担当の**メール**または**対面**で、Web API / 店舗ポータル DL の口が存在しない。
- フォーマットは PDF / PowerPoint / Excel 混在。OCR コストが高い。
- **店舗ごと・契約プランごとにレポート粒度が異なる**ため、テンプレ化が困難。
- データダッシュボード化したい場合、現実解は「営業担当に **Excel 版（数値表が生のもの）** を依頼」→ ダッシュボード側 ETL に乗せる。

### 4.5 「分析カルテ」との関係

- 公式ヘルプの位置づけでは「分析カルテ」は通常のレポートよりさらに細かい分析データをまとめた成果物。
- 業務サポートパックの月次分析レポートと**ほぼ同義**で運用されているケースが多い（代理店記事ベース、信頼度: 中）。

## 5. 既知の制約・自動化への留意点

### 5.1 認証・セキュリティ

- HPG 管理画面、RB ともに **2 段階認証 / ワンタイムパスワード**を導入済（リクルート ID 全体方針）。
  - 単純な ID/PW 自動ログインの RPA はメール / SMS OTP で詰まる。
  - メール受信 → コード入力の自動化は ToS と社内ポリシーで要承認。
- パスワード忘れ時の「認証キー」は**メール送付・有効期限 30 分**。

### 5.2 ToS / 法務観点

- リクルートグループの利用規約は**機械的データ取得（スクレイピング）を原則禁止**する条項を含む可能性が高い。
- 実運用するなら、**自店舗のデータを店舗自身がエクスポート目的で取得する**位置付けで、書面で同意取得しておくのが安全。

### 5.3 CSV 直接 DL の現実的網羅範囲

| カテゴリ | RB | HPG | マンスリー |
|---|---|---|---|
| 予約 1 件単位 | ◎ CSV | △ RB 連携前提 | × |
| PV / コール / ネット予約数（月次集計値） | △（予約は明示、PV は不可） | △ 画面表示のみ確認 | ◯ PDF / Excel |
| 流入フリーワード | × | △ 画面表示のみ確認 | ◎ |
| クチコミ | × | △ 画面表示のみ確認 | △ サマリのみ |
| エリア平均比較 | × | × | ◎ |
| ページ別タップ | × | △ 画面表示のみ確認 | ◯ |
| お気に入り（保存）数 | × | △ 画面表示のみ確認 | ◯ |

### 5.4 マーケダッシュボード Phase 1 への推奨アーキ

1. **RB 予約 CSV 取得（半自動）**:
   - 店舗担当がブラウザで月初 1 回 DL → 共有ドライブへ置く
   - Cloud Functions が DL を検知して BigQuery へロード
   - 上限 10,000 件 / 期間制限を考慮した分割 DL ロジック
2. **HPG 集客指標（PV／コール／FW）**:
   - 短期: マンスリーレポート PDF を OCR or 手入力で月次値だけ Sheets に投入
   - 中期: 店舗側 PC に Chrome Extension を入れ、人手ログイン → 拡張がスクリーンスクレイプ → API へ POST する半自動方式
3. **クチコミ**:
   - 件数 / 評点はマンスリーレポート由来で月次差分管理
   - 個別本文は分析対象が決まるまでスコープ外
4. **ETL タイミング**:
   - 月次集計は月の 5 〜 10 日（マンスリーレポート到着後）に締め
   - 日次系（予約数）は RB 連携 or 日次 DL ジョブ

### 5.5 既知の落とし穴メモ

- RB の CSV 出力ボタンは「予約リスト」「スケジュール」の**両方に存在**するが、抽出ロジックは別。スケジュール側は時間軸主体で見え方が異なる。
- 「他店舗系メニュー」から CSV を出すと**他店舗の予約**が含まれる。本部運用時に要注意。
- 「複数コース予約」は CSV では **1 コース分のみ表示**。合計売上の二重計上 / 漏れに注意。
- HPG 管理画面のセッションは**短時間でタイムアウト**する。長時間スクリプトでは要再ログイン。
- マンスリーレポートのフォーマットは**営業の作成タイミングで変動**するため、PDF パーサのフォーマット依存が高い。
