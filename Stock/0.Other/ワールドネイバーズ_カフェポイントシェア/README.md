# ワールドネイバーズ カフェポイントシェア（アイディアメモ）

シェアハウス「ワールドネイバーズ」の住民に毎月付与されるカフェポイントを、住民同士でシェアできるようにするWebアプリ構想。

## 背景

- 住んでいるシェアハウス「ワールドネイバーズ」では、全住民に毎月 **15,000円分のカフェポイント** が付与される
- このポイントはシェアハウス1階のカフェで使える
- 決済フロー: Webアプリを開く → 金額入力 → OK → 決済完了（店員に完了画面を見せる）
- 店員は誰が注文したか把握しており、システム側でもユーザーが紐付いている様子
- ポイントは **月末失効**。余らせる人がいる一方で使い切る人もいる → 相互にシェアできる仕組みが欲しい

## 目的

- 失効してしまうポイントを住民同士でやり取りできるようにし、無駄を減らす

## アイディア（素朴案）

- 提供側の住民のカフェ決済アプリの **ID/パスワードを保存** しておき、どこかのPC上で **RPA的に決済処理** を起動
- 受け取り側（実際にカフェで注文する人）は **別UI** からポイントのやり取りを行い、そのUIから決済動作をトリガー
- 店員から見ると「目の前のその人が決済処理をした」ように見える
- 画面については **スクリーンショットを送る** 仕組みがあればよい

## ゴール（暫定）

- 住民同士でポイントを融通できる状態（UIから他人のポイントで決済が完了する）
