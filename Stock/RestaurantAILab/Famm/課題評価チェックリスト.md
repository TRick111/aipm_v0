# AI活用講座 課題評価チェックリスト

---

## 評価の観点

このチェックリストは、受講生の課題提出物を評価する際の観点を整理したものです。各項目について「できている」「改善の余地あり」「未実施」などで評価し、具体的なフィードバックを行います。

---

## 1. 基本的なプロンプト設計

### 1.1 役割付与（Role Assignment）
- [ ] プロンプトの冒頭で AI に明確な役割を与えている
- [ ] 役割が具体的で、タスクに適している
- [ ] 複数の役割を使い分けている（該当する場合）

### 1.2 背景情報の提供
- [ ] タスクに取り組む前に、背景や目的を AI に伝えている
- [ ] 必要な制約条件（文字数、トーン、ターゲットなど）を明示している
- [ ] コンテキスト情報が十分に提供されている

### 1.3 指示の具体性
- [ ] 曖昧な表現を避け、具体的な指示を出している
- [ ] 期待するアウトプットの形式を明示している
- [ ] 優先順位や重要度を伝えている

---

## 2. タスク分解と段階的実行

### 2.1 タスクの分割
- [ ] 大きなタスクを適切に小さく分解している
- [ ] 各タスクの目的が明確である
- [ ] タスク間の依存関係を理解している

### 2.2 段階的な進行
- [ ] いきなり最終成果物を作らず、段階的に進めている
- [ ] 中間生成物を作成している（例：構成案→本文、設定→画像）
- [ ] 各段階で確認・調整を行っている

### 2.3 情報の統合
- [ ] 複数のタスクで得た情報を適切に統合している
- [ ] 統合時に情報の損失を防いでいる（要約しすぎない）
- [ ] 最終アウトプットに向けて一貫性を保っている

---

## 3. AI との対話・改善サイクル

### 3.1 質問の活用
- [ ] AI に質問させて要件を詰めている
- [ ] 自分でも AI に質問して情報を深掘りしている
- [ ] 対話を通じて設定やコンセプトを固めている

### 3.2 フィードバックと修正
- [ ] AI のアウトプットに対して具体的なフィードバックを返している
- [ ] 「違う」だけでなく、どこがどう違うかを言語化している
- [ ] 納得いくまで調整を繰り返している

### 3.3 AI による自己評価
- [ ] AI に自身のアウトプットを評価させている
- [ ] 評価結果を次の改善に活かしている
- [ ] 客観的な視点を取り入れている

---

## 4. プロンプトの再利用性・効率化

### 4.1 テンプレート化
- [ ] 繰り返し使えるプロンプトをテンプレート化している
- [ ] 汎用プロンプトを作成している
- [ ] 変数部分と固定部分を区別している

### 4.2 アウトプットの構造化
- [ ] アウトプットをブロック化して使いやすくしている
- [ ] 再利用しやすい形式で保存している
- [ ] 中間生成物を整理して保管している

### 4.3 ワークフローの確立
- [ ] 自分なりの進め方の型ができている
- [ ] 効率的な手順を意識している
- [ ] 実案件を想定した動きができている

---

## 5. 画像生成（キャラクター・ビジュアル）

### 5.1 設定の深掘り
- [ ] 画像生成前にキャラクター設定を十分に深掘りしている
- [ ] 設定を固めてから画像を生成している
- [ ] AI と対話しながら設定を詰めている

### 5.2 外見的特徴の詳細化
- [ ] 以下の要素を具体的に指定している：
  - [ ] 目つき（形・サイズ・黒目比率）
  - [ ] 体型・体つき（細め/普通/ふくよか）
  - [ ] 尻尾（長さ・太さ・先端形状）※該当する場合
  - [ ] 耳（大きさ・角度・形状）※該当する場合
  - [ ] 顔の大きさ・比率
  - [ ] 手足の長さ・大きさ
  - [ ] 毛並み（短/中/長・光沢の有無）※該当する場合
  - [ ] アクセサリー（首輪、装飾品など）
  - [ ] 配色（カラーコード指定）
  - [ ] その他の特徴的要素

### 5.3 ビジュアルの一貫性
- [ ] 同一キャラクターが複数枚で同じに見える
- [ ] ポーズや表情が変わっても特徴が保たれている
- [ ] ブレが生じた際に仕様を追記・修正している

### 5.4 画像生成の指示
- [ ] 構図や要素を具体的に指示している
- [ ] 光源、配置、背景なども指定している
- [ ] 雰囲気だけでなく見た目で分かる要素を指定している

### 5.5 バリエーションとポーズ
- [ ] 複数のポーズやアングルを試している
- [ ] 劇的に違うポーズで幅を出している
- [ ] 用途に応じたバリエーションを作成している

### 5.6 カラーコードの正確性
- [ ] 指定したカラーコードが実際の画像に反映されているか確認している
- [ ] 色味のズレに気づき、修正している

## 6. テキスト生成（記事・コピー・メルマガなど）

### 6.1 ターゲットの明確化
- [ ] ターゲット層を具体的に設定している
- [ ] ターゲットの特性や行動を深掘りしている
- [ ] ターゲットに刺さる表現を意識している

### 6.2 構成の設計
- [ ] いきなり本文を書かず、構成案を先に作成している
- [ ] タイトルや見出しを先に決めている
- [ ] 全体の流れを意識している

### 6.3 文字数の管理
- [ ] AI の文字数カウントを鵜呑みにせず、自分で確認している
- [ ] Web の文字数カウントツールを使用している
- [ ] 指定文字数に対して適切な長さになっている

### 6.4 複数案の生成と選択
- [ ] 複数のアイデアや案を AI に出させている
- [ ] その中から最適なものを選んでいる
- [ ] 選択の理由を明確にしている

### 6.5 独自性の反映
- [ ] AI の出力をそのまま使わず、自分の色を出している
- [ ] テーマや場面、トーンなどで独自性を加えている
- [ ] 誰が作っても同じにならないよう工夫している

## 評価の観点別スコアリング例

各セクションを 3〜5 点で評価する場合の目安：

### 基本的なプロンプト設計（3 点満点）
- 3 点：役割付与、背景情報、具体的指示が全て適切
- 2 点：基本はできているが、一部に改善の余地
- 1 点：基本的な要素が不足している

### タスク分解（2〜3 点満点）
- 3 点：適切に分解し、段階的に進め、統合も完璧
- 2 点：分解はできているが、統合や一貫性に課題
- 1 点：タスク分解が不十分

### AI の自己フィードバック（1〜3 点満点）
- 3 点：積極的に活用し、改善サイクルを回している
- 2 点：部分的に活用している
- 1 点：活用していない、または形式的

### 工夫・独自性（2〜3 点満点）
- 3 点：独自の工夫が多く、高い応用力
- 2 点：基本的な工夫はあるが、独自性は限定的
- 1 点：講義内容の再現のみ

## フィードバック作成時のポイント

### 良かった点を具体的に
- 何ができていたかを明確に指摘
- 講義内容のどの部分を実践できているか
- 特に優れている点を強調

### 改善点は建設的に
- 「できていない」ではなく「次はこうすると良い」
- 具体的な改善方法を提示
- 小さな一歩から始められるアドバイス

### 成長を認める
- 前回からの成長を指摘
- 習熟度の高さを評価
- 今後の可能性を示唆

### 実務への接続
- 実案件でどう使えるかを示す
- 再現性や効率性の観点を加える
- プロフェッショナルとしての視点を提供
