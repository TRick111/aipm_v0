# マーケダッシュボード

## 背景

- Chef's Room の CMO／CFO プランを **外注化** する計画。外注コンサルに媒体ID/PWを渡さずに、月次でクライアントの集客状況を分析・提案できる状態が必要
- 現状は各媒体（食べログ／食べログノート／ホットペッパー／一休／オズモール／Retty／ぐるなび）に手動ログインして数字を集計しており、人手依存・スケール不可
- アウトプットは竹矢さんが既存週報・月報の流れに合わせて **スライド形式** を希望

## 目的

- 既存ダッシュボード（売上分析・PL）に **マーケビュー** を増築し、契約マーケ媒体の月次KPIを一元管理できるシステムを作る
- その上で、外注コンサル向けの新規ロールで、ID/PWを隠したまま月次レポートをスライド形式で納品できるようにする

## ゴール（完了条件）

- 7 媒体（食べログ／食べログノート／ホットペッパー／一休／オズモール／Retty／ぐるなび）の月次主要KPIがダッシュボードで一元管理できる
- 外注コンサル用ロールで媒体ID/PW が一切閲覧できない
- スライド形式の月次レポートが指定店舗・指定月に対してダウンロードできる
- CSV ダウンロードの自動取得（Playwright）が段階的に対応されている
- 麻布しき旗の台・本店／BFA 等の対象店舗で運用が開始されている

## 開発運用ルール

- **実装は必ず新しい Feature ブランチを切って行う**（main / develop への直接コミット禁止）
- 対象リポジトリ: `RestaurantAILab/Dashboard`（ローカル: `~/RestaurantAILab/Dashboard`）
- ブランチ命名: `feature/marke-dash-BL-XXXX-<短い説明>`（例: `feature/marke-dash-BL-0098-prisma-schema`）
- 複数 BL をまとめる場合: `feature/marke-dash-<scope>` 形式可
- 完了時は PR を作成し、レビュー後 main にマージ

## 制約事項

- **HP マンスリークライアントレポートは取得不可**（2026-06-25 確定）。要件定義書 §6.3 の `マンスリークライアントレポート` を出典とする項目は対象外とする:
  - 月次PV、予約経路内訳（電話／ネット）、流入経路（フリーワード等）
  - 取得可能な項目: クチコミ件数・平均点（管理画面）、レストランボード経由の予約集計（予約数集計、リピート率、月次推移、コース別等）
