## 0. この文書の位置づけ

`要件定義書_v1.md`（Phase 1 全体スコープ）から、**最初に着手して 2026-07 初旬リリースを目指す範囲**を切り出した実装定義書。要件定義 v1 の B 案段階リリース戦略における **Phase 1.0** に相当する。

要件定義 v1 から本書で確定した主な縮減・追加:

| 区分 | 内容 | 根拠 |
|---|---|---|
| 縮減 | 対象媒体を **食べログ＋食べログノート＋ホットペッパー** の 3 媒体に限定（一休／オズモール／Retty／ぐるなび を Phase 1.1 以降に後送り） | 受領モックが食べ+HPG のみの構成で作られており、初期実装ではこの 3 媒体に絞り 7 月初旬リリースを優先 |
| 縮減 | O-01 HTML スライド月次レポート出力を **後送り**（Phase 1.1 以降） | PO 判断（現時点で不要） |
| 縮減 | S-01／S-02 外注コンサルロール／ACL を **後送り**（Phase 1.1 以降） | PO 判断（初期実装では社内ロールのみ） |
| 統合 | 食べログと食べログノートを **単一の媒体タブ「食べログ／ノート」に統合**（タブ分割しない） | PO 判断（媒体管理画面では別だが、ダッシュボード上は同じ食べログ系として並べる） |
| 追加 | M-08 媒体別アップロード画面（要件 v1 と同じ要件） | 初期実装で必須 |
| 追加 | X-01 店舗ごと契約媒体 ON/OFF 管理画面 | 初期実装で必須 |

要件 v1 で定義されていた M-14（CSV 自動取得ローカルスキル）／M-14a〜e は **Phase 1.1 以降に後送り**。初期実装は **手動アップロードのみで運用** する。

## 1. ゴール

> **「Chef's Room の社内担当者が、食べログ／食べログノート／ホットペッパーの月次データを手動アップロードして、媒体横断 KPI と媒体別深掘りを 1 画面で確認できる」状態を 2026-07 初旬までに実現する**

外注コンサル向けの UI／ACL は本実装では含めない。コンサル展開は Phase 1.1 以降で対応する。

## 2. 対象範囲

### 2.1 対象システム

`RestaurantAILab/Dashboard`（既存）に **マーケビュー** を追加。

### 2.2 対象媒体（Phase 1 初期実装）

| # | 媒体 | 取込方式 | 備考 |
|---|---|---|---|
| 1 | 食べログ | 手動 CSV/PDF アップロード | アクセス解析／ページ別／来店指標／クチコミ |
| 2 | 食べログノート | 手動 CSV アップロード | 予約／フラット来店／キャンセル／卓稼働率／予約コース |
| 3 | ホットペッパー | 手動 CSV/Excel/PDF アップロード | マンスリークライアントレポート＋レストランボード予約一覧 |

要件 v1 と同じ。麻布しき旗の台・本店、BFA、韓国屋台ポックンパ ネネチキン 東岡崎店（モック準拠）など、Chef's Room と協議の上で順次追加。

- POS 連携・PL 管理（既存機能）

## 3. 機能要件

### 3.1 ダッシュボード画面（5 画面構成）

モック仕様書 v1 §1 の 5 画面構成を踏襲。レイアウト・KPI タイル・スパークライン仕様はモック仕様書 v1 §2〜§7 に準拠する。

#### D-01 月次サマリ（トップ）

| 項目 | 受入基準 |
|---|---|
| KPI タイル | モック仕様書 §3.2 の 8 タイル（総組数／総客数／平均人数／媒体予約比率／ウォークイン組数／食べログ予約 CVR／HPG 予約 CVR／エリア順位）を表示。各タイルに 12 ヶ月スパークライン＋前月比を併記 |
| 媒体別予約構成 12ヶ月推移 | 積み上げ面グラフ（食べログ／HPG／自社直／ウォークイン）。**食べログノートからの予約は「食べログ」帯に合算** |
| エッジケース表示 | データ整合性エラーバナー／HPG クライアントレポート未取得バナーをモック仕様書 §2.6 に準拠して表示 |

#### D-02 集客・予約ファネル

| 項目 | 受入基準 |
|---|---|
| タブ | 「食べログ／ノート」「HPG」の 2 タブ |
| 食べログ／ノートタブ | 食べログ全体 PV → ネット予約組数 → 来店組数のファネル。**来店組数段は「Phase 1.1 で実装予定」のグレーアウト** |
| HPG タブ | HPG 全体 PV → ネット予約組数 → 来店組数のファネル |
| 補助情報パネル | 直近 12ヶ月推移＋ CV 内訳（食べログのみ通話成立／地図印刷 PV） |

#### D-03 媒体別深掘り

| 項目 | 受入基準 |
|---|---|
| タブ | 「食べログ／ノート」「HPG」の 2 タブ |
| 食べログ／ノートタブ | 上段に **食べログ系 KPI 5 タイル**（全体 PV／予約組数／CVR／クチコミ件数／平均評価）。下段に **食べログノート系 KPI ブロック**（月次予約件数／フラット来店件数／予約 vs フラット 比率／キャンセル件数・キャンセル率／卓稼働率）。さらにページ種別別 PV 12ヶ月推移（折れ線多系列）＋当月ページ別 PV ランキング表 |
| HPG タブ | KPI 5 タイル＋ PV／予約組数／CVR の 12ヶ月推移ミニチャート 3 つ＋クーポン・集客・組人数の HPG 専用ボックス |
| 食べログ／ノート統合方針 | **両媒体をタブで分けない**。食べログ系 KPI（PV／CVR／ランキング系）と食べログノート系 KPI（予約運用系）を縦に並べて 1 つの「食べログ／ノート」タブ内で表示する |

#### D-04 クチコミ

モック仕様書 §6 に準拠。食べログ系（来店月ベース）＋ HPG（投稿日ベース）の混在表示。食べログノートはクチコミデータ対象外。

#### D-05 競合・SEO

モック仕様書 §7 に準拠。食べログのエリア順位推移＋ TOP10 ＋ HPG 流入ワード TOP20。

#### 共通 UI

| 項目 | 受入基準 |
|---|---|
| ヘッダ | 店舗切替セレクタ／月セレクタ／データ取得日表示（モック仕様書 §2.1） |
| サイドナビ | 5 画面＋設定画面（モック仕様書 §2.2） |
| 媒体カラー | 食べログ＝オレンジ（#FF7F50）、HPG ＝レッド（#E60012）（モック仕様書 §2.5） |

### 3.2 媒体別アップロード画面（M-08）

| 項目 | 受入基準 |
|---|---|
| 画面構造 | 媒体別タブ（食べログ／食べログノート／ホットペッパー）。タブごとにアップロード可能なファイル形式と KPI マッピングを明示するガイド領域 |
| ファイル形式 | 媒体・レポート種別ごとに受け付ける形式（CSV／Excel／PDF）と必要カラムを明記。違反時はエラーで弾く |
| プレビュー | アップロード前にファイル先頭〜10 行をプレビュー表示。文字コード自動判定（UTF-8／Shift-JIS／CP932） |
| 取込履歴 | 店舗 × 媒体 × 月の取込ログを直近 24 ヶ月分一覧表示。状態（成功／失敗／処理中）＋取込日時＋取込者を表示 |
| 再取込 | 既存月分の上書き再取込が可能。再取込時は履歴に新規行を追加し、旧データはアーカイブ |
| 受信ファイル | Vercel Blob に原本保存（要件 v1 §8 `MarketingMonthlyReport.sourceBlobUrl` 準拠） |
| 権限 | 既存の admin ／店舗ユーザーロールのみアクセス可。コンサルロールは存在しない |

**Phase 1 初期実装での媒体別必要レポート（最低限）**:

| 媒体 | レポート | 形式 | 必須／任意 |
|---|---|---|---|
| 食べログ | 月間アクセスレポート（合計） | CSV | 必須 |
| 食べログ | ページ別アクセスレポート | CSV | 必須 |
| 食べログ | アクセス数ランキング（エリア順位） | CSV | 必須 |
| 食べログ | 来店指標（CV レポート） | CSV | 050 番号契約店舗のみ必須 |
| 食べログ | クチコミ一覧 | CSV | 必須 |
| 食べログノート | 予約一覧 | CSV | 必須 |
| 食べログノート | 分析 総客数 | CSV | 必須 |
| 食べログノート | 分析 予約コース | CSV | 必須 |
| ホットペッパー | マンスリークライアントレポート | Excel／PDF | 必須 |
| ホットペッパー | レストランボード予約一覧 | CSV | 必須 |
| ホットペッパー | アクセスレポート（日別／月別） | Excel | 任意 |
| ホットペッパー | 流入ワード | CSV | 任意 |
| ホットペッパー | クチコミ一覧 | CSV | 必須 |

サンプル CSV は `sample_data/` 配下に格納済み。これをパーサ実装の正とする。

### 3.3 店舗ごと契約媒体 ON/OFF 管理画面（X-01）

| 項目 | 受入基準 |
|---|---|
| 画面位置 | 設定（admin 専用） |
| 操作 | 店舗ごとに 3 媒体（食べログ／食べログノート／HPG）の契約状況をチェックボックス管理 |
| UI 反映 | OFF にした媒体のタブ／ファネル／媒体別ビューを当該店舗の UI から非表示にする |
| アップロード画面の連動 | OFF 媒体のタブもアップロード画面から非表示 |
| データモデル | `StoreSettings.featureFlags.marketingChannels`（要件 v1 §8 を踏襲、Phase 1 初期実装では 3 媒体分のみ） |
| 将来拡張 | Phase 1.1 以降に媒体（一休／オズ／Retty／ぐるなび）を追加する際、本画面に媒体行を追加するだけで対応可能 |

### 3.4 権限

| ロール | 用途 | 本実装でのアクセス範囲 |
|---|---|---|
| admin（社内） | 全店舗・全機能 | 全 5 ダッシュボード画面／M-08 アップロード／X-01 媒体 ON/OFF |
| 店舗ユーザー（既存） | 自店舗のみ | 自店舗の 5 ダッシュボード画面（OFF 媒体は非表示）／M-08（自店舗の媒体のみ） |

外注コンサルロール（要件 v1 S-01／S-02）は Phase 1.1 以降。

### 3.5 運用フロー定義書（X-02）

「誰が、いつ、どの媒体を、どの形式でアップロードするか」を運用ドキュメント化する。3 媒体分の月次運用フロー（取得元 → ファイル形式 → アップロード手順 → 整合性チェック）を `docs/運用フロー_v1.md` として作成。

## 5. データモデル概要

要件 v1 §8 を踏襲。本実装では以下のサブセットのみ使う:

| エンティティ | 説明 | 本実装での使用 |
|---|---|---|
| `MarketingChannel` | 媒体マスタ | 3 行のみ（tabelog／tabelog_note／hpg） |
| `MarketingMonthlyReport` | 店舗×媒体×月の集計値 | 3 媒体分 |
| `MarketingImportLog` | 取り込み履歴 | M-08 アップロード履歴で使用 |
| `StoreSettings.featureFlags.marketingChannels` | 契約媒体フラグ | X-01 で使用（3 媒体分） |
| `StoreGroup`（拡張） | コンサル用グループ | **本実装では拡張しない**（既存のまま） |

`MarketingCredentials` テーブル：本実装でも作成しない（M-14 後送りのため）。
