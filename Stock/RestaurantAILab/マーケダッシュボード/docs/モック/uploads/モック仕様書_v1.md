## 0. このドキュメントの目的

要件定義書 v1・データ取得設計書 v1.2 で確定した「**何を出すか**」を、**「どう見えるか」「どんな値が並ぶか」**まで具体化する。デザイナー / 実装者がモック（クリッカブル試作）を作るための一次資料。

**ゴール**: このドキュメントだけ読めば、Figma に画面を起こせる。サンプル値もすべて明記。

## 2. 共通 UI 要素

### 2.1 ヘッダ（全画面共通）

```
┌────────────────────────────────────────────────────────────────────────────────┐
│ [RAL Logo]  マーケダッシュボード                                                  │
│   店舗: [韓国屋台ポックンパ ネネチキン 東岡崎店 ▾]   期間: [2026-05 ▾]            │
│                                                          データ取得日: 2026-06-03 03:14 │
└────────────────────────────────────────────────────────────────────────────────┘
```

| 要素 | 仕様 | モック値 |
|---|---|---|
| ロゴ + プロダクト名 | 左上 | 「マーケダッシュボード」 |
| 店舗切替プルダウン | 担当店舗のみ表示。外注ロールではID/PW非表示 | 「韓国屋台ポックンパ ネネチキン 東岡崎店」「BAR FUSION ASOBI（BFA）」「麻布しき」等 |
| 期間ピッカー | `YYYY-MM` プルダウン。直近24ヶ月選択可。デフォルト = 前月 | デフォルト `2026-05` |
| データ取得日 | 右端小さく | `2026-06-03 03:14` |

### 2.3 KPI タイル共通仕様

```
┌────────────────────────┐
│ KPI 名 ⓘ              │ ← ⓘ ホバーで定義式ツールチップ
│                        │
│ 312 組  ↗ +8%          │ ← 大きい数字 / 単位 / 前月比（▲ or ↗ + 色）
│                        │
│ ▁▂▄▃▆▅▇█             │ ← 12ヶ月スパークライン（micro chart）
└────────────────────────┘
```

| 要素 | 仕様 |
|---|---|
| KPI 名 | 14px、グレー |
| 大きい数字 | 32px、太字、黒 |
| 単位 | 数字の右、12px、グレー |
| 前月比 | `↗ +8%`（緑）/ `↘ -5%`（赤）/ `→ 0%`（グレー） |
| 12ヶ月スパークライン | 高さ 24px、塗りつぶし面グラフ、色は媒体色 or グレー |
| ツールチップ | KPI 名右の `ⓘ` にホバーで定義式表示 |

### 3.2 KPI タイル 8 個（モック値）

## 10. モック制作の進め方（推奨）

### Step 1: ワイヤフレーム（30分〜1時間）

- §3〜§7 のレイアウト ASCII を Figma のフレーム配置に起こす
- グレースケールでよい
- レスポンシブは PC 横置き（1280px）想定で十分（運用想定）

### Step 2: モックデータを差し込む（1時間）

- §9 のシード値を各タイル / グラフ / テーブルに流し込む
- グラフは Figma プラグイン or 静的画像で OK

### Step 3: 媒体カラー・KPI 強調（30分）

- §2.5 媒体カラー適用
- 数値の前月比 ↗ / ↘ の色（緑/赤）
- 整合性エラーバナーは設計だけ（hidden 状態でOK）

### Step 4: クリッカブル接続（任意、30分）

- ヘッダ期間ピッカー / 店舗切替は1択でOK
- サイドナビ 5 リンクを各フレームに接続
- タブ切替（食べログ ↔ HPG）も接続

### Step 5: レビュー観点

- KPI の数値が異常に見えないか（モック値の桁・単位確認）
- 媒体カラーが他画面と統一されているか
- 注釈ツールチップが必要箇所すべてに付いているか
- §2.6 状態・エッジケース のうち少なくとも「整合性エラーバナー」「データ欠損」の2状態のフレームを用意

### 成果物

- Figma ファイル 1 つ、フレーム 5〜7 個（5 画面 + 状態バリエーション 2）
- ナビゲーション接続済（クリッカブル）
- README で「これは何を示すモックか / 数値は実機サンプル + 一部モック推定」を明記
