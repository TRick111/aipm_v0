# ホットペッパーグルメ／レストランボード — 予約一覧 ダウンロード SOP

最終更新日: 2026-06-29
店舗: 韓国屋台ポックンパ ネネチキン 東岡崎店
対応 Dashboard reportType: `hpg_restaurant_board`（`lib/marketing/reportTypes.ts`）
パーサ実装の正: `lib/marketing/parsers/hpg.ts` の `parseHpgRestaurantBoard`

> **位置付け**: 「食べログ／HPG／一休／ぐるなび／自社サイト等の予約を 1 ファイルに統合できる唯一の経路」（`予約経路` カラムで媒体識別）。Phase 1 マーケダッシュボードの予約データ統合の主軸。

## 1. ログイン URL と必要認証情報

| 項目 | 内容 |
|---|---|
| ログイン URL | https://restaurant-board.com/（または HPG リクルート ID 経由でリンク） |
| 認証方式 | リクルート ID（HPG クライアント管理画面と同系統。要実機確認） |
| 2 要素認証 | **2 要素認証 / ワンタイムパスワード導入済**（リクルート ID 全体方針）。SMS or メール OTP |
| ID / PW の保管 | **要 user 確認**（HPG / RB で同一アカウントか別アカウントか含めて確認必要） |
| セッション保持期間 | **短時間タイムアウト**（リクルート系。具体的な TTL は公式未記載） |
| アカウントロック | 連続失敗で発生想定 |
| 端末制限 | **PC ブラウザ限定**（iPad / iPhone アプリでは CSV 出力不可） |

## 2. ダウンロード手順（手動）— 2 段階エクスポート

1. ブラウザで https://restaurant-board.com/ にアクセス（または HPG 管理画面経由）し、PC ブラウザでログイン
2. 左サイドメニュー「予約台帳」→「予約リスト」を開く
   - URL: `https://restaurant-board.com/CLP/view/reserveListWeb/`
3. 画面右上の **「CSV 出力」ボタン** をクリック
4. ダイアログで対象期間の **開始日／終了日** をカレンダーから選択 → 「絞り込む」
   - 例: 2025-07 全月なら 2025/07/01 〜 2025/07/31
5. 複数店舗の権限がある場合は対象店舗（韓国屋台ポックンパ ネネチキン 東岡崎店）を選択
6. 「CSV ファイルを出力する」 → 確認画面で「作成する」を押下
   - **この時点ではダウンロードは始まらない**。非同期キューに投入される
7. 数分待機（30 秒〜数分）
8. 左上「メッセージ」アイコン or `/CLP/view/message/` をクリック
9. 完成通知が届いたら「ダウンロード」ボタンで CSV 取得
10. ファイル名（既定）: `reserve_{開始日YYYYMMDD}-{終了日YYYYMMDD}_{生成タイムスタンプ}{ハッシュ}.csv`
    - 実機サンプル: `reserve_20260501-20260531_20260626215214W06Hr.csv`
11. 保存後、後述§6 の命名規則に従ってリネーム

## 3. 期間指定の限界

| 項目 | 仕様 |
|---|---|
| 過去の遡及範囲 | **公式に明示なし**。一般的には RB に登録された全期間（=店舗の RB 契約開始以降）が DL 可能と想定 |
| 1 リクエストあたり最大件数 | **10,000 件上限**（FAQ 明記）。超過時は期間を分割 |
| 1 ヶ月分の典型件数 | サンプル店舗（May 2026）は 0 件。ポックンパは要実測。100 席規模店舗なら 1 ヶ月で数百〜数千件想定 |
| 月またぎ境界 | カレンダー指定の日付ベース（0:00〜24:00） |
| 月確定タイミング | RB は予約管理 SaaS のため「月確定」概念はない。**ステータス変更や手動編集が後から入る可能性あり**（過去月の再取得で値が変わるリスク） |
| 非同期処理時間 | 期間・件数依存。数十秒〜数分 |

## 4. 出力ファイルの仕様

| 項目 | 仕様 |
|---|---|
| 文字コード | **Shift_JIS / CP932**（実機サンプルで確認） |
| 改行 | CRLF |
| 区切り | カンマ |
| 拡張子 | `.csv` |
| 行粒度 | 1 行 = 1 予約 |
| 列数 | **22 列**（実機サンプルで確認） |

### 必須カラム（`parseHpgRestaurantBoard` が参照）

パーサが必須トークンとして検出: `予約受付日`, `来店日`

パーサが読むカラム:

```
予約受付日, ステータス, 予約経路, 来店日, 開始時間, 終了時間,
人数, ネット予約フラグ, テーブル, コース名, コース金額
```

### 全 22 列（実機サンプル確認）

```
予約受付日, ステータス, 予約経路, AIR店舗ID, 店舗コード, 店舗名, 来店日,
開始時間, 終了時間, コース名, コース金額, 利用ポイント, 人数, ネット予約フラグ,
テーブル, 予約依頼番号, 発行ポイント, 要望, メモ, キーワード,
スマート支払い利用有無, 決済確定金額
```

- `予約経路` 値の例: `ホットペッパーグルメ` / `電話` / `自社サイト` / `食べログ` 等
- `ネット予約フラグ`: パーサが `1` / `true` / `はい` / `ネット` を `true` と判定
- `AIR店舗ID` + `店舗コード` で多店舗運用時の名寄せ可能

## 5. 12 ヶ月遡及取得の作業見積もり

| 項目 | 見積もり |
|---|---|
| 1 ヶ月あたりの操作 | 期間指定 → 非同期キュー投入 → ポーリング待機（30秒〜数分）→ メッセージから DL → リネーム → 配置 |
| 1 ヶ月あたりの所要時間 | **約 5 分**（非同期待機が支配的。同時に他作業も可） |
| 12 ヶ月分の総工数 | **約 60 分**（実時間。非同期待機を並列化できれば短縮可能） |
| 一括取得の可否 | **可（条件付き）**: 期間指定で **12 ヶ月分を 1 ファイル一括 DL 可能**（10,000 件上限を超えなければ）。**ポックンパが 1 年で 10,000 件を超えるなら月別分割必須** |
| 推奨取得方針 | **まず 12 ヶ月一括で試行**（2025/07/01 〜 2026/06/30）。10,000 件超過 or タイムアウトなら月別 12 回に分割 |

### 一括取得時の所要工数（推奨）

- 12 ヶ月一括 1 ファイルなら **約 10 分**（非同期待機長め + DL + リネーム）

## 7. 既知の落とし穴

| # | 内容 | 対処 |
|---|---|---|
| 1 | **10,000 件上限**を超過すると CSV 生成が失敗（or 切り捨て） | 期間を分割。月別 1 ファイルに切り替え |
| 2 | **PC 版限定**（iPad / iPhone アプリは不可） | PC ブラウザで作業 |
| 3 | **複数コースが同一予約に紐づく場合、1 コース分のみ表示** | 売上集計の二重計上／漏れに注意 |
| 4 | 「他店舗系メニュー」から CSV を出すと他店舗の予約も含まれる | 必ず対象店舗を選択する |
| 5 | セッション短時間タイムアウト | 長時間スクリプトでは再ログイン必要 |
| 6 | **2 要素認証**（OTP）が毎回 or 端末ごとに要求される可能性 | 自動化時は OTP 連携が課題（メール受信→入力など） |
| 7 | 過去月再取得時、ステータス変更・手動編集で値が変わる可能性 | 取得日（fetched_at）を raw テーブルに記録し履歴を残す |
| 8 | Shift_JIS のため Excel で開いて再保存すると UTF-8 化 | Excel で開かずアップロード |
| 9 | 2024-05 に CSV DL 失敗事象あり（解消済）／2024-10 に時間制限解除 | 過去の RB 障害履歴を FAQ で把握しておく |

## 8. 自動化の見通し

| 項目 | 見通し |
|---|---|
| Playwright 自動化の所要工数 | **1 日**（非同期エクスポート 2 段階のため、ポーリングロジックが必要）。Phase 1.1 でレストランボード fetcher を新規実装 |
| Selector の安定性 | 「CSV 出力」ボタン・期間カレンダー・「メッセージ」ページの完成通知リンクの selector が必要。**RB UI 更新で破綻リスクあり**（定期確認推奨） |
| ログイン状態保持 | `state/storageState.json`。**2 要素認証 OTP の自動化が最大の課題** |
| 自動化の段取り | (1) `/CLP/view/reserveListWeb/` で期間指定 → エクスポート要求 POST、(2) `/CLP/view/message/` をポーリング、(3) 完成リンクから download イベント捕捉 |
| 既存スクレイパとの関係 | 既存 `scripts/scrapers/hotpepper/` は HPG クチコミのみ対応。RB は別ドメイン（`restaurant-board.com`）で別途認証必要 |

### 既存スクレイパでの過去 12 ヶ月遡及可否（脚注）

- RB は既存スクレイパ未対応。Phase 1.1 で `scripts/scrapers/restaurant_board/` を新規実装することになる。
- 既存 HPG スクレイパ（クチコミ）の storageState は別ドメインのため流用不可。**RB 用に独立した storageState 管理が必要**。
