## 1. ログイン URL と必要認証情報

| 項目 | 内容 |
|---|---|
| ログイン URL | https://www.cms.hotpepper.jp/CLN/login/ |
| 認証方式 | リクルート ID（HPG クライアント管理画面） |
| 2 要素認証 | **2 要素認証 / ワンタイムパスワード導入済**（SMS or メール OTP） |
| ID / PW の保管 | **要 user 確認**（既存スクレイパ `scripts/scrapers/hotpepper/login_and_save_state.py` で手動ログイン → storageState 保存の運用が想定済み） |
| ID / PW 再通知 | https://www.cms.hotpepper.jp/CLN/mgr/remind/remindId/ （認証キーはメール送付・有効期限 30 分） |
| セッション保持期間 | **短時間タイムアウト**（リクルート系。長時間スクリプトでは要再ログイン） |
| アカウントロック | 連続失敗で発生想定 |

## 2. ダウンロード手順（手動）

1. ブラウザで https://www.cms.hotpepper.jp/CLN/login/ にアクセスしログイン
2. グローバルメニュー「アクセス分析」 → 「フリーワードランキング」（または「キーワード分析」）へ遷移
3. 直接 URL でも可: `https://www.cms.hotpepper.jp/CLP/crp040/showClientReportFreeword/0/0`
4. 対象月を画面プルダウンで指定（例: 2025-07）
5. 画面内の **CSV ダウンロードボタン**（あるいは「Excel 出力」相当）をクリック
6. ファイル名（既定）: `freeword_YYYYMM_MA.csv`（実機サンプル: `freeword_202606_MA.csv`、`MA` は店舗識別子の一部と推定）
7. 保存後、後述§6 の命名規則に従ってリネーム

> **URL 直叩きの可能性**: `showClientReportFreeword/0/0` の `0/0` 部はパラメータ枠と推測。**月指定や直接 GET で取れるかは要実機確認**。当月固定の可能性もある。

## 4. 出力ファイルの仕様

| 項目 | 仕様 |
|---|---|
| 文字コード | **Shift_JIS / CP932**（実機サンプルで確認） |
| 改行 | CRLF |
| 区切り | カンマ |
| 拡張子 | `.csv` |
| 行粒度 | 1 行 = 1 ワード（順位順） |

### 必須カラム（`parseHpgKeywords` が参照）

パーサが必須トークンとして検出: `順位`, `フリーワード`（先頭 5 行までヘッダ前のタイトル行をスキップ可能）

パーサが読むカラム:

```
順位, フリーワード, 流入割合, 同中エリア
```

- 1 行目: 「貴店への流入フリーワードランキング」（タイトル行・**パース時にスキップ必須**）
- 2 行目: ヘッダ
- 3 行目以降: データ
- `順位` は同率の重複あり（例: 3 位が 2 件並ぶ）
- `流入割合` は `%` 付き文字列（パーサで除去・float 化）
- `同中エリア` は同エリア内同ワード検索件数（推定）

## 7. 既知の落とし穴

| # | 内容 | 対処 |
|---|---|---|
| 1 | 1 行目がタイトル行（「貴店への流入フリーワードランキング」）でヘッダではない | パーサ側で先頭 5 行までスキップして `順位, フリーワード` ヘッダを検出済 |
| 2 | Shift_JIS のため Excel で開いて再保存すると UTF-8 化（文字化けの可能性） | Excel で開かずアップロード |
| 3 | 同率順位の重複行あり | 集計時は順位 + ワードの組み合わせで一意化 |
| 4 | プラン依存で TOP 件数が異なる可能性（基本プランは TOP の一部のみ、ラ S 以上で全件想定） | ポックンパの契約プランを user に確認 |
| 5 | HPG 内検索のみ（Google 流入は含まず） | KPI 説明で「HPG 内検索ワード」と注記 |
| 6 | OTP 2 要素認証で毎回コード入力が発生する可能性 | 自動化時は手作業が必要になる可能性 |
| 7 | URL の `/0/0` 部の仕様が不明 | 直接 GET で別月が取れるか要実機確認 |
