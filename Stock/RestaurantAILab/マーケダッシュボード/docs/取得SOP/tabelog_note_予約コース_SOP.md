# 食べログノート — 予約コース ダウンロード SOP

最終更新日: 2026-06-29
店舗: 韓国屋台ポックンパ ネネチキン 東岡崎店
対応 Dashboard reportType: `tabelog_note_courses`（`lib/marketing/reportTypes.ts`）
パーサ実装の正: `lib/marketing/parsers/tabelogNote.ts` の `parseTabelogNoteCourses`

## 1. ログイン URL と必要認証情報

| 項目 | 内容 |
|---|---|
| ログイン URL | https://owner.tabelog.com/owner_account/login/ |
| 認証方式 | 食べログ店舗会員 ID / パスワード（食べログ本体・食べログノート共通） |
| 2 要素認証 | 公式ヘルプ上では必須化記載なし |
| ID / PW の保管 | **要 user 確認**（1Password 等のチーム共有 vault 想定） |
| セッション保持期間 | Cookie ベース（具体的な TTL は公式記載なし） |
| アカウントロック | 連続失敗で発生想定 |

## 2. ダウンロード手順（手動）

1. ブラウザで https://owner.tabelog.com/owner_account/login/ にアクセスしログイン
2. グローバルナビ／サイドメニューから「食べログノート」を選択
3. 左サイドナビの「分析」 → サブメニュー「予約コース」をクリック
   - URL: `https://owner.tabelog.com/tn/analysis/category/course?yearMonth=YYYY-MM`
   - 例: 2025-07 分なら `?yearMonth=2025-07`
4. 画面上に **TOP20 コース** が表示される（21位以降は画面非表示だが CSV には含まれる）
5. 画面内の「CSV ダウンロード」ボタンをクリック
6. ファイル名（既定）: `YYYYMMDD_分析_予約コース.csv`（実機サンプル: `20260626_分析_予約コース.csv`）
7. 保存後、後述§6 の命名規則に従ってリネーム

## 3. 期間指定の限界

| 項目 | 仕様 |
|---|---|
| 過去の遡及範囲 | `yearMonth=YYYY-MM` で任意月指定可能。**過去保持期間の公式明示なし** |
| 1 リクエストあたり最大期間 | **月単位固定** |
| 表示上限 | 画面は TOP20 まで、**CSV は 21位以降も全件出力**（公式ヘルプ明記） |
| 月確定タイミング | 月初に前月分が確定 |
| 当月途中ダウンロード | 値が変動（速報扱い） |

## 4. 出力ファイルの仕様

| 項目 | 仕様 |
|---|---|
| 文字コード | Shift_JIS / CP932 想定 |
| 改行 | CRLF（推定） |
| 区切り | カンマ |
| 拡張子 | `.csv` |
| 行粒度 | 1 行 = 1 コース × 月 |

### 必須カラム（`parseTabelogNoteCourses` が参照）

パーサが必須トークンとして検出: `コース種別`, `順位`

パーサが読むカラム:

```
日付, コース種別, 順位, 数量, コースID, コース名, 金額
```

- `日付` は `2026/04` のような月のみ（実機サンプルで確認）
- `金額` は `-` で欠損表現あり（パーサ側で `undefined` として処理）
- `順位` は同率の重複あり得る

## 5. 12 ヶ月遡及取得の作業見積もり

| 項目 | 見積もり |
|---|---|
| 1 ヶ月あたりの操作 | URL の `yearMonth` 書き換え → CSV ダウンロードボタン → リネーム → 配置 |
| 1 ヶ月あたりの所要時間 | **約 1.5 分** |
| 12 ヶ月分の総工数 | **約 18 分**（ログイン込みで 20〜25 分） |
| 一括取得の可否 | **不可**（月単位 1 ファイル） |
| 取得方針 | **月別 1 ファイル × 12 回** |

> **過去 12 ヶ月遡及の留意点**: §3 同様、保持期間の公式明示なし。最古月から DL 試行で取得可能範囲を確認すること。

## 7. 既知の落とし穴

| # | 内容 | 対処 |
|---|---|---|
| 1 | 画面は TOP20 のみ表示 → 21位以降の存在を見落としやすい | CSV はそのまま 21位以降も含まれる前提で運用 |
| 2 | `金額` カラムに `-`（欠損）が混在 | パーサ側で `undefined` 化済 |
| 3 | 順位の同率重複あり | 集計時は `数量` で並び替え可 |
| 4 | Excel で開いて再保存すると Shift_JIS → UTF-8 化 + 表記揺れの原因に | Excel で開かずそのままアップロード |
| 5 | コース ID は食べログ本体の予約コースマスタと突合可能だが、本マスタは媒体側のみ参照可 | 突合不要なら無視で OK |
| 6 | ノート連携店舗（他社台帳経由）では分析機能が無効 | 食べログノート直利用前提（要 user 確認） |

## 8. 自動化の見通し

| 項目 | 見通し |
|---|---|
| Playwright 自動化の所要工数 | **半日**（予約一覧 SOP と同じパターン。`yearMonth` ループで 12 回 fetch） |
| Selector の安定性 | 「CSV ダウンロード」ボタンは分析画面の標準 UI。安定 |
| ログイン状態保持 | `state/storageState.json` を食べログ本体スクレイパと共有可能性あり |
| 予約一覧 SOP との同時自動化 | **同一ログイン・同一画面遷移で複数 CSV を順次 DL 可能** → 1 セッションで「予約一覧 + 予約コース + 総客数」をまとめて取得すると効率的 |
