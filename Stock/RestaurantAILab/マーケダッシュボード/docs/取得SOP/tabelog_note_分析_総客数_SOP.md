## 1. ログイン URL と必要認証情報

| 項目 | 内容 |
|---|---|
| ログイン URL | https://owner.tabelog.com/owner_account/login/ |
| 認証方式 | 食べログ店舗会員 ID / パスワード（食べログ本体・食べログノート共通） |
| 2 要素認証 | 公式ヘルプ上では必須化記載なし |
| ID / PW の保管 | **要 user 確認** |
| セッション保持期間 | Cookie ベース |
| アカウントロック | 連続失敗で発生想定 |

## 2. ダウンロード手順（手動）

1. ブラウザで https://owner.tabelog.com/owner_account/login/ にアクセスしログイン
2. グローバルナビ／サイドメニューから「食べログノート」を選択
3. 左サイドナビの「分析」 → 分析トップに表示される **「予約数・総客数」** セクションを開く
   - 一次 URL: `https://owner.tabelog.com/tn/analysis?yearMonth=YYYY-MM`
   - 詳細 URL は推定: `https://owner.tabelog.com/tn/analysis/category/total?yearMonth=YYYY-MM`（要実機確認）
4. 集計単位を **「日別」** に切り替える（プルダウン or タブ切り替え）
5. 対象月を `yearMonth=YYYY-MM` で指定
6. 「CSV ダウンロード」ボタンをクリック
7. ファイル名（既定）: `YYYYMMDD_分析_総客数.csv`（実機サンプル: `20260626_分析_総客数.csv`）
8. 保存後、後述§6 の命名規則に従ってリネーム

## 3. 期間指定の限界

| 項目 | 仕様 |
|---|---|
| 過去の遡及範囲 | `yearMonth=YYYY-MM` で任意月指定可能。**過去保持期間の公式明示なし** |
| 1 リクエストあたり最大期間 | **月単位固定**（日別／月別の表示切り替えあり、CSV は当該月のみ） |
| 集計単位 | 日別 or 月別の選択可。**マーケダッシュボードは「日別」を採用**（曜日・週次集計のため） |
| 月またぎ境界 | 食べログノート独自の 06:00〜30:00 境界（要注意） |
| 月確定タイミング | 月初に前月分が確定 |

## 4. 出力ファイルの仕様

| 項目 | 仕様 |
|---|---|
| 文字コード | Shift_JIS / CP932 想定 |
| 改行 | CRLF（推定） |
| 区切り | カンマ |
| 拡張子 | `.csv` |
| 行粒度 | **1 行 = 1 日 × データ区分（組数 or 人数）**（つまり 1 日に 2 行ずつ） |

### 必須カラム（`parseTabelogNoteTotalGuests` が参照）

パーサが必須トークンとして検出: `日付`, `データ区分`

パーサが読むカラム:

```
日付, データ区分, 予約数, ウォークイン数, 合計
```

- `データ区分` は `組数` または `人数`
- パーサは `予約数` `予約` / `ウォークイン数` `ウォークイン` 両方の表記を吸収（フォールバック実装あり）

## 7. 既知の落とし穴

| # | 内容 | 対処 |
|---|---|---|
| 1 | 集計単位の切替（日別／月別）を忘れると月別集約値しか取れない | 必ず **日別** に切り替えてから DL |
| 2 | ウォークイン値は **店舗側がノート画面に手入力** したものだけがカウントされる | データ取得・UI 設計書 §2 の通り、ウォークイン比率は POS 突合で算出（ノート値は補助） |
| 3 | 06:00〜30:00 境界による深夜帯のズレ | UI 側では 00:00〜24:00 暦月で扱う（§1.3 の合意） |
| 4 | `データ区分` が「組数」「人数」混在し 1 日 2 行 | パーサ側で `kind` を区別済み |
| 5 | Shift_JIS のため Excel で開いて再保存すると文字化けの原因 | Excel で開かずアップロード |
| 6 | キャンセル人数は別途減算する設計（CSV の `合計` 自体はキャンセル除外後と推定） | 取り込み時に値の意味を要再確認 |

## 8. 自動化の見通し

| 項目 | 見通し |
|---|---|
| Playwright 自動化の所要工数 | **半日**（予約一覧／予約コースと同じパターン） |
| Selector の安定性 | 「日別／月別」切り替え UI が tab か select かで selector が変わる。**実機で要確認** |
| ログイン状態保持 | `state/storageState.json` を食べログ本体と共有可能性あり |
| 同時自動化 | 予約一覧 / 予約コース / 総客数 を **1 セッション・1 ログインで 3 CSV まとめて取得** が最効率 |
