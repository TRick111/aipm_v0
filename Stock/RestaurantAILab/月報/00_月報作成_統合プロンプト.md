# 月報作成 統合プロンプト（v2.0）

> **v2.0（2026-05-18）**: 5Arrowsフィードバック反映の恒久版。新パイプライン `Scripts/monthly_report_pipeline.py` + `Scripts/build_monthly_html.py` を使用する単純化された運用に切り替え。旧バージョンは `.bak_20260518` で残置。

---

## 目的

月報作成を **2コマンド** で完結させる。AIへの分析委譲は最小限にし、スクリプトで再現性を担保する。

---

## 必須参照ファイル（着手前に必ず読む）

| ファイル | 目的 |
|---|---|
| `./共通ルール.md` | データソース／集計軸／P1抽出キー／ボトル除外／客単価2系統／カラーパレット／分類条件帯ルール 等の全ルール |

## 入力情報

| 項目 | 値 | 例 |

## 実行手順（2コマンド）

```bash
# Step 1: 分析データ生成（analysis.json）
python ./Scripts/monthly_report_pipeline.py \
  --target-month 2026-04 \
  --store-code BFA \
  --sales-csv "/Users/.../会計明細_20260401-20260430 (1).csv" \
  --output-dir "./Flow/202605/2026-05-18/BFA/" \
  --compare-rawdata "./1_input/BFA/rawdata.csv" \
  --daily-report-csv "./1_input/BFA/DailyReport.csv"

# Step 2: HTML月報生成
python ./Scripts/build_monthly_html.py \
  --analysis-json "./Flow/202605/2026-05-18/BFA/analysis.json" \
  --output "./Flow/202605/2026-05-18/BFA/202604_BFA_月次報告資料_v2draft.html" \
  --store-name "BAR FIVE Arrows"
```

## 旧フェーズ構成（参考・bakファイル参照）

旧プロセスはフェーズ1〜5（基礎資料→週別深堀→スライド構成案→日報→HTML）に分かれていたが、v2.0ではスクリプトに統合した：

| 旧 | 新 |
|---|---|
| 旧フェーズ1〜2（基礎資料・週別深堀） | `monthly_report_pipeline.py`（一発でanalysis.json生成） |
| 旧フェーズ3〜4（スライド構成案・日報スライド） | `build_monthly_html.py` 内のスライドリスト定義（25枚） |
| 旧フェーズ5（HTML生成） | `build_monthly_html.py`（D3.js + ダークグリーン + 円グラフ） |

旧プロンプト（01〜05）は参考情報として残置するが、本プロセスでは使用しない。

## DoD（完了条件）

| 確認項目 | 基準 |
|---|---|
| 月次総売上 | エアレジ売上分析画面の表示と一致 |
| スライド数 | 25枚（タイトル含む） |
| 円グラフ | 比率6スライドすべてに描画 |
| 凡例 | 全グラフに配置（単一系列でも単位＋出典の補助凡例） |
| メッセージ帯 | 全本編スライドに「動詞のある1〜2文」 |
| 分類条件帯 | 全本編スライドに箇条書きで集計条件のみを記載（自明な定義・他ページの話は書かない） |
| カラー | ダークグリーン基調（#1B4D3E / #C9A961 / #2D6B4F / #8B3A3A） |
| キャンバス | 1280×840px（PDF印刷時も改ページ正常） |

詳細は `共通ルール.md` §8 参照。
