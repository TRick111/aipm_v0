## 手順（AIが実行）

### 1) 入力データの存在確認
- `{入力売上}` と `{入力口コミ}` が存在すること
- `{入力売上}` に `{対象月}` の全営業日データが含まれること（不足していたら「rawdata更新が必要」と明示）

### 2) 対象月に含まれるISO週の算出
- `{開始日}` 〜 `{終了日}` の営業日が含まれるISO週を列挙する
- 例: 2026年1月 → `2026-W01`（1/1〜）〜 `2026-W05`（〜1/31）
- 月初/月末がISO週の途中にまたがる場合は、**営業日ベース**で月に帰属させる

### 3) 月報スクリプトの実行

`monthly_analysis_script.py` を実行して数値データを自動生成する。

```bash
python ./Scripts/monthly_analysis_script.py \
  -m {対象月} \
  --sales-data {入力売上} \
  --reviews-data {入力口コミ} \
  -o {出力ディレクトリ}
```

**スクリプトが出力するファイル**:

| ファイル | 内容 |
|----------|------|
| `slide_data_part1.txt` | 月次KPI・6ヶ月トレンド・前年同月比較 |
| `slide_data_part2.txt` | 週別推移・好調/不調週・曜日別日平均・時間帯別 |
| `slide_data_part3.txt` | カテゴリ構成比・商品TOP15・構成比増減 |
| `analysis_results.json` | 全分析結果（JSON） |
| `monthly_sales.csv` | 月次売上（全期間） |
| `weekly_in_month.csv` | 月内の週別推移 |
| `weekday_monthly.csv` | 曜日別集計（対象月） |
| `hourly_monthly.csv` | 時間帯別集計（対象月） |
| `category_monthly.csv` | カテゴリ別月間売上 |
| `product_monthly.csv` | 商品別月間売上 |
| `reviews_monthly.csv` | 対象月の口コミ |
| `analysis_memo.txt` | 分析メモ |

> スクリプト実行後、AIは出力データをもとに **分析コメント・インサイト** を付与して月報基礎資料を作成する。

### 4) スクリプト出力の確認と補足分析

スクリプトが自動生成する指標を確認し、必要に応じてAIが補足分析を加える。

#### 4-1. 月次KPI（`slide_data_part1.txt` → 月間サマリー用KPI）

スクリプト出力のKPIテーブルを確認:
| 指標 | 内容 |
|------|------|
| 売上 | 対象月の総売上 |
| 客数 | 対象月の総客数 |
| 客単価 | 月間売上 ÷ 月間客数 |
| 営業日数 | 対象月の実営業日数 |
| 日平均売上 | 月間売上 ÷ 営業日数 |

**比較対象**（スクリプトが自動算出）:
- **前月比**: 前月との比較
- **前年同月比**: 前年同月との比較

**AIが補足**: 比較数値の解釈・背景要因のコメント

#### 4-2. 月次トレンド（`slide_data_part1.txt` → 直近6ヶ月の推移データ）
- 直近6ヶ月の月次売上・客数・客単価・組数・営業日数・日平均売上
- 前年同月との比較表

**AIが補足**: トレンド（上昇/下降/横ばい）の読み取りと要因分析

#### 4-3. 週別推移（`slide_data_part2.txt` → 月内の週別推移、`weekly_in_month.csv`）
- 月内の各ISO週の売上・客数・客単価・営業日数・日平均売上・前年同週比
- 好調週 / 不調週の特定（日平均売上ベース）

**AIが補足**: 月内での波のパターンと要因解釈

#### 4-4. 曜日別集計（`slide_data_part2.txt` → 曜日別日平均、`weekday_monthly.csv`）
- 曜日別の日平均売上・日平均客数・客単価・営業日数
- 前月比（売上）

**AIが補足**: 強い曜日/弱い曜日の特定、前月比の解釈

#### 4-5. 時間帯別集計（`slide_data_part2.txt` → 時間帯別月間売上、`hourly_monthly.csv`）
- 時間帯バケット別（18-20/20-22/22-24/24-26時）の月間売上・構成比・客数

**AIが補足**: ピーク時間帯とアイドルタイムの特定

#### 4-6. カテゴリ別分析（`slide_data_part3.txt`、`category_monthly.csv`）
- カテゴリ別の月間売上・構成比・販売数
- 前月比での構成比変化（ポイント）

**AIが補足**: 成長/縮小カテゴリのコメント

#### 4-7. 商品別分析（`slide_data_part3.txt`、`product_monthly.csv`）
- 商品別売上TOP15（月間）
- 構成比と前月比の変化
- 構成比増減TOP5

**AIが補足**: 新規ランクイン/ランクアウトの特定

### 5) 月報作成基礎資料（Markdown）を作成
- スクリプト出力（`slide_data_part1〜3.txt` + 各種CSV）のデータと、AIの分析コメント・インサイトを統合し、**月報作成基礎資料** としてMarkdownで作成する
- `分析観点.md` に記載の判断基準に従って解釈・コメントを付与する
- 保存先: `{出力ディレクトリ}/月報作成基礎資料.md`

### 1. 売上分析

#### 1.1 月次サマリー（KPI）
- 月間売上・客数・客単価・組数の実績
- 前月比・前年同月比
- **インサイト抽出**:
  - ✅ ポジティブな傾向（成長、改善）
  - ⚠️ 懸念事項（減少、悪化）
  - 💡 推奨アクション（具体的な施策）

#### 1.2 週別推移分析（月内トレンド）
- 月内の各週の売上・客数・客単価の推移
- 月内でのトレンド（月初/中旬/月末の傾向）
- 特異週の特定（イベント・天候等の影響）
- **インサイト抽出**:
  - ✅ 好調週の要因
  - ⚠️ 不調週の要因
  - 💡 月内の波を平準化する施策

#### 1.3 曜日別分析（月全体の傾向）
- 曜日別の日平均売上・客数・客単価
- 前月の曜日別平均との比較
- 強い曜日/弱い曜日の特定
- **インサイト抽出**:
  - ✅ 安定して好調な曜日
  - ⚠️ 継続的に不調な曜日
  - 💡 曜日別の集客・単価向上施策

#### 1.4 時間帯別分析（月全体の傾向）
- 時間帯別（18-26時）の月間売上・客数
- ピーク時間帯の特定
- 前月との時間帯分布比較
- **インサイト抽出**:
  - ✅ ピーク時間帯の強み
  - ⚠️ アイドルタイムの存在
  - 💡 時間帯別の施策

---

### 2. 商品・カテゴリ分析

#### 2.1 カテゴリ別分析（月間）
- カテゴリ別の月間売上・構成比
- 前月比での構成比変化
- 前年同月比での構成比変化
- **インサイト抽出**:
  - ✅ 成長カテゴリ
  - ⚠️ 縮小カテゴリ
  - 💡 カテゴリ戦略の提案

#### 2.2 商品別分析（月間TOP15）
- 商品別売上TOP15の月間実績
- 構成比と前月比の変化
- 新規ランクイン/ランクアウト
- **インサイト抽出**:
  - ✅ 安定した人気商品
  - ⚠️ 急落した商品、特定商品への依存
  - 💡 商品戦略の提案

---

### 3. 口コミ分析（月間）

- 対象月の新規投稿数と平均評価点
- 月内の投稿トレンド（週ごとの投稿数推移）
- ポジティブ/ネガティブな傾向の月間サマリー
- 前月比での評価変動
- **インサイト抽出**:
  - ✅ 高評価の要因
  - ⚠️ 低評価・クレームの傾向
  - 💡 顧客満足度向上の施策

---

### 4. 総合的な考察とアクションプラン

#### 4.1 月間総括
- **強み**: 各分析から得られた✅ポジティブ要素をまとめる
- **課題**: 各分析から得られた⚠️懸念事項をまとめる
- **優先順位**: 深刻度×影響度で課題を整理

#### 4.2 アクションプラン
- **短期施策（翌月中）**: 即効性・低コスト・既存リソースで対応可能
- **中期施策（2-3ヶ月）**: システム導入・投資が必要
- **長期施策（3-6ヶ月）**: 戦略的な方向性転換

#### 4.3 KPI設定（翌月目標）
- 前月実績をベースに、測定可能で達成可能な目標を設定
- 売上・客数・客単価それぞれの目標値と根拠
