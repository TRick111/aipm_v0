添付のスライド構成案をもとに、以下の手順に沿って、飲食店の売り上げに関する週次レポートを作成してください。


## 手順

### 1) スライド構成案の読み込みと確認
- `{スライド構成}` を全文読み込む
- スライド総数を確認し、控える（例: 34枚）
- **スライド枚数は構成案に厳密に従う**（追加・省略は一切不可）
- ファイル名は対象期間をファイル名に入れ、「yyyymmdd_yyyymmdd＜店名＞週次報告資料」とする。例：「20260126_20260201_BAR_FIVE_Arrows_週次報告資料」

### 2) デザイン方針


#### 基本原則

1. **情報の簡潔性**: 1スライドには1つの主要なメッセージのみを盛り込み、情報を詰め込みすぎない
2. **段階的な説明**: 複雑な内容は複数のスライドに分割し、段階的に理解を促す
3. **焦点の明確化**: グラフやデータを含むスライドでは、最も注目すべき点を視覚的にハイライトする

---
#### 2.1 カラーパレット

元のデザインで使用されているカラーパレットは、情報の種類を直感的に伝える役割を果たしています。

| 用途 | 色 | HEXコード | プロンプト指示例 |
| :--- | :--- | :--- | :--- |
| **主要カラー（ヘッダー等）** | ダークネイビー | `#1E3A5F` | 「ダークネイビーの背景」「信頼感のあるプロフェッショナルな雰囲気」 |
| **ポジティブ・達成** | グリーン | `#22C55E` | 「ポジティブな数値は明るいグリーンで」「達成を#22C55Eでハイライト」 |
| **注目・変化** | オレンジ | `#F97316` | 「強調部分は鮮やかなオレンジで」「変化を#F97316でハイライト」 |
| **背景** | ライトグレー | `#F1F5F9` | 「清潔感のあるライトグレー背景（#F1F5F9）」 |

#### 2.2 タイポグラフィとテキスト

飲食店店長様向けに、可読性とインパクトを両立させます。

- **数値の強調**: 「数字『4.2』を非常に大きく、太字で、グリーンのフォントで表示」のように、サイズ・太さ・色を具体的に指定
- **シンプルな言葉選び**: 専門用語を避け、「売上」や「お客様の声」といった平易な言葉でプロンプトを作成
- **情報量の制限**: 1スライドあたりのテキストは、タイトル、キーメッセージ、2〜3の補足情報に留める

#### 2.3 レイアウトと構成

元のデザインに見られる2カラムレイアウトやカード形式は、情報を整理し、視線を誘導するのに効果的です。

- **2カラム**: 「2カラムレイアウトで作成。左カラムは定量データ、右カラムは定性分析」
- **カード形式**: 「スライド下部に3つの主要指標を別々のカードで表示」
- **焦点の明確化**: 「問題から解決策への結果を示す大きな矢印を使用」のように、視覚的な誘導を指示

#### 2.4 グラフとデータの可視化

グラフの「どこを見るべきか」を明確にすることが最も重要です。

- **シンプルなグラフ**: 「過去6ヶ月のトレンドを示すシンプルな折れ線グラフを生成」
- **ハイライト**: 「棒グラフで『12月15日』の棒をハイライトし、『SNSキャンペーンで来客数急増』と吹き出しを追加」
- **色分け**: 「複合グラフで売上はグリーン、来客数はブルーを使用」

### 3) データの正確性（最重要）

#### 表データの完全一致
- スライドに表を含める場合、表内の文字・数値は `{スライド構成}` の記載と **1文字も違わず一致** させること
- 構成案の表をスライドに転記する際は、必ずセル単位で照合すること

#### 固有名詞の正確性
- 商品名・カテゴリ名・店舗名などの固有名詞は、表記揺れ・誤字・脱字が **絶対にない** よう、構成案の記載を正確に転記すること
- 以下のようなミスは不可:
  - 英字の大文字/小文字の違い（例: `Hojicha Negroni` → `Hojicha negroni` は NG）
  - 漢字/ひらがなの表記揺れ（例: `山葵と梨のスマッシュ` → `わさびと梨のスマッシュ` は NG）
  - 中点・スペースの有無（例: `ジャパニーズカクテル` → `ジャパニーズ・カクテル` は NG）
  - カテゴリ名の省略（例: `ウイスキー（ハイボール・ボトル・オリジナル）` → `ウイスキー（HB・ボトル）` は NG）

#### 数値の正確性
- 金額・パーセンテージ・人数などは構成案の値をそのまま使用する
- 丸め・再計算・独自の集計は行わない

### 4) 出力形式
- **NanoBanana形式** の画像で出力する
- **Manus上で編集可能** な状態を保つこと（テキスト・グラフがオブジェクトとして選択・編集できること）
- 1スライド = 1画像として出力
- アスペクト比: 16:9（標準プレゼンテーション）


### 5) セルフチェック（作成後に必ず実行）

以下の全項目を確認してからファイルを保存すること:

- [ ] スライド枚数が構成案と一致しているか
- [ ] 各スライドのタイトルが構成案と一致しているか
- [ ] 表内の文字・数値が構成案と完全に一致しているか（1セルずつ照合）
- [ ] 商品名・カテゴリ名に表記揺れがないか（構成案と1文字ずつ比較）
- [ ] 各スライドのメッセージが一目で理解できるか（文字量が過剰でないか）
- [ ] NanoBanana形式で出力されているか
- [ ] Manus上で編集可能な状態か

---

## 注意事項

1. **スライド枚数厳守**: 構成案のスライド枚数を1枚たりとも変えないこと。スライドの統合・分割は不可
2. **テキスト最小化**: 忙しいオーナーが一目で分かるデザインにする。長文は禁止
3. **固有名詞の正確性**: 商品名・カテゴリ名は1文字単位で構成案と照合する。推測や言い換えは不可
4. **表データの転記精度**: 表を含むスライドは、作成後にセル単位で構成案と突き合わせ確認する
5. **NanoBanana + Manus編集可能**: 出力形式を必ず守る。画像化されたテキストではなく、編集可能なオブジェクトとして出力すiる
