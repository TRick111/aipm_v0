# Manus Nanobananaスライド デザイン方針ガイド

## 1. はじめに

本ドキュメントは、コンサルティングレポート（PDF）のデザインを、ManusのNanobananaスライド（画像生成モード）を用いて再現するためのデザイン方針を定義するものです。主な読者である飲食店の店長様が直感的かつ迅速に内容を理解できるよう、シンプルで視覚的なコミュニケーションを重視したスライドデザインを目指します。

### 基本原則

1. **情報の簡潔性**: 1スライドには1つの主要なメッセージのみを盛り込み、情報を詰め込みすぎない
2. **段階的な説明**: 複雑な内容は複数のスライドに分割し、段階的に理解を促す
3. **焦点の明確化**: グラフやデータを含むスライドでは、最も注目すべき点を視覚的にハイライトする

---
## 3. デザイン要素の再現方針

### 3.1 カラーパレット

元のデザインで使用されているカラーパレットは、情報の種類を直感的に伝える役割を果たしています。

| 用途 | 色 | HEXコード | プロンプト指示例 |
| :--- | :--- | :--- | :--- |
| **主要カラー（ヘッダー等）** | ダークネイビー | `#1E3A5F` | 「ダークネイビーの背景」「信頼感のあるプロフェッショナルな雰囲気」 |
| **ポジティブ・達成** | グリーン | `#22C55E` | 「ポジティブな数値は明るいグリーンで」「達成を#22C55Eでハイライト」 |
| **注目・変化** | オレンジ | `#F97316` | 「強調部分は鮮やかなオレンジで」「変化を#F97316でハイライト」 |
| **背景** | ライトグレー | `#F1F5F9` | 「清潔感のあるライトグレー背景（#F1F5F9）」 |

### 3.2 タイポグラフィとテキスト

飲食店店長様向けに、可読性とインパクトを両立させます。

- **数値の強調**: 「数字『4.2』を非常に大きく、太字で、グリーンのフォントで表示」のように、サイズ・太さ・色を具体的に指定
- **シンプルな言葉選び**: 専門用語を避け、「売上」や「お客様の声」といった平易な言葉でプロンプトを作成
- **情報量の制限**: 1スライドあたりのテキストは、タイトル、キーメッセージ、2〜3の補足情報に留める

### 3.3 レイアウトと構成

元のデザインに見られる2カラムレイアウトやカード形式は、情報を整理し、視線を誘導するのに効果的です。

- **2カラム**: 「2カラムレイアウトで作成。左カラムは定量データ、右カラムは定性分析」
- **カード形式**: 「スライド下部に3つの主要指標を別々のカードで表示」
- **焦点の明確化**: 「問題から解決策への結果を示す大きな矢印を使用」のように、視覚的な誘導を指示

### 3.4 グラフとデータの可視化

グラフの「どこを見るべきか」を明確にすることが最も重要です。

- **シンプルなグラフ**: 「過去6ヶ月のトレンドを示すシンプルな折れ線グラフを生成」
- **ハイライト**: 「棒グラフで『12月15日』の棒をハイライトし、『SNSキャンペーンで来客数急増』と吹き出しを追加」
- **色分け**: 「複合グラフで売上はグリーン、来客数はブルーを使用」
