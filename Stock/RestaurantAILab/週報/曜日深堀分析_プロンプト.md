# 曜日深堀分析プロンプト

## 目的
週報作成後、特定の曜日（好調日と不調日）を深掘り分析する。

## 前提条件
- 週報基礎資料が作成済みであること
- 分析対象週のrawdataが存在すること

## プロンプトテンプレート

```
1.2（曜日別分析）をより詳しく見てほしい。
最も好調だった日と最も不調だった日をピックアップして、以下の観点で分析を行って。

深掘り対象となった曜日については、以下をさらに分析する：

1. パーティサイズ分析: 1名/2名/3-4名/5名以上の構成変化
2. 時間帯バケット分析: ピーク時間（18-19、20-21等）の人数・売上推移
3. 会計レベルの確認: 大口会計の有無、単価分布（中央値、P10、P90）
4. 商品ミックス分析: 高単価商品の販売数、カテゴリ別売上構成
5. 例外データの確認: 営業時間外会計、特殊計上の混入
```

## 分析の流れ

### Step 1: 対象日の特定
`weekday_decomposition.csv` から以下を確認:
- **最も好調だった日**: 売上差が最大プラスの曜日
- **最も不調だった日**: 売上差が最大マイナスの曜日

### Step 2: 深掘り分析の実行
以下の5つの観点で両日を比較分析:

#### 1. パーティサイズ分析
- 1名/2名/3-4名/5名以上の会計数・客数・売上を集計
- 構成比を計算し、グループ客の有無を確認
- 平均客単価をパーティサイズ別に算出

#### 2. 時間帯バケット分析
- 時間帯別（営業時間18-26時）の会計数・客数・売上を集計
- ピークタイムの特定と空白時間帯の確認
- 平均売上/会計を時間帯別に算出

#### 3. 会計レベル分析
- 最大会計、最小会計、中央値、P10、P90、平均を算出
- 会計金額分布（~5千円、5千~1万、1万~1.5万、1.5万~2万、2万円以上）
- 高額会計の有無と構成比

#### 4. 商品ミックス分析
- カテゴリ別売上構成TOP3
- 高単価商品（¥1,500以上）の販売数と総売上
- 飲み放題プランの有無
- 人気商品のリピート状況

#### 5. 例外データ確認
- 営業時間外会計（深夜0-6時）の有無と金額
- 1名あたり¥10,000超の会計数
- 10名以上のパーティ有無

### Step 3: 比較レポート作成
以下のセクションで構成:
1. サマリー比較表（売上、客数、会計数、客単価等）
2. 5つの観点別の詳細分析
3. 総合考察（不調だった日の主要因）
4. 改善アクションプラン（優先度付き）

## 期待される成果物

### Markdownレポート
週報基礎資料の「1.2 曜日別分析」セクションに追加:
- `#### 深掘り分析: 水曜日(好調) vs 金曜日(不調)` のような見出し
- サマリー比較表
- 5つの観点別詳細分析
- 総合考察と改善アクション

### Pythonスクリプト出力
- 各曜日の詳細分析結果（テキスト形式）
- パーティサイズ分布
- 時間帯別売上
- 会計レベル統計
- 商品ミックス

## 使用するツール

### 自動分析の場合
```bash
python deep_dive_weekday_analysis.py -w 2026-W01 -e 2026-01-03 --best-date 2025-12-31 --worst-date 2026-01-02
```

## 注意事項

1. **営業日基準の日付**: 深夜0-5時は前日の営業日としてカウント
2. **会計レベルの集計**: トランザクションレベルではなく、account_id単位で集計
3. **カレンダー曜日 vs ビジネス日付**: business_dateベースで分析
4. **ISO 8601週番号**: 年をまたぐ週は翌年の週番号（例: 2025-12-29〜2026-01-03 = 2026-W01）

## カスタマイズポイント

### パーティサイズの区分変更
デフォルト: 1名/2名/3-4名/5名以上
→ 店舗特性に応じて変更可能

### 高単価商品の閾値変更
デフォルト: ¥1,500以上
→ 店舗の価格帯に応じて変更可能

### 会計金額分布の区分変更
デフォルト: ~5千円、5千~1万、1万~1.5万、1.5万~2万、2万円以上
→ 店舗の客単価に応じて変更可能
