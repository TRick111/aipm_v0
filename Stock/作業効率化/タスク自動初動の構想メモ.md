# タスク自動初動の構想メモ

記録日: 2026-03-25

## 着想

タスクには「発生してから初動がある程度決まっている」ものが多く、その初動の一部はAIで対応できる。

**基本フロー:**
1. タスクが登録される
2. AIがそのタスクに対する「初動プロンプト」を自動生成する
3. 人間がプロンプトの内容を確認・承認する
4. 承認されたらAIが実行し、結果を返す

## Tachyonとの関係

この構想は **Tachyonの概念を会議の枠を超えて拡張したもの** と位置づけられる。

- **Tachyon（現在）**: 会議中の音声 → 文字起こし → AIがタスク提案 → 承認 → 実行
- **本構想**: タスク登録 → AIが初動プロンプト提案 → 承認 → 実行

共通するのは「AIが次のアクションを提案し、人間が承認して実行する」というループ。Tachyonでは会議がトリガーだが、本構想ではタスク登録自体がトリガーになる。

## 今後の検討事項

- 具体的な実装形態（AIOS内の仕組み / 独立ツール / Tachyonの拡張）
- どのようなタスクが「初動が定型的」に該当するかの整理
- 承認フローのUI/UX（LINE通知？ダッシュボード？）
- Tachyonとの統合 or 別プロジェクト化の判断
