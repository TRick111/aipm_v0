# マンション内LAN

## 🎯 次のアクション
Cloudflare WARP（無料）を Mac / iPhone に導入して 1〜2 日試用し、「初回アクセス遅延」が消えるかを評価する。MTU は 1500 のまま、他ソフトへの影響なく改善できるかが第一の検証ポイント。WARP で改善するなら本採用、改善しなければマンション管理会社への問い合わせ（ダブル NAT 解消・上位ルーターのセッション制限緩和・ICMP 透過）に進む。

## 🚧 現在のブロッカー
なし

## 📋 概要
集合住宅で発生する自宅 Wi-Fi の「初回アクセス遅延」問題の真因解明と恒久対策プロジェクト。MTU 1024 で改善することは 2026-04-22 に確認済だが、他ソフトへの影響があるため MTU を変えずに直す方法を探る。

2026-05-04 の再診断で、(1) ダブル NAT 構成、(2) ICMP の重度レート制限／フィルタ、(3) TCP SYN がランダムにロスして 1 秒の再送ペナルティが発生、という新しい所見を得た。「初回が遅く 2 回目以降が速い」体感は HTTP keep-alive・DNS / TLS セッションキャッシュで新規接続フェーズが省略されるため。

関係者: 田中利空（オーナー / 実行者）、マンション管理会社（上位 NAT 機器の運用主体）。

## 🧠 決定事項（Why ログ）
- MTU 1024 への恒久変更は採用しない: Tailscale 等の他ソフトへの影響と、根本原因解決にならないため
- IPv6 は「使用しない」のまま維持: 中途半端な IPv6 開通は Happy Eyeballs の挙動でむしろ症状を悪化させる
- 個別回線契約は当面検討対象外: 賃貸かつ管理会社経由のため初手にしない
