# ネットワーク最適化診断

**日時**: 2026-04-22

---

## 重大な発見: MTUミスマッチ

### 問題

集合住宅ゲートウェイ(192.168.6.241)が **MTU 1024** に制限しているが、
ルーター(Buffalo WSR-A2533DHP3)のWAN側MTUは **1500** に設定されている。

この不一致により:
- 1024バイトを超えるパケットが集合住宅GWでドロップされる
- TCP接続の初期ハンドシェイクは小さいパケットなので成功する
- データ転送が始まるとMSS 1460のセグメントが送られ、ドロップ→再送→タイムアウトが発生
- 結果: 一部サイトでTCP接続に数秒〜7秒のペナルティが発生

### 証拠

```
Ping MTU テスト（宛先: 1.1.1.1）
  996B (+28 header = 1024): 0% loss ← 通る
  997B (+28 header = 1025): 100% loss ← 通らない

集合住宅GW(192.168.6.241)への同一テスト
  996B = 0% loss
  1000B = 100% loss
  → 制限は集合住宅GWで発生

ルーター(192.168.11.1)へのテスト
  1472B = 0% loss ← LAN内は問題なし
```

### 影響

| サイト | デフォルト(MTU 1500) | IPv4強制 |
|--------|---------------------|----------|
| google.com | **7.57秒** (TCP接続で7.5秒停滞) | 0.066秒 |
| cloudflare.com | 1.04秒 | 0.04秒 |
| yahoo.co.jp | 2.03秒 | 0.04秒 |
| github.com | 2.10秒 | — |

注: IPv4強制時に速いのは、IPv6フォールバックの回避とPMTU Discoveryの挙動の組み合わせ。

## 対策

### 対策1: ルーターのWAN側MTUを1024に変更（推奨度: ★★★ 最優先）

ルーター管理画面 → Internet → [拡張設定] → Internet側MTU値
- 現在: **1500**
- 変更先: **1024**

**効果:**
- ルーターが送信するパケットが1024B以下に制限される
- 集合住宅GWでのパケットドロップが解消
- 全サイトへのTCP接続が高速化（7.5秒 → 0.07秒レベル）
- 全デバイスに効果あり

**副作用:**
- スループットが微減（ヘッダ比率が上がるため、理論上約2%の効率低下）
- 実用上は無視できるレベル

### 対策2: Mac側のMTUを1024に変更（補助）

```bash
sudo ifconfig en1 mtu 1024
```

または: システム設定 → ネットワーク → Wi-Fi → 詳細 → ハードウェア → MTU を 1024 に設定

**効果:** Mac単体での即時改善。ルーター設定変更前の暫定対策として。
