# 情報インプット

## 背景
- 日々のSNS・ニュース・技術記事等で「あとで読みたい」情報が発生するが、ブックマークが散逸し読まれないまま埋もれることが多い。
- AIとの対話で「読みたい」と言うだけでリストに追加し、読了したら消化できるシンプルな運用が必要。

## 目的
- 読みたい記事・ポスト・動画等のURLをリスト管理し、未読/読了の状態を追跡する。
- AIとの対話だけでリストの追加・消化（読了マーク）を完結させる。

## ゴール（完了条件）
- 「あとで読みたい」と言えばリストに追加される運用が回っている
- 読了後に「読んだ」と言えばリストから外れる運用が回っている

## 運用ルール
- **追加**: ユーザーが URL や情報を提示し「読みたい」「あとで見る」等と発言 → AIが `reading_list.md` に追記
- **消化**: ユーザーが「読んだ」「消化した」等と発言 → AIが該当アイテムのステータスを `読了` に変更、または行を削除
- **リスト確認**: ユーザーが「読みたいリスト見せて」等 → AIが `reading_list.md` を表示
